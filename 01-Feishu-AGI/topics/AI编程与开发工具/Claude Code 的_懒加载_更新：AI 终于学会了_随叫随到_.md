### 📋 核心分析

**战略价值**: Claude Code 通过 Tool Search「懒加载」机制，将 MCP 工具的上下文占用从 15 万 token 压缩至 2000 token（节省 98.7%），彻底解除大规模 MCP 工具集成的上下文瓶颈。

**核心逻辑**:

- Claude 上下文窗口约 20 万 token，相当于一本 300 页书的信息量，是 AI 单次对话的硬性上限
- 工具说明书（tool definitions）是上下文的主要消耗源之一，7 个以上 MCP 服务器的工具定义可占用 67,000+ token，即上下文的 1/3
- MCP（Model Context Protocol）是 Anthropic 推出的统一工具接入标准，类比 AI 的 App Store，社区已开发数千个 MCP 服务器
- 旧机制是「全量预加载」：每次对话开始前，所有工具定义全部注入上下文，无论本次对话是否用到
- Tool Search 触发条件：当所有 MCP 工具说明书总量**超过上下文窗口的 10%** 时，自动切换为懒加载模式
- 懒加载机制：工具不预先注入，而是维护一份「工具通讯录」，AI 按需搜索并调用，中间结果不占用主上下文
- 实测效果：同等工具集场景下，上下文占用从 150,000 token → 2,000 token，**节省 98.7%**
- 对用户无感：工具数量未超过 10% 阈值时，行为与旧版完全一致，无需任何配置
- MCP 开发者需重点维护 `server instructions` 字段，这是 AI 在懒加载模式下判断「何时调用该工具」的核心依据，以前可忽略，现在是关键
- Cloudflare 提出的「Code Mode」更激进：AI 直接写脚本调用工具（如从 Google Drive 取数据直存 Salesforce），中间结果完全绕过模型上下文，效率提升更显著

### 🎯 关键洞察

**为什么 10% 是触发阈值？**
原因：20 万 token 的 10% = 2 万 token，这是 Anthropic 判断工具定义开始「显著挤压」有效对话空间的临界点。低于此值，预加载的即时性优于懒加载的搜索开销；超过此值，懒加载的空间收益远大于搜索延迟成本。

**懒加载 vs 预加载的本质差异**：

| 维度 | 预加载（旧） | 懒加载 Tool Search（新） |
|------|------------|------------------------|
| 工具注入时机 | 对话开始前全量注入 | 按需搜索后单次注入 |
| 上下文占用 | 全部工具定义（如 67,000 token） | 仅当前调用工具定义（约 2,000 token） |
| 适用场景 | 工具少（< 上下文 10%） | 工具多（> 上下文 10%） |
| 用户感知 | 无需操作 | 无需操作（自动切换） |
| 开发者影响 | `server instructions` 可忽略 | `server instructions` 必须认真写 |

**Code Mode（Cloudflare 方案）的逻辑**：
传统接力赛路径：`工具A → 模型 → 工具B → 模型`（每步结果都过模型，占上下文）
Code Mode 路径：`模型写脚本 → 脚本直接执行 A→B`（中间结果完全不经过模型）
本质是把「模型作为数据中转站」变成「模型作为代码生成器」，上下文只存代码，不存数据。

### 🛠️ 操作流程

1. **准备阶段（MCP 开发者）**: 检查并完善每个 MCP 服务器的 `server instructions` 字段，明确描述「该工具适用于什么场景、能做什么操作」，这是懒加载模式下 AI 检索工具的唯一线索。

2. **核心执行（Claude Code 用户）**: 正常配置所有需要的 MCP 服务器，无需任何额外操作。当工具总定义量超过上下文 10%（约 2 万 token），Claude Code 自动切换 Tool Search 模式，工具按需调用。

3. **验证与优化**: 观察长对话中 AI 是否出现「忘记前文」的情况——若上下文空间被有效释放，长对话稳定性应显著提升。如发现某工具未被正确调用，优先检查该工具的 `server instructions` 描述是否足够清晰。

### 📝 避坑指南

- ⚠️ MCP 开发者若不更新 `server instructions` 字段，在懒加载模式下工具可能被 AI 忽略，导致功能失效，这是最高优先级的适配项
- ⚠️ Tool Search 是自动触发机制，不存在手动配置入口，不要试图在配置文件里找开关
- ⚠️ 懒加载模式下，AI 每次调用工具前有一次搜索开销，对响应速度有微小影响，工具数量极多时更明显
- ⚠️ Code Mode 目前是 Cloudflare 的实践方案，并非 Claude Code 原生功能，需要工具链支持脚本直接执行
