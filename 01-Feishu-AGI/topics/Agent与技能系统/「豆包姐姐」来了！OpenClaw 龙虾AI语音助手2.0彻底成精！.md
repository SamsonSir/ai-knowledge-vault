### 📋 核心分析

**战略价值**: OpenClaw Assistant 2.0 将桌面 AI 助手从「纯语音交互」升级为「语音 + 全桌面视觉理解」，基于豆包 Seed 2.0 Pro 多模态模型，实现任意应用场景下的屏幕感知与实时语音讲解。

**核心逻辑**:
- 1.0 只有语音控制，2.0 新增屏幕视觉理解，核心差异是接入了豆包 Seed 2.0 Pro 多模态模型，让 AI 能直接"看"屏幕内容（代码、PDF、视频、图片）
- 技术栈组合：TEN Framework + Electron（桌面壳）+ Deepgram Nova-2（语音转写）+ MiniMax 流式 TTS（语音合成）+ 豆包 Seed 2.0 Pro（多模态分析）
- 截图流程共 7 步：用户触发 → 隐藏助手窗口（防止自身出现在截图中）→ 系统 API 截屏 → 图像压缩到合适分辨率 → Base64 编码 → 调用 Seed 2.0 API → 返回结果 + 语音合成输出
- 三种截图模式各有用途：单图截图（即时分析）、连续截图（多张关联解读）、视频学习模式（默认每 30 秒自动截图，智能去重，结束后生成完整笔记）
- 语音链路关键优化：Deepgram 用 WebSocket 实现毫秒级实时转写；MiniMax 流式 TTS 第一个音节到达即开始播放，无需等全段生成；AudioWorklet 在独立线程处理音频，不阻塞主界面
- 选择 Seed 2.0 Pro 的三个具体理由：代码/图表/文档识别准确率高、平均响应时间 2 秒以内、中文内容理解优化
- 三个角色（小龙虾、Amy 姐、豆包姐姐）核心功能相同，差异仅在音色和表达风格的 Prompt 界定
- 性能优化策略：智能采样 + 增量更新（只在需要时截图）、按用途选择不同截图分辨率、图像压缩 + 智能裁剪（降低 API 延迟同时保证识别率）
- 上下文连续性通过"视觉记忆"机制实现，AI 能记住之前看到的内容，保持对话连贯
- 1.0 已开源（GitHub：`https://github.com/andyhuo520/openclaw-assistant-mvp`），2.0 预计春节后开源，包含完整多模态 + 全桌面控制代码

### 🎯 关键洞察

**为什么「隐藏助手窗口」是截图流程的必要步骤**：如果不在截图前隐藏 UI，AI 分析的内容会包含自身窗口，干扰对目标内容的理解。这是桌面级 AI 助手与浏览器插件的核心工程差异之一——浏览器插件天然隔离在页面之外，而桌面应用必须主动处理自身遮挡问题。

**视频学习模式的去重逻辑价值**：30 秒定时截图如果不做智能去重，静止画面会产生大量重复帧，既浪费 API 调用，也会让生成的笔记充斥冗余内容。去重后只保留有实质变化的帧，笔记质量和 token 效率都大幅提升。

**AudioWorklet 的必要性**：传统 Web Audio API 在主线程处理音频，一旦 UI 有渲染压力（比如 AI 返回大段文字时），音频会卡顿。AudioWorklet 将音频处理移到独立线程，彻底解耦，是流式 TTS 体验流畅的底层保障。

### 📦 配置/工具详表

| 模块/功能 | 关键设置/代码 | 预期效果 | 注意事项/坑 |
|----------|-------------|---------|-----------|
| 语音转写 | Deepgram Nova-2，WebSocket 连接 | 毫秒级实时转写 | 需保持 WebSocket 长连接稳定 |
| 语音合成 | MiniMax 流式 TTS | 第一音节即播放，无等待感 | 需支持流式接收，不能用普通 HTTP 请求 |
| 音频处理 | AudioWorklet（独立线程） | 不阻塞主界面渲染 | 浏览器/Electron 版本需支持 AudioWorklet API |
| 多模态分析 | 豆包 Seed 2.0 Pro API，图像 Base64 编码传入 | 平均 2 秒内返回分析结果 | 截图前需压缩分辨率，否则 payload 过大影响延迟 |
| 截图 - 视频模式 | 默认 30 秒间隔，启用智能去重 | 自动生成结构化学习笔记 | 去重阈值需调优，过严会漏帧，过松会冗余 |
| 桌面壳 | Electron + TEN Framework | 覆盖全桌面应用（VS Code、Adobe、Excel、本地 PDF 等） | 非浏览器插件，需本地安装，权限要求更高 |

- 起始姿势：双脚与肩同宽
- 下蹲要点：膝盖不超过脚尖
- 呼吸节奏：下蹲吸气，起立呼气
- 常见错误：腰部过度前倾

**响应速度实测**: Seed 2.0 Pro 在编程辅导和文档理解场景下平均响应时间 2 秒以内

### 📝 避坑指南

- ⚠️ 截图前必须先隐藏助手窗口，否则 AI 会分析到自身 UI，干扰目标内容识别
- ⚠️ 图像传入 Seed 2.0 API 前必须压缩分辨率，未压缩的高清截图会显著增加延迟
- ⚠️ MiniMax TTS 必须使用流式接口，普通 HTTP 请求需等全段生成完才能播放，体验极差
- ⚠️ 音频处理必须用 AudioWorklet，主线程处理在 AI 返回大段内容时会导致音频卡顿
- ⚠️ 视频学习模式的去重逻辑需要调优，阈值过严漏关键帧，过松笔记冗余且浪费 token
- ⚠️ 2.0 尚未开源（截至 2026-02-15），多模态完整代码需等春节后 GitHub 更新
