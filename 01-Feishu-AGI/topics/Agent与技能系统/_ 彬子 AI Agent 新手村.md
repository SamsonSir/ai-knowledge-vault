### 📋 核心分析

**战略价值**: 构建三层防御机制解决 LLM Agent 的记忆完整性问题，将日志完整率从 ~60% 提升至 ~95%，实现从"金鱼记忆"到可检索知识库的演进。

**核心逻辑**:
- **问题本质**: 依赖模型主动记录不可靠，多客户端/多 Session 场景下遗漏严重
- **解决路径**: 主动记录（Memory Flush）→ 被动兜底（心跳扫描）→ 深度加工（夜间反思）
- **关键洞察**: 记忆不是靠自觉，是靠机制；心跳不是打扰，是安全感；反思不是复盘，是进化

### 🎯 关键洞察

**Memory Flush 的边界与局限**
Memory Flush 在 Session 接近 Compact 阈值前触发写日志，但触发时机完全绑定上下文消耗速度。高频对话场景记录勤快，低频场景可能隔数天才记一笔。这是机制设计的"频率盲区"，必须叠加周期性兜底方案。

**心跳扫描的消息清洗工程化**
60 分钟周期的心跳任务核心挑战是信号噪声分离。工程实现包含七层过滤：角色过滤（仅 user/assistant）、噪音过滤（系统元数据/低信号回复）、Assistant 消息分块打分（头尾块+高分中间块，关键词加权）、Compaction Summary 结构化清洗（白名单章节+行数限制）、状态推导（needs_reply/done/blocked/in_progress）、去重、特殊格式处理（语音转录合并）。最终按权重取 Top 30，实现有限 Token 预算内的信息密度最大化。

**夜间反思的双阶段设计**
第一阶段（00:45）用长上下文模型（Qwen3.5-plus, 1M）做 24 小时全量扫描，生成结构化备忘录，核心是对比原始日志与扫描素材的四种处理逻辑（跳过/合并/新增/丢弃）。第二阶段（01:00）换用强反思模型（Codex-GPT-5.4）做深度加工，输出聚焦判断演进、事故链路、根因定位、有效动作识别，并执行知识回写（lessons/decisions/projects 三类文件）与 freshness 标记。

**Skill 化与 Subagent 架构**
多步骤任务通过 SKILL.md 固化，主 Session 分配 Subagent 静默执行，完成后 announce 回主 Session 生成用户可见摘要。此设计解耦了执行稳定性与用户打扰，是生产级 Agent 系统的关键模式。

### 📦 可执行模块

| 模块 | 内容描述 | 适用场景 |
|------|---------|---------|
| Memory Flush 调优 | 关闭或自定义 prompt，减少与心跳机制的冗余 | Token 敏感、心跳记录已稳定运行的系统 |
| 心跳扫描脚本 | `general-session-scanner.sh 60` + 七层消息清洗 | 多客户端/多 Session 的完整性兜底 |
| 夜间反思流水线 | `lcm-daily-sync.py` → 素材备忘录 → 深度反思 → 知识回写 | 长周期任务的全貌校准与经验固化 |
| 知识文件结构 | lessons/ decisions/ projects/ 三类目录 + freshness 标记 | 可检索、可验证的知识沉淀 |
