### 📋 核心分析

**战略价值**: Fellou 是首批 AI Agentic 原生浏览器之一，核心差异在于浏览器本身即 Agent 执行环境，可直接操作已登录页面，比「AI Agent + 无头浏览器」方案体验更优，但当前版本执行稳定性存在明显缺陷。

**核心逻辑**:

- **下载入口**: `https://fellou.ai/download`，提供 macOS 适配版本选择
- **启动即任务**: 登录后右侧自动弹出任务执行与对话窗口，默认触发一个「发送 Twitter」的新手引导任务，属于品牌运营设计（自动发推告知用户正在使用 Fellou）
- **三大核心功能**: 总结网页 / 深度研究 / 执行工作流，三者均在浏览器内原生运行
- **总结网页实测**: 对 `https://arxiv.org/html/2504.10148v1` 论文执行「通俗中文总结」，功能可用，但与现有浏览器插件（如 Sider、Monica）无明显差异化优势
- **深度研究实测**: 登录小红书后，指令「查找杭州美食攻略」，Fellou 自动浏览帖子并生成结构化报告（含八大美食聚集地、老字号分级、菜系分类、避雷指南），内容质量较高
- **爬取任务失败案例**: 指令「爬取帖子基本信息」，执行过程中鼠标自动滚动时元素识别顺序出现反复跳转，最终任务提示成功但输出 NULL，疑似 token 超限导致
- **强介入模式（当前唯一模式）**: 任务执行期间，切换到其他 tab 会触发 Agent 自动跳回原页面，导致无法同时使用浏览器做其他操作——适合「扔任务等结果」场景，不适合作为主力日常浏览器
- **核心竞争力**: 浏览器本身可访问需要登录的页面（小红书、Twitter 等），相比 AI Agent + 虚拟机无头浏览器方案，登录态保持更自然，交互体验更接近真实用户操作
- **任务规划 vs 执行落差**: 规划能力强（能生成完整的多步骤操作计划），但页面元素识别稳定性差，复杂滚动场景下容易失序

### 🎯 关键洞察

**为什么 Agentic 浏览器比无头浏览器方案更有潜力**:

- 原因：无头浏览器（Headless Chrome）在 AI Agent 场景下需要额外处理登录态、验证码、动态渲染等问题，且与用户真实会话隔离
- 动作：Fellou 直接复用用户已登录的浏览器会话，Agent 操作的就是用户真实看到的页面
- 结果：登录墙问题天然解决，小红书、Twitter、内部系统等需要身份验证的页面可直接操作，无需额外配置 Cookie 注入或账号共享

**强介入模式的本质限制**:

- 当前 Fellou 的 Agent 执行是「独占式」的，任务运行期间浏览器控制权归 Agent，用户无法并行操作
- 这意味着它更像一个「自动化执行终端」而非「增强型日常浏览器」
- 合理使用姿势：开启任务 → 切换到其他设备或应用做别的事 → 回来取结果

### 🛠️ 操作流程

1. **准备阶段**:
   - 前往 `https://fellou.ai/download` 下载对应 macOS 版本
   - 安装并登录账号
   - 提前在 Fellou 内登录需要操作的目标平台（小红书、Twitter 等），确保登录态有效

2. **核心执行**:
   - 登录后右侧面板自动弹出，完成新手引导任务（自动发 Twitter）
   - 在右侧对话窗口输入自然语言指令，例如：「登录小红书，搜索杭州美食攻略，汇总成结构化报告」
   - 任务启动后不要切换 tab（强介入模式），让 Agent 独占执行
   - 若需要并行工作，切换到其他设备，等待任务完成后回来取结果

3. **验证与优化**:
   - 检查输出是否为 NULL（NULL = 大概率 token 超限或页面识别失败）
   - 遇到 NULL 输出时，缩小任务范围重试（如从「爬取所有帖子」改为「爬取前5条帖子」）
   - 长页面滚动任务当前稳定性差，避免依赖此类场景做关键业务

### 📝 避坑指南

- ⚠️ **不要把 Fellou 当主力日常浏览器**：强介入模式下任务执行会劫持浏览器控制权，切换 tab 会被强制拉回，严重影响日常使用体验
- ⚠️ **复制链接功能当前无效**：Tab 上的「copy link」按钮存在 UI 但功能未实现，需要其他方式获取当前页面 URL
- ⚠️ **点「+」不会开空白页**：会强制弹出 AI 指令输入框，无法绕过，如果只是想开新页面会很烦
- ⚠️ **长页面爬取任务大概率失败**：滚动过程中元素识别顺序会反复跳转，最终输出 NULL，疑似 token 上限问题，当前版本不适合做大规模数据爬取
- ⚠️ **任务规划强但执行弱**：不要因为规划步骤看起来完整就认为结果可靠，务必验证最终输出内容
