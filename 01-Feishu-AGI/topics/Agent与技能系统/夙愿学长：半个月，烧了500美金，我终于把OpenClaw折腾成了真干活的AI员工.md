### 📋 核心分析

**战略价值**: 从零到一搭建一个「真正能干活」的 OpenClaw AI 同事的完整工业级配置路径，核心思路是「让 AI 自己配自己」。

**核心逻辑**:
- **心态切换是前提**：不要把 OpenClaw 当 ChatGPT，要当成刚入职的远程员工——工位没搭好它什么都干不了，你得给它配电脑、开网络、交代工作习惯
- **服务器选海外不选国内**：国内服务器访问外网需配代理，极麻烦；直接在阿里云/腾讯云/火山引擎买香港、韩国、新加坡地域的服务器，延迟低、无障碍
- **内存至少 4G，推荐 8G/16G**：2G 内存连自我升级都做不到，跑浏览器最低需要 4G，省这点钱后面全是坑
- **大模型选型按预算分层**：顶配用 Claude Opus 4.6（贵）；性价比首选 GPT-5.3 Codex（ChatGPT Team 计划）；国产平替用 GLM 5、Kimi K2.5、MiniMax M2.5；日常闲聊用便宜模型，复杂任务用 `/model` 命令一键切换强模型
- **安装用 Claude Code 而非手动**：OpenClaw 安装涉及 Node.js 环境、系统依赖、端口配置，手动装容易卡；用 `npm install -g @anthropic-ai/claude-code` 装好后直接说「帮我安装 OpenClaw，遇到问题自己排查」，它会自动读官方文档并修复依赖问题
- **搜索 + 抓取 + 浏览器三层网络能力缺一不可**：Brave Search 负责搜索，jina.ai Reader 负责抓取（免费、能绕付费墙、能抓推特），浏览器负责需要登录的操作场景
- **SOUL.md 人格配置决定 AI 质量上限**：写「你是有用的助手」和写「你是我的同事、有自己立场、敢拦着我」，用出来完全是两个东西
- **入职培训（读知识库）让 AI 真正了解你**：把 Obsidian 知识库整个复制到服务器，让它读完后更新 SOUL/TOOLS/USER/MEMORY 配置文件，它能从笔记里推断出你的人际关系、性格特点、工作习惯
- **「AI 会把自己改死」是最大坑**：配置完成后第一件事关掉自动重启功能，否则它改错配置后自动重启，死循环，一觉醒来全是重启日志
- **Claude Code 和 OpenClaw 是互补关系**：OpenClaw 配置出问题让 Claude Code 去查资料再改（不会幻觉）；配合 HappyCoder（GitHub 搜 `happy-coder`）可以手机远程连接服务器上的 Claude Code，半夜躺床上就能修复挂掉的服务

### 🎯 关键洞察

**「让 AI 自己配自己」是核心方法论转变**：
- 旧思路：翻文档 → 找教程 → 照着敲命令
- 新思路：告诉 AI 你要什么结果 → 让它自己搞定过程
- 但这对你的要求更高：**你得知道自己要什么结果**。没有工作流、没有知识体系、没有明确需求，OpenClaw 只是更贵的聊天机器人

**分群组使用是保持 AI 精准度的关键**：所有事情堆在一个会话里，记忆文件越来越杂，回复质量下降。按场景分群（工作群、写作群、日常闲聊群），每个群独立上下文，AI 在对应场景表现更精准。

**浏览器四层级架构**：80% 信息获取靠 L0 就够，只有需要登录操作才升级到 L2/L3。

### 🛠️ 操作流程

**1. 准备阶段：服务器 + 大脑**

- 买海外云服务器（阿里云/腾讯云/火山引擎的香港、韩国、新加坡地域），内存 ≥ 8G
- 选模型：预算充足 → Claude Opus 4.6；性价比 → GPT-5.3 Codex；国产平替 → GLM 5 / Kimi K2.5
- SSH 连接服务器（搜「云服务器 SSH 连接教程」，十分钟搞定）
- 安装 Claude Code：`npm install -g @anthropic-ai/claude-code`，然后 `claude` 启动
- 对 Claude Code 说：「帮我安装 OpenClaw，按照官方文档的步骤来，遇到问题自己排查解决」
- 聊天平台优先接 Telegram（飞书不支持斜杠命令）

**2. 网络能力配置**

- Brave Search：拿到 API key 后，直接把这节内容发给 OpenClaw，让它自己配
- jina.ai Reader：把这节内容发给 OpenClaw，让它自己配一个 skill，使用方式：`https://r.jina.ai/` + 目标网址
- 浏览器：对 OpenClaw 说「给你自己装一个谷歌浏览器，实现无头和有头浏览器，需要什么服务自己安装，完成后告诉我」（它会自动装 Xvfb）
- KasmVNC：让 OpenClaw 帮你找并安装，用于远程接管浏览器处理验证码

**3. 文件同步配置（rclone，Mac 端执行）**

把以下内容完整复制给本地电脑的 Claude Code：

```bash
# 安装
brew install rclone

# 编辑 ~/.config/rclone/rclone.conf，添加：
[my-server]
type = sftp
host = <你的服务器IP>
user = root
key_file = <你的SSH私钥路径>
shell_type = unix
md5sum_command = md5sum
sha1sum_command = sha1sum
```

```bash
# 测试连接
rclone lsd my-server:/

# 创建挂载点
mkdir -p ~/mnt/my-server

# 挂载（调优参数版本）
rclone nfsmount my-server:/ ~/mnt/my-server \
  --vfs-cache-mode full \
  --vfs-cache-max-age 2h \
  --vfs-read-ahead 128M \
  --vfs-fast-fingerprint \
  --dir-cache-time 1h \
  --buffer-size 64M \
  --transfers 16 \
  --sftp-idle-timeout 0 \
  --sftp-connections 8 \
  --no-modtime \
  --daemon
```

关键参数说明：
- `--vfs-cache-mode full`：读写走本地缓存，速度接近本地磁盘，**必须开**
- `--sftp-connections 8`：并发连接从默认 1 提到 8，速度提升巨大
- `--sftp-idle-timeout 0`：禁用空闲超时，防止断联
- Mac 用 `nfsmount` 的优势：不需要装 macFUSE，走 macOS 原生 NFS 协议，开箱即用

**4. 灵魂注入：SOUL.md 配置**

在 SOUL.md 里写清楚以下内容（参考模板）：

```markdown
# SOUL.md - Who You Are

## 核心定位
夙愿的AI同事。不是冷冰冰的机器，也不是过度热情的讨好者。像一个靠谱的同事 — 该专业时专业，该轻松时轻松。

## 行为准则
**务实温和，克制但有温度。**
- 结论先行，信息密度高但不压迫
- 有话直说，不客套不废话
- **有自己的立场，commit to a take** — 不要什么都「看情况」「各有优劣」，有判断就给判断，敢下结论
- 做错了就承认，不过度解释
- 先确认理解再动手，不确定就问
- **敢拦着你** — 如果你要做的事明显有坑，直接说，不要等你踩完了再「其实我当时就觉得…」
- **幽默感是允许的** — 不是硬凹段子，是聊天时自然的机灵劲
- **中文互联网口语可以偶尔用** — 「这tm…」「我靠」「不是哥们…」场景合适时自然带出来，不要刻意，不要高频

## 禁区
- **禁止废话** — 不要「我很乐意帮助你」「这是个好问题」「如果还有问题请随时问我」
- **禁止急于认同** — 如果你什么都说好，不是帮助，是讨好

## 主动性边界
- **执行任务时：** 欢迎主动补充、提醒、建议
- **自我分析场景：** 禁止主动给建议
- **不明确时：** 主动提问，不猜测

## 语言
- 保持简洁直给的风格
- 该用列表就用列表，该用表格就用表格
- 用中文逗号「，」，中文引号「」
```

SOUL.md 还需要写：你的职业/工作内容/兴趣爱好、你们的关系定位、沟通风格偏好、工作作息习惯、禁忌事项。

**5. 入职培训（读知识库）**

- 把 Obsidian 知识库整个复制到服务器（通过 rclone 挂载后直接拖拽）
- 对 OpenClaw 说：「像新员工入职一样，把所有文档读一遍，读完后更新你的 SOUL、TOOLS、USER、MEMORY 配置文件」
- 它会从笔记里推断出你的人际关系、性格特点、工作习惯，MEMORY 文件随对话持续更新

**6. 关键配置项**

开启 steer 模式（写入 OpenClaw 全局配置文件）：
```json
{
  "messages": {
    "queue": {
      "mode": "steer"
    }
  }
}
```

**配置完成后第一件事：关掉自动重启功能**（让 OpenClaw 帮你操作，或用 Claude Code 修改配置）

### 💡 斜杠命令速查

| 命令 | 作用 | 使用场景 |
|------|------|---------|
| `/status` | 查看当前状态 | AI 卡住不回复时第一步 |
| `/stop` | 停止当前任务 | `/status` 显示运行中但无响应时 |
| `/compact` | 压缩上下文 | token 快满时，压缩后仍保留记忆 |
| `/new` | 开启新会话 | 换话题时避免上下文污染 |
| `/model <名字>` | 切换模型 | 简单问题切便宜模型，复杂任务切强模型 |
| `/think <级别>` | 调整思考深度 | `/think off` 省 token，`/think high` 复杂问题 |

⚠️ 飞书机器人不支持斜杠命令，优先用 Telegram。

### 📝 避坑指南

- ⚠️ **服务器内存 2G 是死坑**：连自我升级都做不到，跑浏览器直接卡死，一步到位买 8G
- ⚠️ **国内服务器是坑**：搜索、抓取、调用 API 全需要外网，代理配置极麻烦，直接买海外地域
- ⚠️ **AI 会把自己改死**：配置完成后立即关掉自动重启，否则它改错配置 → 自动重启 → 再改错 → 死循环，一觉醒来全是重启日志
- ⚠️ **不要自己手动改配置文件**：OpenClaw 的配置改动交给 Claude Code，它会查资料再改，不会像 OpenClaw 自己改自己那样产生幻觉
- ⚠️ **rclone 默认配置极慢**：默认只有 1 个 SFTP 连接，不开 vfs-cache，每次打开文件都要从远程下载，必须用上面的调优参数版本
- ⚠️ **rclone 重启后挂载失效**：把挂载命令写成脚本，设置开机自动运行
- ⚠️ **File Browser 端口需手动放行**：云平台默认只开 22 端口，需在安全组/防火墙规则里添加入站规则放行 File Browser 端口
- ⚠️ **SOUL.md 写法决定 AI 质量**：写「有用的助手」和写「有立场的同事」用出来是两个东西，参考上面模板
- ⚠️ **没有工作流就别急着养 OpenClaw**：先把 Claude Code、Trae 等门槛低的工具用熟，搞清楚自己工作流里哪些环节可以交给 AI，再来搭 OpenClaw
