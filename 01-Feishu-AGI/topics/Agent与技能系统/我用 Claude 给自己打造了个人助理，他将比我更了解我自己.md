### 📋 核心分析

**战略价值**: 基于 SDD（规格驱动开发）思路，用 Claude Code CLI + OpenSpec 构建一个拥有长短期记忆、持续学习用户画像的个人 AI 助理系统。

**核心逻辑**:

- **根本问题定位**：所有 AI 对话工具（DeepSeek、Gemini 等）每次新开窗口都会丢失上下文，用户被迫反复自我介绍，本质是 AI 缺乏长短期记忆机制。
- **解法来源于 AI Coding 领域**：SDD（Specification Driven Development）已在 AI Coding 中解决了"AI 不了解项目背景"的问题，代表工具有 Spec kit、OpenSpec，直接将该思路迁移到个人助理场景。
- **工具选型为 Claude Code CLI**：它本身是一个 Agent，调用底层模型时已内置系统提示词（诚信、透明、尊重用户、实用性等），可在此基础上叠加个人定制，而非从零搭建。
- **CLAUDE.md 是系统提示词入口**：在项目根目录新建 `CLAUDE.md`，其内容会追加（而非替换）到 Claude Code CLI 原有系统提示词，既保留原有良好行为，又注入个人化指令。
- **CLAUDE.md 写入内容分两层**：第一层是角色定义（"你是我的个人助理，用自然温暖方式交流"）+ 用户基础信息（年龄、性别、职业、工作内容、薪资、家庭构成、受教育程度）；第二层是文件索引路径，指向所有存量知识库目录。
- **价值观不手动定义，让 AI 自主提取**：作者刻意不在 CLAUDE.md 中写价值观，而是让 AI 通过后续每次对话内容自主归纳，目标是让 AI 比用户自己更了解自己。
- **存量索引构建**：项目创建时一次性导入 5 类历史内容：投资思考、工作日志、个人笔记、和 AI 的聊天记录（来自 DeepSeek/Gemini 历史导出）、公众号文章，分别存入对应子目录，构成 AI 对用户的初始理解基础。
- **增量索引依赖改造后的 OpenSpec 工作流**：原 OpenSpec 流程（发起提案→深度对话→规划任务→执行任务→归档提案）天然适配，但原提案入口是面向开发需求的，需魔改提示词使其适配个人助理场景；改造方式是直接让 AI 自己修改提示词，改造后发起提案时 AI 会给出 4 个常见提案类型选项 + 自定义类型。
- **日常使用策略**：不强制每次都走提案流程，可直接聊天；当某次对话内容有价值时，再触发归档流程，将对话存入知识库，实现增量积累。
- **MCP 和 Skills 扩展能力边界**：当前配置的 MCP 为视觉理解、联网搜索、网页读取、GitHub 仓库；Skills 包含 doc-coauthoring、docs、internal-comms、markdown-preview、pdf、pptx、skill-creator、xlsx、react-best-practices、vercel-deploy-claimable、web-design-guidelines，后续随承担工作增加继续扩展。

### 🎯 关键洞察

**为什么 SDD 能解决 AI 记忆问题**：SDD 的核心是把"上下文"外化为结构化文件，而不是依赖模型的 context window。模型每次启动时读取这些文件，等效于"恢复记忆"。这个机制在 AI Coding 中已被验证（AI 读取项目 spec 后能准确理解业务背景），迁移到个人助理场景的逻辑完全一致——只是把"项目代码和需求文档"换成了"个人信息和生活记录"。

**为什么选 Claude Code CLI 而非直接用 Claude.ai**：Claude Code CLI 已内置 Agent 框架（工具调用、文件读写、MCP 接入），不需要自己搭架子；同时 CLAUDE.md 机制提供了干净的系统提示词注入点，开箱即用。

**绝对诚实是前提条件**：系统提示词中写入的个人信息（包括薪资、家庭构成等敏感信息）必须真实，因为目标是构建数字分身，虚假信息会导致 AI 建立错误的用户模型，后续所有推断都会偏差。

### 🛠️ 操作流程

1. **准备阶段**:
   - 在 GitHub 创建 private 仓库，命名如 `smart-me`
   - 在本地 clone 仓库，创建以下目录结构：
     ```
     smart-me/
     └── openspec/
         └── docs/
             ├── 和 AI 对话/
             ├── 投资思考/
             ├── 工作日志/
             ├── 个人笔记/
             └── 公众号文章/
     ```
   - 将历史笔记、投资记录、工作日志全部拷贝到对应目录
   - 导出 DeepSeek、Gemini 等工具的历史聊天记录，存入「和 AI 对话」目录

2. **核心执行**:
   - 在项目根目录创建 `CLAUDE.md`，写入：
     - 角色定义（温暖自然的个人助理）
     - 个人基础信息（年龄/性别/职业/工作/薪资/家庭/学历）
     - 各目录的文件索引路径（如上表所示）
   - 引入 OpenSpec，让 AI 自行改造提案入口提示词，使其适配个人助理场景（非开发场景）
   - 配置 MCP：视觉理解、联网搜索、网页读取、GitHub 仓库
   - 配置 Skills：按上表列表添加

3. **验证与优化**:
   - 启动 Claude Code CLI，进入 `smart-me` 项目目录
   - 发出第一个问题：「请基于当前项目下的内容，思考一下我是什么风格、什么价值观的人」
   - 验证 AI 是否能准确归纳出用户画像
   - 日常使用：直接聊天，有价值的对话触发归档流程存入知识库，实现增量积累
   - 不需要每次都走提案流程，简单咨询直接聊即可

### 📝 避坑指南

- ⚠️ CLAUDE.md 是追加而非替换系统提示词，不要试图在里面覆盖 Claude Code CLI 的核心行为约束（诚信、透明等），这些是有益的
- ⚠️ 个人信息必须真实完整，包括薪资、家庭构成等敏感字段，虚假信息会导致 AI 用户模型偏差，后续所有推断失真
- ⚠️ 历史聊天记录（DeepSeek/Gemini）要在项目创建初期一次性导入，越早导入 AI 对你的理解越深，拖延会导致前期对话价值损失
- ⚠️ OpenSpec 提示词改造不要手动改，直接让 AI 自己改，描述清楚目标场景即可，人工介入容易破坏提示词结构
- ⚠️ 价值观不要手动写入 CLAUDE.md，让 AI 从对话和文档中自主提取，手动定义会引入主观偏差和自我美化
