### 📋 核心分析

**战略价值**: 用 Skills 封装「文章配图」全流程，将「读文→规划→文案→提示词→批量出图」五步自动化，同时在每步插入人工确认节点，兼顾效率与可控性。

**核心逻辑**:

- **痛点定位**：手动找图库耗时，AI 直出图需每次手写提示词 + 人工点击，流程碎片化，只适合封面场景，正文配图几乎无法规模化。
- **Skills 的定位**：介于「单次 Prompt」和「完整产品」之间的中间层，适合高频但开发成本不值得做成产品的需求（配图、漫画、思考辅助等）。
- **五步流程设计原则**：不一把梭，而是每步确认，原因是 API 出图有成本，全量重来代价高；分步纠偏可把错误消灭在早期环节。
- **人比 AI 更懂自己的文章**：每个位置该放什么类型的图、起什么效果，AI 无法完全猜到，必须让人参与每步决策。
- **Skills 可持续迭代**：每次使用后可追加知识和语料，Skills 会越来越懂使用者，模型越强，沉淀的 Skills 价值越大。
- **兼容两个主流工具**：配图助手 Skills 在 Claude Code 和 Codex 均可运行，安装路径不同（`.claude/skills/` vs `.codex/skills/`）。
- **出图 API 可替换**：默认使用 `https://apimart.ai/`，每张图成本约 0.4 元；如需换模型，直接告诉 AI 修改 Skills 内的 API 配置即可。
- **API 非强依赖**：不配置 API 时，前四步（需求澄清→规划→文案→提示词）仍可正常运行，只有第五步「批量出图」需改为手动搬运提示词。
- **Skills 的核心价值观**：慢慢变成「真正懂你的伙伴」，通过持续迭代积累个人化知识，而非一次性工具。

### 🛠️ 操作流程

**1. 下载 Skills**

从 GitHub 下载配图助手 Skills：

```
https://github.com/yunshu0909/yunshu_skillshub
```

让 AI 自动下载的指令：
```
请你帮我下载 https://github.com/yunshu0909/yunshu_skillshub 的内容到这个文件夹里
```

**2. 配置出图 API**

下载后先不要放入 Skills 目录，先配置 API Key。

- 默认出图 API：`https://apimart.ai/`
- 每张图成本约 0.4 元
- 如需替换其他出图模型，直接告诉 AI 对应的 API 信息，让 AI 修改 Skills 内的配置

**3. 安装到 Skills 库**

AI 安装方式：
```
请把这个 skills 安装到你的 skills 库里 @对应的skills地址
```

手动安装方式（复制文件到对应目录）：

| 工具 | Skills 目录路径 |
|------|----------------|
| Claude Code | `.claude/skills/` |
| Codex | `.codex/skills/` |

**4. 使用配图助手（五步流程）**

| 步骤 | 名称 | 触发方式 / 操作 | 人工确认点 |
|------|------|----------------|-----------|
| 第一步 | 需求澄清 | 输入 `/image-assistant` 或说「我需要配图助手」，提供文章内容和预期风格 | 确认 AI 对文章的理解是否准确 |
| 第二步 | 配图规划 | AI 自动输出每张图的设计思路、文字预算、模板建议 | 确认规划方向后进入文案阶段 |
| 第三步 | 文案定稿 | AI 逐张确定图上文字内容和布局方式 | 逐张确认文案和布局无误 |
| 第四步 | 提示词封装 | AI 生成批量调用 API 所需的提示词指令 | 确认提示词后进入出图阶段 |
| 第五步 | 批量出图 | AI 批量调用 API 生成图片，保存到指定文件夹 | 无需干预（或手动搬运，若未配置 API） |

---

### 📝 避坑指南

- ⚠️ **先配置 API 再安装**：下载后不要直接放入 Skills 目录，必须先填好 API Key，否则第五步出图环节无法自动完成。
- ⚠️ **不配置 API 不等于不能用**：前四步完全可用，只是第五步需要手动把提示词搬到出图工具里操作。
- ⚠️ **不要一把梭让 AI 直接出所有图**：跳过确认步骤直接批量出图，一旦方向偏差就要全部重来，API 调用成本会快速累积。
- ⚠️ **Skills 需要持续迭代**：初次使用效果不一定完美，使用后要主动给 Skills 补充知识和语料，才能越用越顺手。
