### 📋 核心分析

**战略价值**: 将 GitHub 检索能力封装成一个可复用的 Skills，让 Codex / Claude Code 作为研究员代理执行「需求对齐 → 多轮检索 → 硬过滤 → 文档阅读 → 相关性重排」全流程，非技术用户也能零门槛调研 GitHub 项目。

**核心逻辑**:

- **痛点定位**：英文搜索能力弱 + 不懂如何评判项目质量，是非技术用户进入 GitHub 的两道墙；Skills 方案绕过这两道墙，让 Agent 代劳。
- **核心机制是 Agent 作业流，不是简单搜索**：整个流程有循环、有重试、会自我纠正，跑完一次约需 10-15 分钟，等同于一个真人研究员的调研产出。
- **需求对齐阶段**：Agent 收到需求后不会直接搜索，而是先与用户确认真实意图，避免检索方向跑偏。
- **多轮检索 + 自动优化检索词**：Agent 自动拆解出若干检索词，执行检索后评估结果质量，若质量不达标则自动优化检索词并重新检索，直到满意为止。
- **硬过滤条件筛选**：在原始检索结果中先用硬条件（如 Star 数、活跃度、语言等）过滤掉不符合的项目，缩小候选集。
- **文档阅读层**：对过滤后的候选项目，Agent 会实际阅读项目 README / 文档，而非仅看元数据，确保理解项目实际功能。
- **相关性重排，而非 Star 排序**：最终排序逻辑以「需求关联度」为第一优先级，Star 数只是参考。案例中 OpenClaw 官方主仓 Star 最高，但因用户需求是「玩法汇总」，玩法合集仓库被排在第一位。
- **自动分类输出**：工具类搜索结果会自动区分「可直接用的桌面软件」与「需要自己安装的框架层」，降低用户选择成本。
- **端到端安装能力**：找到目标项目后，可直接让 Codex 执行安装，部分项目（如 Open Video Downloader）安装后直接打开桌面版，开箱即用，无需手动折腾。
- **用户侧极简操作**：用户只需做两件事——讲出需求、看报告，中间所有检索、过滤、阅读、排序全由 Agent 自动完成。

### 🎯 关键洞察

**为什么排序不按 Star 数而按需求关联度？**

Star 数衡量的是项目的「综合受欢迎程度」，而用户的需求往往是垂直的。以 OpenClaw 案例为例：官方主仓 Star 最高，但它是工具本体，不包含玩法内容；用户真正需要的是「玩法清单合集」仓库。如果按 Star 排序，用户会先看到官方仓，然后自己再去找玩法，多走一步。Agent 通过阅读文档理解项目内容后重排，直接把最匹配需求的结果放第一位，这是纯关键词搜索做不到的。

**为什么要有「需求对齐」这一步？**

用户表达的需求往往是模糊的，比如「找 Agent 记忆模块」可能指：现成可用的 SDK、论文实现、框架插件、或完整 Agent 框架。如果跳过对齐直接搜索，检索词可能完全跑偏，浪费后续所有步骤的时间。需求对齐把模糊意图转化为精确检索方向，是整个流程质量的源头保障。

### 🛠️ 操作流程

1. **安装 Skills**
   - 方式 A（手动）：访问 `https://github.com/yunshu0909/yunshu_skillshub`，下载仓库，将 `github-repo-search` 放入 Codex / Claude Code 的全局 Skills 文件夹。
   - 方式 B（让 Agent 自动装）：打开 Codex 或 Claude Code，直接输入：
     > 安装这个 GitHub 库的 Skills 到 Codex 和 Claude Code 的全局 Skills 文件里：`https://github.com/yunshu0909/yunshu_skillshub`

2. **核心执行（提出需求）**
   - 向 Codex 描述你的检索需求，建议包含：目标用途、期望项目类型（工具/框架/学习资料）、是否需要开箱即用。
   - 示例需求描述风格参考：
     - 「帮我找 OpenClaw 的玩法汇总仓库」
     - 「找能下载 B 站和 YouTube 视频的工具，最好有桌面版」
     - 「找 Agent 记忆模块，要成熟可用的，最好能开箱即用」
   - Agent 会先与你做需求对齐，确认意图后自动开始检索，全程约 10-15 分钟。

3. **验证与后续操作**
   - 收到报告后，检查排序第一的项目是否真正匹配你的需求（而非只看 Star 数）。
   - 如果找到目标项目，可直接让 Codex 执行安装：「帮我把 [项目名] 安装到本地」。
   - 桌面软件类项目安装后可直接打开使用；框架类项目需要进一步配置。

### 📝 避坑指南

- ⚠️ Skills 必须放入「全局 Skills 文件夹」，放错路径 Agent 无法识别，建议用方式 B（让 Agent 自动安装）避免路径问题。
- ⚠️ 需求描述越具体越好，模糊需求会导致需求对齐阶段来回确认，拖慢整体流程；建议一次性说清「用途 + 项目类型 + 是否需要开箱即用」。
- ⚠️ 整个检索流程约需 10-15 分钟，不要中途打断，否则多轮检索和重排逻辑会被截断，结果质量下降。
- ⚠️ 框架类项目（非桌面软件）安装后通常还需要额外配置，不能期望所有项目都像 Open Video Downloader 一样开箱即用。
- ⚠️ Claude Code 可用但效果略逊于 Codex 桌面版，优先用 Codex 桌面版跑这个 Skills。
