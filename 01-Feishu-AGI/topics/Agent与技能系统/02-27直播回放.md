### 📋 核心分析

**战略价值**: 通过为 OpenClaw（龙虾）打造像素风可视化办公室，解决非技术用户无法感知 Agent 运行状态的体验痛点，并将其开源为可复用的 UI 框架。

**核心逻辑**:

- **龙虾（OpenClaw）是 2026 年最火 Agent**，其核心差异在于"可进化"设计：能记住用户历史对话，当新条件触发时自动重跑历史记录，行为更像有羁绊的宠物而非工具
- **安装成本**：Mac mini（约 6000 元）+ 两天安装时间；依赖稳定网络，Gateway 频繁挂掉是常见问题
- **模型选择优先级**：Claude Code（首选）→ Kimi / Minimax / 智谱 GLM 5（国内备选）；Minimax 价格最低，M2.5 模型支持本地运行，适合入门；GLM 4.7 是 leo 首次跑通的模型
- **严禁让龙虾自己换模型**：自主换模型大概率导致龙虾挂掉，必须用户手动操作
- **扩展能力靠插件**：安装 Browser Wind 插件 → 获得上网/爬取能力；安装 Stable Diffusion → 获得本地画图能力；接入 Gemini 模型 → 支持语音输入；List Tab 的播客 API → 自动生成播客并定时推送
- **可视化办公室项目起因**：命令行对非编程用户不友好；飞书/Telegram 聊天时龙虾不回消息，用户无法判断其是否在工作，体验黑盒
- **项目实现路径**：李浩文通过飞书与云端 OpenClaw 对话 → 提需求（像素风状态面板）→ 龙虾自主完成编码、绘制界面、部署公网、绑定域名，全程无需人工写代码
- **项目传播效果**：Twitter 阅读量超 10 万，GitHub 获 120+ Stars，项目名为 `star-office-ui`，已开源
- **精灵表（Sprite Sheet）是动画核心技术**：将 GIF 转成精灵表上传给龙虾，龙虾可通过坐标定位切换家具/角色状态，实现任意动画状态切换
- **飞书 API 限制问题**：国内飞书使用龙虾存在 API 调用限制，已开通无限 API 申请通道，24 小时内可开通
- **日运营成本**：API 消耗约 5 美元/天；养一只龙虾约 30 元人民币/天；可选字节火山 Coding Plan 包月 API 降低成本

### 🎯 关键洞察

**为什么可视化状态面板有价值**：
龙虾是异步运行的 Agent，用户发出指令后可能几分钟甚至更长时间没有响应。在命令行或 IM 工具里，这段沉默会让用户产生"它挂了吗？""它在干嘛？"的焦虑，导致频繁打断询问，反而干扰任务执行。可视化办公室将龙虾的内部状态（休息/工作/同步文件/报 Bug/报警）映射为像素动画场景，用户一眼就能判断状态，无需打断。

**为什么龙虾能自己完成整个项目**：
龙虾底层使用 Claude Code（或 GPT 5.3 CODEX），上下文支持能力强，能快速定位并修复问题。李浩文只负责提需求和提供样板间风格参考，龙虾自主完成了所有编码和部署工作。这本身就是该项目最好的 demo——用龙虾来建龙虾的办公室。

**上下文过长的处理技巧**：
飞书调用 OpenClaw 时，随对话上下文增长，延迟会显著加重（替换一张图片可能等待半小时无响应）。解决方案：新开一个聊天窗口，让龙虾重新读取项目，绕过长上下文导致的性能下降。

### 🛠️ 操作流程

**一、龙虾可视化办公室搭建 SOP**

1. **需求沟通阶段**
   - 通过飞书机器人与云端 OpenClaw 对话
   - 明确需求：像素风格状态面板，将龙虾状态（休息/工作/报警等）映射为动画场景
   - 让龙虾列出功能清单并确认可实现范围

2. **素材准备阶段**
   - 用 Nano Banana 生成精装房图片，再将其"变回毛坯"，提取独立家具元素为 PNG
   - 将龙虾头像转换为 16×16 像素风格形象
   - 用 Nano Banana 延展角色的不同动作和表情包状态
   - 通过 VO 3.1 生成动作视频
   - 用 honeycam 将视频处理成透明背景小体积 GIF
   - 将 GIF 转换为精灵表（Sprite Sheet）格式

3. **核心执行阶段**
   - 将精灵表上传给龙虾
   - 龙虾通过显示坐标功能定位各素材位置
   - 龙虾自主完成：绘制办公室界面 → 实现状态切换逻辑 → 提供公网链接 → 部署域名
   - 通过飞书向龙虾发送不同风格样板间，让其更换背景

4. **验证与优化阶段**
   - 测试各状态切换是否正常（休息/工作/同步文件/报 Bug/报警）
   - 验证植物、海报、猫等可点击元素的状态切换
   - 若飞书对话上下文过长导致无响应，新开聊天窗口让龙虾重读项目
   - 后续迭代：接入 Nano API，让龙虾自主设计新元素，加快迭代速度

**二、龙虾多人授权机制配置**

1. 让龙虾设计授权码机制
2. 朋友向龙虾发送接入请求
3. 龙虾生成授权码发给朋友
4. 朋友将授权码转发给龙虾主人
5. 主人批准后，朋友才能正式与龙虾交流

### 📝 避坑指南

- ⚠️ **严禁让龙虾自主换模型**：会直接导致龙虾挂掉，必须用户手动切换
- ⚠️ **飞书上下文膨胀问题**：对话越长延迟越严重，替换一张图片可能等待 30 分钟无响应，解决方案是新开聊天窗口
- ⚠️ **龙虾记忆不可靠**：虽有记忆能力，但经常"忘事"，重要信息需主动提醒
- ⚠️ **龙虾进化常失败**：自身进化（自动重跑历史记录）成功率不稳定，不要完全依赖
- ⚠️ **多人共享记忆混乱**：多人同时使用同一龙虾会导致记忆混乱，必须配置授权码机制隔离
- ⚠️ **国内飞书 API 限制**：默认有调用次数限制，需单独申请无限 API，24 小时内开通
- ⚠️ **GIF 素材处理要求**：必须处理成透明背景，且体积要小，否则影响加载和显示效果
- ⚠️ **网络稳定性是前提**：龙虾高度依赖稳定网络，Gateway 不稳定是最常见的挂掉原因
