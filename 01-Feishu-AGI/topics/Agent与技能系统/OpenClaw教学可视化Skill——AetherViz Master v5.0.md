### 📋 核心分析

**战略价值**: AetherViz Master v5.0 是一个零代码、AI驱动的教学可视化工具，输入任意主题即可秒生成可交互的3D/2D教学网页，彻底绕开传统"需求→设计→开发"流程。

**核心逻辑**:

- **交互维度决定学习效率**：文字<图片视频<3D互动演示，维度越高越接近人类本能的"玩中学"模式，这是该工具的底层设计哲学。
- **Gemini 3.1是关键突破口**：此前用其他模型尝试同类任务均失败（生成内容残缺或不可用），Gemini 3.1的代码理解与生成能力首次达到"可真正干活"的水准，能把模糊需求转化为可执行代码。
- **自动学科识别（6大领域）**：AI解析输入关键词，自动匹配学科模式与视觉风格——"光合作用"→生物模式（叶绿体/细胞结构）；"行星运动"→天文模式（深空背景/轨道线）；"二分查找"→编程模式（数组可视化/算法动画）；覆盖物理、化学、生物、数学、天文、编程。
- **三种渲染模式无缝切换**：Three.js 3D（分子结构、力学分析）、SVG 2D（函数图像、统计数据）、混合模式，由AI自动判断最优呈现方式。
- **教学功能全家桶集成于单HTML文件**：公式面板（KaTeX实时渲染，支持数学/物理/化学公式）、测验面板（内置练习题+即时出结果）、回放控制（暂停/拖动/回放）、参数控制（滑块实时调参，自己"玩"实验）——这些功能集成在AI生成的单HTML文件中，目前独此一家。
- **极致性能指标**：60fps流畅渲染、玻璃拟态UI、OrbitControls支持3D场景任意旋转拖拽、单HTML文件直接浏览器打开无需配置环境。
- **真正零代码**：用户只需输入主题，生成的网页直接可用，不需改一行代码，适用人群覆盖老师（课件）、学生（自学）、培训机构（演示）。
- **解决知识呈现与知识复杂度的错配**：现代知识体系已是三维四维（量子隧穿、相对论时空、基因编辑），但教学工具仍停留在一维文字/二维图片时代，3D交互是唯一能匹配复杂概念的呈现方式。
- **提示词工程是核心杠杆**：同一需求，会提问与不会提问的人得到的结果天差地别，Gemini 3.1能理解"让学生看得懂、玩得起来"这类模糊需求并转化为代码，是质的飞跃。

### 🎯 关键洞察

**为什么这个工具能成立（逻辑链）**：

- 原因：传统教学可视化开发链路长（PM→设计→开发），一个简单页面需要数天，教育工作者根本无法自给自足。
- 动作：用Gemini 3.1作为代码生成引擎，配合精心设计的提示词工程（AetherViz Master作为"桥梁"），将"主题描述"直接映射为"完整可用网页"。
- 结果：教育工作者专注"教什么"，AI负责"怎么做网页"；学生专注"理解概念"，而非"理解PPT"。

**版本迭代意义**：v5.0是经过5个版本迭代才稳定的产物，核心突破在于渲染模式的自动判断逻辑和教学功能的集成方式，不是一蹴而就。

### 🛠️ 操作流程

**方式一：Claude Code（推荐）**

1. **准备阶段**: 确保已配置Claude Code环境，安装AetherViz Master Skill。
2. **核心执行**: 直接在Claude Code中输入你想学习的主题，例如：
   - `"勾股定理"`
   - `"DNA复制"`
   - `"二分查找"`
3. **验证与优化**: 系统自动调用AetherViz Master生成网页，浏览器打开验证交互效果，可调整主题描述重新生成。

**方式二：本地运行**

1. **准备阶段**: 确保本地已安装Git和Python。

2. **核心执行**:
```bash
git clone https://github.com/andyhuo520/aetherviz-master.git
cd aetherviz-master
python -m http.server 8080
# 浏览器打开 https://localhost:8080
```

3. **验证与优化**: 在页面输入主题，等待几秒，交互式教学页面生成完毕，直接可用。

### 📝 避坑指南

- ⚠️ **AI引擎不可随意替换**：开发者明确测试过其他模型效果不达标，Gemini 3.1是当前可用的核心引擎，换模型前需自行验证生成质量。
- ⚠️ **提示词质量直接影响输出**：主题描述越具体，生成效果越好。"力学"比"物理"好，"牛顿第二定律小车实验"比"力学"更好。
- ⚠️ **本地运行需Python环境**：`python -m http.server 8080` 依赖Python3，Windows用户注意命令可能是 `python` 或 `python3`。
- ⚠️ **项目今天才诞生（2026-02-22）**：v5.0是首发版本，生产环境使用前建议充分测试边缘case。
