### 📋 核心分析

**战略价值**: 将电商评论分析从「小时级n8n工作流」升级为「分钟级Claude Code Skill」，通过Python多线程并发+22维AI打标，自动输出CSV数据表+Markdown报告+HTML可视化看板三件套。

**核心逻辑**:

- **零额外API成本打标**：标签挖掘阶段直接调用Claude CLI内置模型，无论是Claude订阅还是第三方Coding plan均可使用，不产生额外API费用
- **10倍速度提升的技术原理**：引入Python多线程并发机制，后台同时启动数十个AI子进程并行处理；单条评论处理时长从10s压缩至1~2s；100条评论仅需2分钟，原n8n版本处理500条需1.5小时
- **动态批次调度**：Skill根据评论总量自动调整「批次并发数」和「单批次评论数」，无需手动配置
- **模糊列名匹配**：脚本内置智能识别逻辑，自动提取不同来源CSV中的评论字段，兼容「评论内容」「评价」「body」「review」等各类列名，不再绑定卖家精灵模板
- **22维标签体系覆盖8大维度**：人群（性别/年龄段/职业/购买角色）、场景（使用场景）、产品（功能满意度/具体功能）、质量（材质/做工/耐用性）、服务（发货/包装/客服/退换货/保修）、体验（舒适度/易用性/外观/价格感知）、市场（竞品对比/复购意愿）、情感（总体评价）
- **评论价值评分机制**：综合字数、有效标签数、是否提及竞品进行打分，报告生成阶段Agent优先分析高分评论，过滤「好评」类无效噪声
- **三档AI引擎组合**：打标用Claude CLI（省token），文字报告和HTML看板推荐Gemini 3.1 Pro（前端构建质量断档领先其他模型）；若本地无Gemini API KEY可在终端实时输入或回退Claude本地模式
- **HTML看板核心价值点**：满意度/使用场景/情感分布渲染为交互式图表（鼠标悬停显示数值）；VOC模块提取典型用户画像，每个用户包含「画像描述+核心需求+原声引用+Agent解析」四要素
- **建议单次处理上限500条**：首次运行建议先跑10条验证流程，再按需扩量，不建议超过500条
- **Skill可作为上游模块**：评论分析Skill可作为「产品设计Skill」和「Listing生成Skill」的前置调用，构成电商业务链条

### 🎯 关键洞察

**n8n工作流 vs Skill 的本质差异**：n8n是固定管道，每次改需求要改节点；Skill是可对话的业务框架，直接跟Agent说需求即可完成调整，无需改代码。作者判断「固定工作流是过渡，Skill才是未来」。

**Gemini 3.1 Pro在HTML生成上的优势**：作者测试多个主流模型后，Gemini在前端构建（交互式图表、黑金风格看板）上表现断档领先，这是选择Gemini作为看板渲染引擎的核心原因。

**评论价值分层的商业意义**：以蓝牙耳机为例，AI从评论中识别出「约16%负面反馈集中在蓝牙连接稳定性（频繁断连、单耳无声）」，人工翻评论需大半天，Agent几分钟定位核心痛点。VOC模块还还原了「小耳道女性用户因尺寸宣传不符导致退货」的完整路径，直接指向产品改进方向。

### 🛠️ 操作流程

1. **准备阶段**
   - 安装：`npx skills add buluslan/review-analyzer-skill`
   - 准备评论CSV文件（任意来源，列名不限）
   - 可选：申请Gemini API KEY（`https://aistudio.google.com/api-keys`）

2. **核心执行**
   - 在Claude Code终端发送：`请帮我使用电商评论分析Skill，分析【把评论文件拖到对话框】`
   - 交互菜单依次确认三项：
     - ① 打标评论数量（首次建议10条，上限500条）
     - ② AI引擎组合（打标用Claude CLI，报告+看板推荐Gemini 3.1 Pro）
     - ③ 报告署名（公司名或个人ID，可选）
   - 若选Gemini模式，Skill自动检测本地KEY，未配置则提示输入或回退

3. **验收成果**
   - 等待进度条跑完（100条约2分钟）
   - 获取三个输出文件：
     - `*.csv`：带22维标签+评论价值分数的打标数据表
     - `*.md`：含用户画像/核心卖点/主要痛点/改进建议/潜在机会的深度报告
     - `*.html`：双击浏览器直接打开的黑金风格交互看板（含图表+VOC模块）
