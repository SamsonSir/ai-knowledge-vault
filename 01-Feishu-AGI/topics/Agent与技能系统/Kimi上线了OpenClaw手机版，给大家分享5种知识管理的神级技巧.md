### 📋 核心分析

**战略价值**: 用 KimiClaw（Kimi 的 OpenClaw 手机端）+ 定时任务 + 长期记忆，构建一套「零摩擦、自动运转」的个人知识管理系统，替代手动整理 RSS/笔记的低效工作流。

**核心逻辑**:

- **KimiClaw 两种接入方式**：①一键部署（云端新建实例，自带 K2.5 Thinking 模型 + Kimi Search 联网搜索 + 40G 云盘，约 1 分钟完成）；②关联已有实例（保留本地 OpenClaw 的全部配置、记忆、已安装 Skill，无需重新配置，持续迭代同一个主力实例）。
- **YouTube 定时摘要**：安装 `https://clawhub.ai/therohitdas/youtube-full` 这个 Skill，设定每天 08:00 自动抓取指定频道过去 24-48 小时新视频，生成含核心摘要（2-3 要点）+ 视频标题/频道名/链接的报告；Agent 会记住频道偏好列表，支持随时增删。
- **seen-videos.txt 去重机制**：维护一个本地文件记录已处理视频 ID，每次任务只处理文件中不存在的新视频，处理后写入 ID，彻底避免重复消耗 Token。
- **目标驱动的每日任务规划**：把长期目标（职业/产品/个人成长/决策原则）写入 Agent 记忆，让它每天 08:00 主动策划 4-5 项当天可在电脑上完成的具体任务，Agent 充当「目标管理教练」角色。
- **MVP 原型自动生成**：在目标规划提示语末尾追加「根据今天我跟你对话信息开发一个能解决我问题的小应用原型（MVP）」，KimiClaw 支持 HTML 预览，可直接生成可视化原型。
- **Next.js 看板展示任务系统**：用提示语驱动生成含「待办/进行中/已完成」三列的 Kanban Board，要求实时同步任务状态，KimiClaw 内置 HTML 预览支持直接渲染。
- **邮件 Newsletter 自动过滤**：用独立邮箱账号 + Playwright MCP 订阅 Andrej Karpathy 推荐的 92 个长文订阅源（含邮件订阅部分，参考 `https://gist.github.com/emschwartz/e6d2bf860ccc367fe37ff953ba6de66b`），每天 20:00 定时读取过去 24 小时订阅邮件，整理核心要点摘要 + 原文链接，完成后主动询问筛选评价，根据反馈更新记忆优化偏好。
- **第二大脑 Next.js 系统**：用提示语驱动生成一个本地 Web App，功能包含：全量记忆/对话列表（支持搜索）、Cmd+K 全局搜索、按日期/内容类型多维筛选、Minimal UI 设计风格，作为所有笔记/对话/记忆碎片的统一入口。
- **财报追踪「左脚踩右脚」Cron Job**：每周日 18:00 运行，自动搜索下周科技/AI 公司财报日历 → 过滤关注公司（NVDA/MSFT/GOOGL/META/AMZN/TSLA/AMD 等）→ 等待人工确认 → 为每家公司财报时间分别设置单次 One-shot Cron Job → 财报发布后立即抓取并生成格式化报告（Beat/Miss、营收、EPS、核心指标、AI 亮点、Guidance）→ 记录常追踪公司偏好，每周自动推荐。
- **Obsidian 打通**：KimiClaw 连接本地 OpenClaw 后，安装 `obsidian-direct` Skill，将 Obsidian 作为知识库，所有自动化产出（摘要/笔记/任务/记忆）可直接写入 Obsidian vault。

### 🎯 关键洞察

**为什么用 Agent 做知识管理而不是手动整理**：

- 传统 RSS/网页版 AI 总结的核心缺陷是「无记忆」——每次都是全新对话，无法根据你的反馈持续优化筛选偏好。
- OpenClaw 的长期记忆机制让 Agent 能记住「你喜欢什么、不喜欢什么」，随时间迭代出越来越精准的信息过滤器。
- 定时任务解决了「主动推送 vs 被动拉取」的问题：信息主动找你，而不是你去找信息。
- 知识库本质上是给 AI 看的索引，人只需要消费 AI 整理好的输出，而不是维护知识库本身。

**「左脚踩右脚」原地飞天逻辑**：让 Agent 自己生成 Cron Job 来触发下一个 Cron Job（财报日历 → 单次精准定时任务），用 Agent 的规划能力动态生成调度计划，而不是人工预设固定时间表。

### 🛠️ 操作流程

**1. 准备阶段**

- 选择 KimiClaw 接入方式：有本地 OpenClaw → 选「关联已有实例」；没有 → 选「一键部署」（1 分钟完成）。
- 准备独立邮箱账号，用于订阅 Newsletter，与主邮箱隔离。
- 在 Clawhub 安装 `youtube-full` Skill：`https://clawhub.ai/therohitdas/youtube-full`。

**2. 核心执行（按场景）**

场景一：YouTube 定时摘要
```
每天早上 8 点，抓取下面YouTube频道的最新视频，生成一份包含核心摘要：
@mreflow

针对每条新视频（过去 24-48 小时内发布）：
获取视频字幕。
用2-3个要点总结核心内容。
附上视频标题、频道名称及链接。

特殊说明：
如果某个频道ID无法识别，请通过搜索找到正确的频道。
请记住我的频道列表，方便我后续增删。
```

场景二：YouTube 去重版
```
每天上午9:00，在 YouTube 上搜索关于"OpenClaw"的新视频。

排重机制：
维护一个名为 seen-videos.txt 的文件，记录已处理过的视频 ID。
仅针对该文件中不存在的新视频获取字幕。
处理完成后，将新视频 ID 存入 seen-videos.txt。

单条视频处理要求：
获取视频字幕。
生成3个要点的精简摘要。
标注出任何与我工作相关的内容。
```

场景三：目标驱动每日任务规划
```
这是我的个人目标，记下来并在后续所有任务中作为默认参考：

职业发展：
1 小红书账号做到10万订阅
2 建立可复现的内容方法论与系列栏目，持续增长与稳定爆款

产品发布：
1 在Q3前发布我的产品
2 以跑通核心闭环的MVP为交付标准，优先可用与迭代

个人成长：
1 每月阅读1本设计相关书
2 每本书输出结构化笔记，并落地到内容或产品中

决策原则：
1 优先做能同时推动增长与产品的事
2 优先做可复现可沉淀的事

每天早上 8:00，请根据我的长期目标，主动策划 4-5 项你今天可以在我电脑上完成的任务。
```

可选追加（自动生成 MVP）：
```
根据今天我跟你对话信息开发一个能解决我问题的小应用原型（MVP）
```

场景四：Kanban 看板
```
请使用 Next.js 为我构建一个看板（Kanban Board），用于展示你正在处理的所有任务。

功能要求：
状态分列：看板需包含"待办 (To Do)"、"进行中 (In Progress)"和"已完成 (Done)"三个列表。
实时更新：当你完成任务时，看板状态必须实时同步更新。
```

场景五：邮件 Newsletter 过滤
```
我希望你每天晚上 8 点运行一次定时任务，读取过去 24 小时内的所有订阅邮件（Newsletter）。
用Clawhub搜索合适的skills完成这个任务。
执行细节：
整理一份核心要点摘要，并附上原文链接以便深度阅读。
完成后询问我对本次内容筛选的评价。
根据我的反馈更新你的记忆，不断优化筛选偏好。
```

场景六：第二大脑系统
```
我想搭建一个第二大脑系统 (Second Brain)，用于复盘我们所有的笔记、对话记录和记忆碎片。使用 Next.js 完成开发。

功能清单：
全量列表：一个支持搜索的列表，展示所有记忆点和历史对话。
全局搜索 (Cmd+K)：支持通过快捷键唤起全局搜索，覆盖系统内所有内容。
多维筛选：支持按日期和内容类型进行过滤。
设计风格：界面需保持极致简约（Minimal UI）。
```

场景七：财报追踪 Cron Job
```
每周日18:00，运行一次Cron Job执行以下操作：
财报检索：搜索下周科技与AI公司的财报日历。
目标筛选：过滤出我关注的公司（如 NVDA, MSFT, GOOGL, META, AMZN, TSLA, AMD 等）。
人工确认：等待我确认具体要追踪哪几家公司。

追踪逻辑（当我回复要追踪的公司后）：
精准调度：为每家公司的财报发布时间分别设置一个单次定时任务（One-shot Cron Job）。
结果抓取：财报发布后，立即搜索业绩结果。
格式化摘要：整理一份包含以下内容的报告：超预期/逊于预期（Beat/Miss）、营收、每股收益（EPS）、核心指标、AI 相关亮点以及业绩指引（Guidance）。
偏好记忆：记录我常追踪的公司，以便每周自动向我推荐。
```

**3. 验证与优化**

- 首次运行后检查 `seen-videos.txt` 是否正确写入视频 ID。
- 邮件任务完成后主动给 Agent 反馈（「这条不感兴趣」/「这类内容多推」），让记忆系统迭代。
- 定期检查 KimiClaw 的记忆列表，确认目标和偏好被正确记录。
- Obsidian 集成后验证 `obsidian-direct` Skill 是否正确写入 vault 路径。

### 📝 避坑指南

- ⚠️ RSS 方案目前容易断连，Newsletter 场景优先用独立邮箱 + Playwright MCP 方案，RSS 有稳定解决方案再切换。
- ⚠️ 频道 ID 识别失败时不要手动查，直接在提示语里加「如果某个频道ID无法识别，请通过搜索找到正确的频道」，让 Agent 自行处理。
- ⚠️ 有本地 OpenClaw 实例的，务必选「关联已有实例」而非「一键部署」，否则历史记忆和 Skill 配置全部丢失，需要重新积累。
- ⚠️ `seen-videos.txt` 必须存在 OpenClaw 实例的持久化存储中，否则重启后去重失效，重复消耗 Token。
- ⚠️ 财报 One-shot Cron Job 是动态生成的，需要人工在每周日 18:00 后回复确认公司名单，才会触发后续精准调度。
