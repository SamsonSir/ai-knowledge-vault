### 📋 核心分析

**战略价值**: OpenClaw 通过「定时器 + 心跳机制」将被动 AI 工具改造成主动运转的本地个人 AI 助手，核心差异在于 agent 能自己给自己找事干。

**核心逻辑**:

- OpenClaw 原名 Clawdbot，因收到 Anthropic 律师函改名 Moltbot，最终定名 OpenClaw，截至直播当晚 GitHub star 超 20 万，是 GitHub 历史上增速最快的开源项目之一
- 时间线：2024年2月 Claude Code 发布 → 3月 Manus 出现 → 11月大量非程序员用户涌入 → 开源社区爆发，OpenClaw 跑得最快
- 核心 agent 框架叫 Pi agent，不到 150 行代码，只定义了 bash、read、write、edit 四个工具，就能跑起一个可工作的 agent
- 公式：**小龙虾 = Claude Code + 定时器 + 自己没事给自己找事干**，两个机制让它从被动工具变成主动助手
- 定时任务（Cron）支持三种模式：一次性执行、周期性执行、标准 Cron 表达式；关键是 agent 自己也可以主动添加任务，不只是用户设置
- 心跳（Heartbeat）机制：每隔 30 秒系统自动给 agent 发一条消息，内容来自 heartbeat.md，agent 有事就做，没事返回特定关键词，系统收到后不打扰用户
- Soul 机制：把系统提示词中「agent 是谁、行为风格」那块单独抽出来存成 soul.md，agent 启动时自动读取；可在社区分享 soul.md，别人下载后 agent 即拥有对应风格
- 本地部署 vs 云端部署差距极大：云端无本地数据，能力等同 Manus；本地可读桌面所有文件、清理磁盘、调整电池策略、查找文件，是云端做不到的
- 模型选择：复杂任务首选 Claude，国产模型跑通流程够用但有精度问题（实测某国产模型清理磁盘后汇报可用空间从 25G 越算越小变成 21G）；有 Claude Code 订阅的用户可将 agent core 替换成 Claude Code CLI，复用订阅不走 API，成本更低
- OpenClaw 一周新增约 5000 个 commit，对比普通工程师每天 10-20 个 commit 连续干一年才积累 5000 个，说明 99% 是非人类在更新，作者 Kidd 大概率让 agent 简单看一眼就 merge，导致近期出现莫名 bug，通常过一两周更新后自愈

### 🎯 关键洞察

**心跳机制是本质差异所在**：Claude Code 是「踹一下动一下」，OpenClaw 是「自己隔 30 秒醒一次看有没有活干」。这个差异让 agent 从工具属性升级为搭档属性。

**权限与风险的张力**：来新璐实测，让 agent「继续干你所有能干的事」，结果清理磁盘时差点删掉甲方客户数据。本地部署权限越大能干的事越多，但失控风险也越高。这是目前本地 agent 最核心的工程问题，没有银弹。

**agent 驱动开发的极端案例**：OpenClaw 一周 5000 commit 是 agent 驱动开发的真实写照，但代价是代码质量管控几乎失效，用户需要接受「用最新版可能踩 bug，等两周自愈」的使用节奏。

### 📦 配置/工具详表

| 模块 | 关键设置/代码 | 预期效果 | 注意事项/坑 |
|------|------------|---------|-----------|
| Pi agent 核心 | 不到 150 行代码，定义 bash/read/write/edit 四工具 | 可工作的最小 agent | 这是 agent core，功能极简，能力上限取决于外挂机制 |
| Cron 定时任务 | 支持一次性、周期性、标准 Cron 表达式；agent 自身也可主动添加任务 | agent 可提前备料，如每晚 12 点扫 GitHub issues | agent 自主添加任务需注意任务堆积失控 |
| Heartbeat 心跳 | 每 30 秒触发一次，内容读取 heartbeat.md；无事可做时返回特定关键词 | agent 主动轮询待办，有生命感 | heartbeat.md 内容质量决定 agent 主动行为质量 |
| Soul 灵魂 | 将人格/行为风格 prompt 存为 soul.md，启动时自动读取 | 风格固化、可分享、可复用 | 和 skill.md 逻辑相同，只是用在人格设定上 |
| 模型替换 | 有 Claude Code 订阅可将 agent core 替换为 Claude Code CLI | 复用订阅，不走 API 计费 | 国产模型跑复杂任务有精度问题，建议仅用于熟悉流程 |
| 本地部署 | 部署在本机，agent 可访问本地文件系统 | 清理磁盘、调整电池策略、查找文件等 | 权限越松风险越大，需自行做好安全边界 |

### 🛠️ 操作流程（手搓最小龙虾 Claw 0）

1. **准备阶段**
   - 克隆来新璐为直播准备的开源教学仓库 Claw 0（GitHub 搜索 Claw 0 / Share AI）
   - 确认本地已安装 Claude Code CLI 或准备好模型 API Key
   - 了解 Pi agent 结构：bash、read、write、edit 四个工具，不到 150 行代码

2. **核心执行**
   - 配置 soul.md：写入 agent 人格设定和行为风格 prompt
   - 配置 heartbeat.md：写入待办任务和周期性提醒内容
   - 配置 Cron 任务：按需设置一次性 / 周期性 / 标准 Cron 表达式任务
   - 启动 agent，验证心跳是否每 30 秒触发一次
   - 验证 agent 无事可做时是否正确返回特定关键词（不打扰用户）

3. **验证与优化**
   - 交给 agent 一个后台任务（如翻译文档并推送到仓库），开始做其他事
   - 一段时间后检查任务是否自动完成（直播现场：20 分钟后英语和日语文件夹已出现在仓库）
   - 测试 agent 调用系统命令能力（如 FaceTime 命令行参数直接拨号）
   - 逐步收紧或放开权限边界，观察 agent 行为，找到适合自己的安全阈值

### 💡 具体案例/数据

- **后台翻译任务**：直播开始前交给 agent 任务——将 Claw 0 教学文档翻译成英语和日语并推送仓库。直播进行约 20 分钟后刷新 GitHub 页面，英语和日语两个版本文件夹已静默出现，来新璐本人都没注意到它在后台完成了。
- **FaceTime 拨号**：来新璐让 agent 给直播间网友打 FaceTime，agent 通过 FaceTime 命令行参数直接操作，驱动模型为智谱 GLM，未配置任何浏览器 MCP，电话真实拨出并接通，网友陆萌被拉进直播做即兴连线。
- **国产模型精度问题**：用某国产模型清理磁盘，agent 记录了每项清理了多少空间，但最终汇报可用空间时算错，从原来 25G 越算越小变成 21G。
- **一周 5000 commit**：OpenClaw 一周新增约 5000 个 commit；普通工程师每天 10-20 个 commit，连续工作一年才积累 5000 个。项目作者 Kidd 大概率让 agent 简单 review 后直接 merge，导致近期 bug 增多。

### 📝 避坑指南

- ⚠️ 云端部署能力极度受限，和 Manus 无本质区别，本地部署才是 OpenClaw 的正确打开方式
- ⚠️ 给 agent 开放过大权限（如「干你所有能干的事」）可能导致误删重要文件，本地数据务必提前备份
- ⚠️ 国产模型跑复杂计算任务有精度问题，生产环境建议用 Claude；熟悉机制阶段用国产模型跑通流程即可
- ⚠️ OpenClaw 当前版本 99% 由 agent 自动提交，代码质量管控弱，遇到莫名 bug 等一两周更新通常自愈，不必立刻深挖
- ⚠️ heartbeat.md 内容写得越清晰，agent 主动行为越可控；内容模糊会导致 agent 乱找事干
- ⚠️ 有 Claude Code 订阅的用户，将 agent core 替换成 Claude Code CLI 可显著降低成本，避免 API 重复计费
