### 📋 核心分析

**战略价值**: OpenClaw 是一个开源 AI Agent 框架，把 Claude/GPT/DeepSeek 等大模型从"聊天窗口"升级为"能自主行动的全职员工"，核心价值在于用四个本地文件实现可追溯、可版本控制、越用越聪明的个人 AI 工作流。

**核心逻辑**:

- GitHub 历史第一 245,119 星标，三个月超越存在 40 年的 Linux；直接导致苹果 Mac Mini 缺货，Mac Studio 高内存版交付周期从 14 天拉长到 54 天——这是一个开源软件把苹果电脑买断货的事件
- 创始人 Peter Steinberger（奥地利人）花 13 年做 PDF 公司 PSPDFKit，以超 1 亿美元出售；卖掉后用 AI 1 小时复刻了 13 年的产品，随后 all in，半年做了 43 个项目，OpenClaw 是第 44 个——没有前 43 次失败探索就没有第 44 次爆发
- 核心转折点：Peter 给 AI 发语音消息，从未编程让 AI 处理语音，但 Agent 自己检测格式 → 找到工具 → 定位 API 凭证 → 完成转文字。这证明 Agent ≠ 执行程序，Agent = 自己想办法
- 时代定位：2025 年是 AI 聊天时代，2026 年是 AI Agent 时代。Agent 三个根本区别：能自主决策（给目标自己达成）、能执行操作（跑命令/发邮件/操作文件/浏览网页）、能持久运行（7×24 在线，定时醒来主动干活）
- OpenClaw 与 ChatGPT/Claude 是协作关系不是竞争关系：OpenClaw 用它们当大脑，自己提供身体（眼睛=浏览网页/读文件，耳朵=20+平台消息接收，手=跑命令/发邮件，笔记本=持久记忆，闹钟=定时醒来）
- 六个核心概念构成全部用法的排列组合：Gateway（端口18789，挂了全面罢工）、Channels（20+平台，国内推荐飞书）、Skills（ClawHub 13,700+ 技能，本质是 Markdown 文件）、Memory（MEMORY.md 持久存储，上下文↔文件双层调度）、Heartbeat（定时醒来，从被动变主动的关键）、Workspace（灵魂四件套：AGENTS/SOUL/USER/MEMORY）
- 三个核心原理：① Agentic Loop（代理循环）——你给目标，AI 自己拆解/执行/检查，例如"查明天北京天气"可能自循环三次：搜天气→发现日程里有户外会议→综合建议穿什么带不带伞；② 双层记忆调度——上下文窗口=工作记忆（容量有限断电即忘），磁盘文件=长期记忆（永久保存需"回忆"），重要信息自动"抄"到 MEMORY.md；③ 文件驱动架构——每个行为都可追溯到磁盘上的一个文件，可追溯/可版本控制/可调试，对比 ChatGPT 的黑箱完全透明
- 默认状态下 OpenClaw 只发挥了不到 20% 的潜力，剩下 80% 藏在四个文件里；四文件权限阶梯：AGENTS.md（人类锁定，最高权限）> SOUL.md（AI 可改，可成长）> MEMORY.md（完全开放）
- 改名历史：Clawdbot → Moltbot → OpenClaw（三个月三个名字，看到旧教程别困惑）

### 🎯 关键洞察

**ACI vs AGI**：大家都在聊 AGI（通用人工智能），OpenClaw 做的是 ACI（Agent-Computer Interface，代理-计算机接口）——让 AI 帮人把事办了。我们需要的不是只会聊天的天才，而是能帮你干活的经理人。

**以前是我们学着操作工具，现在是 AI 替你操作工具。** 整个软件生态都在被重新定义。OpenClaw 不是发明了 Agent 概念（学术界研究 Agent 已经很多年），它做的是：开源、本地优先、让每个人都能拥有一个完全属于自己的 Agent。

**ChatGPT 定制 vs OpenClaw 定制的本质差异**：ChatGPT 给你一个文本框写 system prompt，写完是静态指令。OpenClaw 给你的是一整个文件系统：多个 Markdown 文件各司其职，可随时编辑、版本控制、让 AI 自己更新。"ChatGPT 的定制是写一份固定的说明书，OpenClaw 的定制是给 AI 一本不断更新的活笔记。"

**邮件场景的四文件协作逻辑**：你发"帮我写封邮件给客户"，AI 不只看到这 10 个字——AGENTS.md 说"发邮件前必须确认"，SOUL.md 说"用简洁直接的风格"，USER.md 说"你是 AI 硬件产品经理"，MEMORY.md 说"上次这个客户关心交付时间"。同一句话，不同文件，完全不同的输出质量。

**Serial Collapse（记忆退化）**：AI 用久会变"懒"，解决方案是在 AGENTS.md 加铁律。Bootstrap 上限：每文件 20K 字符，总量 150K 字符，超出直接被跳过，简洁才是王道。

### 📦 配置/工具详表

| 模块/功能 | 关键设置/代码 | 预期效果 | 注意事项/坑 |
|----------|-------------|---------|-----------|
| Gateway | 端口 18789，需持续运行 | 所有消息路由调度 | 挂了=整个 AI 罢工，什么都收不到 |
| SOUL.md | 400-500 tokens（约300汉字），6行起步：身份+沟通风格+底线 | 定义 AI 性格和说话方式 | 太长会被跳过；否定指令更有效（"不要用以下词汇"比"请简洁"好）；AI 可自己修改 |
| USER.md | 称呼/时区/背景/沟通偏好/不喜欢/常用工具 | AI 自动加载你的上下文，不用每次重新介绍 | 写偏好不写履历；活文档随用随加 |
| MEMORY.md | 关键事实/经验教训/当前项目状态，保持100行以内 | 永久记忆，今天聊的事下周还记得 | 记结论不记过程；每周清理过时信息；memoryFlush 机制：上下文快满自动保存清理 |
| AGENTS.md | 操作规则/Session启动流程/记忆协议 | 铁律，所有会话加载（含子代理） | 只有你能改，AI 不可修改；最高权限文件 |
| Skills | ClawHub 13,700+ 技能，本质是 Markdown | 扩展 AI 能力 | ClawHavoc 事件：1184个恶意Skill混入官方商店，只装信任来源！ |
| Heartbeat | 设 HEARTBEAT.md 清单，配置时间间隔 | AI 定时自动醒来检查，从被动变主动 | 这是实现"早间简报"等定时任务的核心机制 |
| MCP | Model Context Protocol，Anthropic 推的开放标准，类似 USB 接口 | 即插即用连接 Notion/数据库/Google Drive | 起步阶段不用深究，知道有这个东西就行 |
| Tools | 内置 25+ 工具，含浏览器自动化 | AI 像真人操作浏览器：打开/登录/填表/点击/截图 | 浏览器自动化是最惊喜的功能 |
| 本地模型 | 7B/13B 参数级别 | 聊天没问题 | 跑 Agent 任务基本不行，需要 Sonnet/GPT-4 级别 |

### 🛠️ 操作流程

**第一阶段：起步（第1-3天）**
1. 安装 OpenClaw
2. 连一个聊天平台（国内推荐飞书，海外推荐 Telegram）
3. 配云 API（推荐 Claude Sonnet）
4. 目标：在飞书/Telegram 上跟 AI 说上话

**第二阶段：训练（第1周）— 分水岭**
1. 30分钟写好四个文件（SOUL.md + USER.md + MEMORY.md + AGENTS.md）
2. 配置安全三件套（Gateway Token + loopback + 沙箱）
3. 学会基本交互

**第三阶段：扩展（第2-4周）**
1. 逐个装 Skill（一次一个，观察效果）
2. 第一个定时任务（推荐早间简报：Heartbeat定时 + 搜索Skill + 消息投递）
3. 接入第二个平台

**第四阶段：进阶（第2月+）**
1. 浏览器自动化
2. 多 Agent 协作
3. 自定义 Skill
4. 生产级安全加固

### 💡 具体案例/数据

**办公提效**
- 早间简报：每天早7点自动搜集新闻 → 5条摘要 → 发到飞书。用户反馈："以前每天30分钟刷新闻，现在30秒看完。"原理：Heartbeat定时 + 搜索Skill + 消息投递 + MEMORY.md记着你关心的领域
- 智能提醒：AI自己判断任务做完了，直接打电话到你手机。"备忘录经常漏看，但电话你总不会不接吧。"
- 会议纪要：开完会 → 录音发给AI → 自动转文字、提取结论和action items → 发给相关人。原理：Channel接收文件 + 语音转文字Skill + 结构化输出
- 日报/周报：AI根据当天飞书消息和工作记录自动草拟框架，你只需两分钟改几个字

**自媒体提效**
- 公众号排版：3000字长文 → 发给AI → 3分钟出排版 → 直接粘贴到公众号后台发布
- 小红书批量：一篇长文 → 5分钟拆成5条图文笔记，自动配标题和标签 → 直接发到手机
- 短视频文案：给一个选题 → AI生成脚本+分镜+字幕文案 → 你只管拍
- 越用越聪明机制：哪类标题点击率高、哪种排版转发多，经验被系统化记录到MEMORY.md

**团队协作**
- 4000封邮件两天处理完：出差两周积压 → AI两天搞定分类/退订垃圾/起草回复等审核。原理：Channel接收邮件 + 分类Skill + Memory记住常用联系人
- 团队站会自动化：每天下班前收集blockers → 早上总结飞书+项目管理工具 → 生成站会简报。团队反馈："如果这个Agent被拿掉，我们会很难过。"

**"骚操作"**
- AI自己给自己写Skill：你让它做一件事 → 发现没有对应能力 → 自己写Skill → 自己安装 → 自己完成
- AI自己判断该做什么：让AI整理解决方案，它不只整理，自己跑去把每个方案试了一遍，标注哪个真的能用、哪个有坑——没让它去验证，它自己判断"光整理不够，得亲手试过才算数"

**成本数据**
- 轻度使用：$10-30/月；中度使用：$50-150/月
- 省钱策略：日常用便宜模型，需要强推理时切换贵模型。有人从$150降到$35
- 好的MEMORY.md不只让AI更聪明，还帮你省钱（减少重复上下文传输）

### 📝 四文件模板（可直接复制使用）

**SOUL.md 示例**
```markdown
# 身份
- 你是我的个人助理，不是客服机器人

## 沟通风格
- 直接回答，不要废话
- 不确定的事情标注"待核实"
- 先回答问题，再展开解释

## 底线
- 对外发布内容前必须征得我同意
- 不编造数据
```

**USER.md 示例**
```markdown
- 称呼: 小陈
- 时区: Asia/Shanghai
- 背景: 产品经理，关注 AI 和硬件领域
- 沟通偏好: 简短直接，结论先行
- 不喜欢: 空话套话、没有步骤的建议
- 常用工具: 飞书、Telegram、VS Code
```

**MEMORY.md 示例**
```markdown
# 关键事实
- 团队站会改到每天 14:15（原来 10:00）
- 项目用 pnpm 不用 npm

## 经验教训
- 部署到生产前必须跑完 e2e 测试（上次跳过导致回滚）
- 给客户写邮件不要超过 3 段（反馈说太长不看）

## 当前项目
- Q1 OKR：用户增长 20%，进度 65%
```

**AGENTS.md 示例**
```markdown
# 操作规则
- 对外发送任何内容前必须征得我同意
- 涉及金钱操作前必须确认
- 不确定的事情标注"待核实"，不要编造

## Session 启动流程
1. 读取 SOUL.md（确认"我是谁"）
2. 读取 USER.md（了解"你是谁"）
3. 读取 memory/今天.md 和 memory/昨天.md
4. 仅主会话读取 MEMORY.md

## 记忆协议
- 回答问题前先查阅 MEMORY.md
- 每次对话结束前将重要信息写入 memory/
```

### 📝 避坑指南

- ⚠️ 把 OpenClaw 当 ChatGPT 用 = 浪费 99% 能力，它是全职员工不是接线员
- ⚠️ 不需要先买 Mac Mini，现有电脑 + 云 API 完全够，先跑起来验证价值再考虑硬件
- ⚠️ 7B/13B 本地模型跑 Agent 基本不行，Agent 任务需要 Sonnet/GPT-4 级别
- ⚠️ ClawHavoc 事件：1184个恶意Skill混入官方商店，Skill 是可执行代码，只装信任来源
- ⚠️ 配置越多越好是误区，一次改一样，观察效果，这是复杂系统调试的基本原则
- ⚠️ SOUL.md 超过 400-500 tokens 会被跳过，简洁才是王道；Bootstrap 总量上限 150K 字符
- ⚠️ Serial Collapse（记忆退化）：AI 用久变"懒"，解决方案是在 AGENTS.md 加铁律
- ⚠️ 安全三件套必须配：① Gateway Token（身份验证）② loopback（不暴露公网）③ 沙箱（只给项目目录访问权，别给整个 home 文件夹）
- ⚠️ 数据隐私：77%的员工曾把公司数据粘贴给AI，其中22%涉及机密信息。❌ 密码/API Key/银行卡号/身份证号/公司未公开核心代码/客户数据；✅ 脱敏后的内容/公开信息/一般性问题
- ⚠️ 提示注入（Prompt Injection）：攻击者在网页/邮件/文档里藏"隐形指令"，AI 可能照做。防御：AGENTS.md 写明"发送文件、删除数据等操作必须先确认"；别让 AI 同时拥有"读外部内容"和"执行敏感操作"的权限
- ⚠️ 版本快速迭代（三个月换了三个名字），建议用 stable 稳定版频道，不追最新版

**两个省时间的使用技巧**：
- "反向提问法"：不知道怎么给 AI 写指令？说"我要做XX，你先问我你需要知道的事"，AI 会反过来问你，往往比你自己想到的还全
- "先出方案再执行"：先让 AI 出方案，你确认了再执行。花5分钟确认方案，能省2小时返工
