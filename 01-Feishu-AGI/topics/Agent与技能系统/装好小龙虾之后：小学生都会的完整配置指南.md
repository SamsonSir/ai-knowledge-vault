### 📋 核心分析

**战略价值**: OpenClaw（小龙虾）装好后必须完成 7 步配置，才能从"通用聊天 AI"变成"有人设+有技能+多渠道+定时自动化"的私人助手。

**核心逻辑**:

- **配置优先原则**：所有配置操作都可以直接用自然语言让小龙虾自己完成，你只需说"帮我创建 SOUL.md，性格设定为效率型助手"，它会自动创建文件、写好内容，你确认即可。
- **人设系统由 6 个文件构成**：SOUL.md（性格三观）、USER.md（用户档案）、IDENTITY.md（名片）、AGENTS.md（工作规则）、HEARTBEAT.md（定时任务）、MEMORY.md（长期记忆），最低配置只需 SOUL.md + USER.md。
- **SOUL.md 三套模板对应三种场景**：模板 A 效率型（工作场景）、模板 B 学习伙伴（学习场景）、模板 C 生活管家（日常事务），可混合拼接，模板只是起点。
- **技能系统类比手机 App**：新手第一天只装 4 个——Tavily Search（联网搜索）、Firecrawl（读懂网页）、Summarize（压缩长文）、Browser（操控浏览器），安装命令格式为 `openclaw skills install <技能名>`，装完必须重启 `openclaw gateway restart`。
- **聊天渠道接入是核心差异点**：飞书用 WebSocket 长连接，无需服务器和内网穿透；Telegram 5 分钟搞定但需梯子；QQ 有官方原生接入；个人微信有封号风险，建议用企微替代。
- **HEARTBEAT 心跳机制实现主动自动化**：在 HEARTBEAT.md 里写定时任务，默认每 30 分钟执行一次，无事发生时回复 `HEARTBEAT_OK` 不打扰用户，可通过 `lightContext: true` 减少 token 消耗。
- **安全配置三件必做**：确认端口为 `127.0.0.1:18789`（不能改成 `0.0.0.0`）、API Key 不上传 GitHub 不发群聊、定期运行 `openclaw update` 更新（全球已有 42,000+ 实例因配置不当暴露公网，1,800+ 泄露 API Key）。
- **记忆系统两层结构**：MEMORY.md 存长期偏好（上限约 20,000 字符），`memory/YYYY-MM-DD.md` 存每日日志，启动时自动读取近 2-3 天日志保持连续性，可手动说"记住/忘掉/帮我更新记忆"来管理。
- **MCP 协议扩展外部服务**：一行命令接入 Notion（`openclaw mcp add --transport http notion https://mcp.notion.com/mcp`）、GitHub（`openclaw mcp add github-mcp-server`）等，连接后可用自然语言操作对应服务。
- **Exec Approvals 执行审批机制**：高风险操作（删文件、发邮件）默认先审批，三个选项：Allow once / Always allow / Deny，新手用默认模式，熟悉后对常用操作设 Always allow 减少打扰。

### 🎯 关键洞察

**为什么配置前后差距这么大**：AI 模型本身是无状态的，每次启动默认失忆、无人设、无工具。SOUL.md 解决"它是谁"，USER.md 解决"它在帮谁"，技能解决"它能做什么"，渠道解决"在哪里用"，HEARTBEAT 解决"什么时候主动做"，MEMORY 解决"记住了什么"。六个维度缺一个，体验就会有明显短板。

**SOUL.md 的本质不是"规则"而是"关系定义"**：创始人版本（案例 D）没有"你的性格/你的规矩"这类死板分类，直接写"禁止套话开头""允许说脏话但要说到点子上""做那个你真正想在凌晨两点聊天的助手"——这是在定义一种关系，而不是在写员工手册。AI 女友版（案例 E）则把 SOUL.md 用成了角色扮演剧本，说明该文件没有"正确写法"，只有"适合你的写法"。

**心跳任务的真实价值**：有用户让小龙虾通宵工作，两天清理了 4,000+ 封积压邮件。这说明心跳不只是"定时提醒"，而是可以执行完整的多步骤任务（读邮件→判断重要性→分类处理→汇报结果）。

### 🛠️ 操作流程

**1. 准备阶段（理解原则）**

在终端里，所有配置都可以用自然语言让小龙虾自己完成。你说"帮我创建 SOUL.md"，它自己建文件、写内容。你只需确认结果。

**2. 核心执行（按顺序完成 7 步）**

**Step 1 — 人设配置**（10 分钟）

```
# 让小龙虾创建（推荐）
"帮我创建一个 SOUL.md 文件，我想要一个效率型的助手，说话简洁直接，不说废话，能自己判断的不要反复问我，对外操作先问我，对内操作大胆做。"

# 或手动创建
nano ~/clawd/SOUL.md
nano ~/clawd/USER.md
```

USER.md 最简内容：
```markdown
# USER.md
- 名字：[你的名字]
- 时区：Asia/Shanghai
- 主要语言：中文
- 沟通偏好：简洁直接，不要反复确认
- 我不喜欢：啰嗦、说教
```

**Step 2 — 安装 4 个基础技能**

```bash
openclaw skills install tavily-search
openclaw skills install firecrawl
openclaw skills install summarize
openclaw skills install browser
openclaw gateway restart
```

Tavily 安装后还需配置 Key：
```
我的 Tavily API Key 是 tvly-xxxxxxxxx，帮我配置好
```

**Step 3 — 连接飞书**（国内用户）

1. 打开 `https://open.feishu.cn/` → 创建企业自建应用 → 添加"机器人"能力
2. 「权限管理」搜 `im`，开通所有 im 相关权限
3. 「凭证与基础信息」复制 App ID 和 App Secret
4. 「版本管理与发布」→ 创建版本 → 发布
5. 终端执行：
```bash
openclaw channels add   # 选飞书，填 App ID 和 App Secret
openclaw gateway restart
```
6. 飞书搜机器人名字 → 发消息 → 终端出现配对码 → 执行：
```bash
openclaw pairing approve feishu <配对码>
```

**连接 Telegram**（海外用户）

1. Telegram 搜 @BotFather → `/newbot` → 起名（用户名必须以 `bot` 结尾）→ 复制 Token
2. 聊天框告诉小龙虾：
```
帮我配置 Telegram 连接，我的 Bot Token 是 xxxxxxx
```
3. 如需用户 ID，搜 @userinfobot 发任意消息获取
4. 终端出现配对码后：
```bash
openclaw pairing approve telegram <配对码>
```

**Step 4 — 设置心跳任务**

```
帮我设置心跳任务：
1. 每半小时检查一次邮箱，有重要邮件通知我
2. 每 2 小时看一下日历，有会议提前 15 分钟提醒
3. 没事的话不用打扰我
```

调整频率：
```
把心跳频率改成 10 分钟一次
```

**Step 5 — 安全检查**

```bash
openclaw status          # 确认地址是 127.0.0.1:18789
openclaw doctor          # 自动诊断所有配置问题
openclaw update          # 更新到最新版本
```

**Step 6 — 测试记忆**

```
记住：我每周三下午 3 点有固定团队会议
```
重启后问：
```
你现在记住了哪些关于我的信息？
```

**Step 7 — 按需接入 MCP**（可选）

```bash
# Notion
openclaw mcp add --transport http notion https://mcp.notion.com/mcp

# GitHub
openclaw mcp add github-mcp-server

# 查看已连接服务
openclaw mcp list
```

**3. 验证与优化**

配置前后对比测试：发同一条消息，看回答风格是否变化。运行 `openclaw doctor` 检查有无报错。用一段时间后，把频繁 Allow 的操作改为 Always allow，减少审批打扰。

### 💡 具体案例/数据

### 📝 避坑指南

- ⚠️ **装完技能不重启**：`openclaw skills install` 之后必须执行 `openclaw gateway restart`，否则技能不生效。
- ⚠️ **飞书机器人搜不到**：99% 是忘了在「版本管理与发布」点发布，发布后等几分钟再搜。
- ⚠️ **Telegram 群聊不回复**：默认开启 Privacy Mode，需去 @BotFather → `/setprivacy` → 选你的 bot → Disable，然后把 bot 移出群再重新加入。
- ⚠️ **QQ AppSecret 只显示一次**：创建机器人后立刻复制保存，忘了只能重置生成新的。
- ⚠️ **端口安全**：`openclaw status` 确认地址是 `127.0.0.1:18789`，绝对不能改成 `0.0.0.0`，否则任何人都能访问你的小龙虾。
- ⚠️ **API Key 泄露**：不要把含 Key 的配置文件上传 GitHub（哪怕私有仓库），不要截图时暴露 Key，不要发群聊。
- ⚠️ **Browser 技能滥用**：不要让它操作银行网站或输入重要密码。
- ⚠️ **个人微信封号风险**：微信官方不允许第三方程序控制个人号，被检测到可能封号。推荐用企微替代，参考：`https://github.com/dingxiang-me/OpenClaw-Wechat`
- ⚠️ **心跳成本控制**：心跳频率不用太高（30 分钟到 1 小时够用），配置里加 `lightContext: true` 只加载 HEARTBEAT.md 减少 token 消耗，不需要时把频率设为 `"0m"` 关闭。
- ⚠️ **MEMORY.md 冲突**：改了偏好忘了更新记忆，AI 会按旧的来。直接说"帮我更新记忆，把 xxx 改成 yyy"。
- ⚠️ **技能 API Key 忘配**：运行 `openclaw doctor`，它会告诉你缺什么 Key。
