### 📋 核心分析

**战略价值**: 用火山引擎 API 替代 5GB 本地模型，重构口误识别逻辑 + 新增可视化审核界面，将口播剪辑 Agent 升级为「越用越懂你」的自更新系统。

**核心逻辑**:

- **痛点1 → 解法**：原版需下载 5GB 本地模型，耗时数十分钟导致大量用户放弃 → 改为调用火山引擎「音视频字幕生成」API，无需本地模型，启动即用
- **痛点2 → 解法**：原版在命令行文本里逐行找口误，体验极差 → 新增本地 Web 审核界面，类剪映口播剪辑 UI，Shift+拖动选中/取消片段
- **识别模型升级**：从原本地模型换成豆包音视频字幕模型（火山引擎），作者实测识别准确率超过剪映，剪映漏掉的口误被全部识别
- **口误识别系统重构**：AI 逐段语义分析，按规则分类判断：静音、重复句、卡顿词、语气词等，不再是粗暴匹配
- **规则透明化 + 可定制**：所有识别规则存放在独立文件夹，用户直接改 `.md` 文件即可修改规则，新建 `.md` 文件即可添加新规则，无需改代码
- **自更新机制**：审核界面提供「复制删除列表」按钮，将用户实际删除结果复制给 AI，AI 分析用户表达习惯，下次剪辑时识别更精准 —— 形成「用户反馈 → AI 学习 → 更准确」闭环
- **倍速播放功能**：剪映没有的功能，说话慢的视频可加速审听，提升审核效率
- **全流程时间压缩至 15 分钟**：下载 Skills 1 分钟 + 配置 API Key 3 分钟 + 剪口播 10 分钟 + 加字幕 3 分钟
- **专有名词词典**：字幕步骤支持在词典文件中添加专有名词，提升字幕准确率
- **输出命名规范**：剪辑完成后自动生成文件名结尾为 `cut` 的视频文件，便于识别

### 🛠️ 操作流程

**第1步：下载 Skills（1分钟）**

告诉 Claude：
```
帮我下载剪辑 Skills：
- 地址：https://github.com/Ceeon/videocut-skills
- 存放路径：.claude/skills/（直接克隆到此目录，不要创建子文件夹）
```

**第2步：配置 API Key（3分钟）**

1. 登录火山引擎模型管理后台
2. 开通「音视频字幕生成」能力（有免费试用额度，20小时）
3. 获取 API Key
4. 在 Claude 中输入 `/v`，选择 `videocut:安装`，输入 API Key，完成配置

**第3步：剪口播（10分钟）**

1. 输入 `/v`，选择 `videocut:剪口播`，输入视频路径
2. AI 自动执行三步：
   - 调用火山 API 转录音视频
   - 启动本地审核服务
   - 返回审核界面链接
3. 打开链接进入审核界面，操作方式：
   - `Shift + 拖动` → 选中或取消选中片段
   - 「播放」→ 自动跳过已选中片段（同剪映）
   - 「倍速播放」→ 加速审听慢速视频
   - 「复制删除列表」→ 将删除结果复制给 AI 学习习惯
   - 「执行剪辑」→ 确认后自动完成剪辑
4. 剪辑完成，生成文件名结尾为 `cut` 的视频

**第4步：加字幕（3分钟）**

1. 输入 `/v`，选择 `videocut:字幕`
2. 在词典文件中添加专有名词，提升字幕准确率

### 📝 避坑指南

- ⚠️ Skills 克隆路径必须是 `.claude/skills/`，且不能创建子文件夹，否则 Claude 无法识别
- ⚠️ 火山引擎需手动开通「音视频字幕生成」能力，不是默认开通的
- ⚠️ 识别规则文件夹内的规则直接影响口误判断逻辑，上线前建议先跑一段测试视频验证规则是否符合自己习惯
- ⚠️ 「复制删除列表」功能是训练 AI 习惯的关键，差异大时务必使用，否则自更新机制无法生效
