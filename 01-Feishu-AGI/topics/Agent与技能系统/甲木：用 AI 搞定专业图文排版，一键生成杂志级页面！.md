### 📋 核心分析

**战略价值**: 用 Claude Code 的 Agent Skills 机制，将「文本→杂志级HTML/PDF」的排版流程封装成可复用 Workflow，彻底消灭重复性排版耗时。

**核心逻辑**:

- **Skills 的本质是文本化 Workflow**：抛弃拖拽节点/流程图，把所有逻辑浓缩进可迭代的 Markdown 文件，与 AI 的关系从「点按钮」变成「对话式协作」。
- **单次 Prompt 不够用**：排版 = 风格选择 + 结构映射，AI 需要明确知道什么是标题、哪里留白、哪里强调，一次 prompt 无法稳定复现。
- **Skills 的 references/ 目录解决了上下文窗口限制**：把 12 种风格模板拆成模块放进 `references/styles.md`，AI 每次都能参考完整体系，而不是在「一两个示例」里模仿。
- **风格选择必须显性化**：排版是审美决策，跳过风格确认步骤必然导致后期反复返工，因此 Skill 强制调用 `AskUserQuestion` 工具让用户选择。
- **12 种风格覆盖全场景**：从经典优雅到暗黑科技，每种风格绑定明确适用场景（见下方表格），用户也可上传自定义风格。
- **结构极度克制，只有 3 个文件**：`SKILL.md`（角色+流程）、`references/styles.md`（风格模板库）、`scripts/html_to_pdf.py`（HTML→PDF），没有多余依赖。
- **4 步强制流程写死在 SKILL.md 里**：分析结构 → 讨论风格 → 生成 HTML → 按需导出 PDF，每步有明确规范，不可跳过。
- **HTML 生成规则严格**：首段 `drop-cap` 首字下沉、重要语句 `blockquote` 引言样式、章节间风格专属分隔线、正确标题层级 h1>h2>h3、必须包含智能分页 CSS、所有 CSS 内联或 CDN 引入（Tailwind/Google Fonts）、输出完整自包含 HTML。
- **「非自建不可用」是正确的**：每个人的流程、标准、审美不同，作者的 Skill 未必适合你，但「把重复耗注意力的事封装成 Skill」这个思路是通用的。
- **已开源，可直接 Fork 改造**：仓库同时包含杂志排版 Skill 和私董会 Skill。

### 🎯 关键洞察

**为什么 Skills 结构能解决排版问题的根本矛盾？**

- 原因：单窗口 prompt 的上下文有限，AI 只能在「一两个示例」里模仿，风格一致性无法保证。
- 动作：把 12 种风格模板统一放进 `references/styles.md`，作为长期可复用参考资源，每次执行时 AI 都能读取完整体系。
- 结果：从「排一次」扩展到「排一百次」，风格稳定可复现，且支持用户自定义扩展。

**为什么强制风格确认步骤不能省？**

- 排版本质是审美决策，不显性化就会在生成后反复返工（「内容是对的，但看起来很累」）。
- 通过 `AskUserQuestion` 工具把决策前置，一次确认，后续自动化执行。

### 🛠️ 操作流程

**1. 准备阶段**

- 安装 Claude Code，确保可以使用 Agent Skills 功能
- Clone 开源仓库：`https://github.com/isjiamu/jiamu-skills`
- 将 `magazine-layout/` 目录放入你的 Skills 路径

**2. 核心执行（SKILL.md 定义的 4 步强制流程）**

- Step 1 — 分析文本结构：AI 自动识别标题/副标题、章节/小标题、重要引言或金句、列表和数据要点、作者/日期信息
- Step 2 — 风格选择（不可跳过）：AI 读取 `references/styles.md`，展示 12 种风格选项，调用 `AskUserQuestion` 工具等待用户确认；用户可选预设风格、混合多种风格元素、自定义颜色/字体/页面尺寸/间距
- Step 3 — 生成 HTML：严格使用选定风格模板结构；保留用户原文全部内容，绝不删减或总结；首段 `drop-cap`、重要语句 `blockquote`、章节间风格专属分隔线、正确标题层级 h1>h2>h3；必须包含智能分页 CSS；CSS 内联或 CDN 引入 Tailwind/Google Fonts；输出完整自包含 HTML 文件
- Step 4 — 导出 PDF（按需）：调用 `scripts/html_to_pdf.py` 将 HTML 转为 PDF

**3. 验证与优化**

- 直接在浏览器打开生成的 HTML 文件预览效果
- 如需调整风格，重新触发 Step 2 选择其他风格或自定义参数
- 如需 PDF，在 Claude Code 中告知 Agent 调用转换脚本
- 可将自定义风格模板追加到 `references/styles.md` 供后续复用

### 📝 避坑指南

- ⚠️ **不要跳过风格选择步骤**：SKILL.md 里强制要求 `AskUserQuestion`，如果你修改 Skill 绕过这步，后期必然反复返工，因为排版审美决策无法被 AI 自动猜对。
- ⚠️ **自定义风格要写进 styles.md**：临时在对话里描述风格偏好，下次执行时 AI 不会记住，必须持久化到 `references/styles.md`。
- ⚠️ **HTML 必须自包含**：生成时确保所有 CSS 内联或通过 CDN 引入，否则离线查看或转 PDF 时样式会丢失。
- ⚠️ **Skills「非自建不可用」**：直接用作者的 Skill 可能不完全匹配你的审美和流程标准，建议 Fork 后按自己需求修改 `SKILL.md` 和 `styles.md`。
- ⚠️ **PDF 转换依赖 html_to_pdf.py**：确保 Python 环境和相关依赖已安装，否则 Step 4 会失败。
