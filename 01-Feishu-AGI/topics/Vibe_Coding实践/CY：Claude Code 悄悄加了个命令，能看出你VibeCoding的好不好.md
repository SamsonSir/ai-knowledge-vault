### 📋 核心分析

**战略价值**: `/insights` 命令通过分析本地全量会话记录，生成个性化 HTML 诊断报告，精准定位你的 VibeCoding 低效模式并给出可直接复制粘贴的改进方案。

**核心逻辑**:

- **触发方式极简**：在 Claude Code 中直接输入 `/insights` 即可启动分析，无需任何参数。
- **报告输出路径固定**：生成至 `~/.claude/usage-data/report.html`，自动在浏览器中打开，支持中文版（运行后告知 Claude 输出中文版本）。
- **分析完全本地化**：会话数据不上传，分析在本机进行，隐私友好。
- **增量缓存机制**：Facet 缓存至 `~/.claude/usage-data/facets/`，每次最多分析 50 个新会话，后续运行速度更快。
- **会话过滤规则**：自动排除子会话、少于 2 条消息、不足 1 分钟的对话；每个会话提取 12+ 项元数据（token 用量、工具调用次数、编程语言等）。
- **长文本分块处理**：超过 30,000 字符的会话按 25,000 字符分块摘要，保留关键文件名和错误信息，防止上下文丢失。
- **Facet 结构化评估**：对每个会话评估目标类型、满意度、成果、摩擦点等维度，形成结构化标签。
- **跨会话聚合分析**：统计项目领域分布、交互风格、成功模式、摩擦规律，识别系统性问题而非单次偶发。
- **多次运行结果有差异**：每次报告侧重点不同，推测原因是近期会话权重更高，建议定期运行以追踪改进趋势。
- **升级命令**：`npm update -g @anthropic-ai/claude-code`，确保使用最新版本才能使用 `/insights`。

### 🎯 关键洞察

`/insights` 的核心价值在于「反馈闭环」——VibeCoding 最大的问题是用户感知不到自己的低效模式（比如反复给 Claude 补充上下文、频繁纠错），而这个命令把隐性摩擦显性化。有用户描述其反馈质量「像训练有素的人类管理者给出的反馈」，关键在于它不是通用使用指南，而是基于你真实历史数据的定制分析。

### 📦 报告结构详表

| 报告模块 | 内容说明 | 实际价值 |
|---------|---------|---------|
| What's working | 分析你与 Claude 的独特交互风格，识别高影响力工作 | 强化已有优势，了解自己的高效模式 |

| Quick wins to try | 推荐可直接使用的 Claude Code 功能和工作流技巧，含 skills/agents/hooks 的具体可复制代码 | 直接可操作，无需二次转化 |
| Executive Summary | 综合所有分析的一页总览 | 快速掌握全局，适合定期回顾 |

### 🛠️ 操作流程

1. **升级到最新版**:
   ```bash
   npm update -g @anthropic-ai/claude-code
   ```

2. **运行分析**:
   在 Claude Code 会话中输入：
   ```
   /insights
   ```

3. **查看报告**:
   报告自动在浏览器打开，路径为：
   ```
   ~/.claude/usage-data/report.html
   ```

4. **切换中文**（如报告为英文）:
   直接告诉 Claude：「请输出一个中文版本的报告」

5. **定期复查**:
   建议每隔一段时间重新运行，利用增量缓存机制追踪改进效果，缓存位置：
   ```
   ~/.claude/usage-data/facets/
   ```

### 📝 避坑指南

- ⚠️ 版本不够新则无 `/insights` 命令，先执行 `npm update -g @anthropic-ai/claude-code` 再使用。
- ⚠️ 会话太短（< 2 条消息或 < 1 分钟）会被过滤，不计入分析，数据样本越多报告越准确。
- ⚠️ 多次运行报告内容不完全一致，近期会话权重更高，不要只跑一次就下结论。
- ⚠️ Quick wins 里的代码建议可直接复制粘贴，不要只看不用，这是该功能最高价值的部分。
