### 📋 核心分析

**战略价值**: 一套完整的「音乐驱动角色动画 + 像素风场景」Vibe Coding落地路径，覆盖从MVP验证、序列帧制作、视差背景、歌词同步到性能优化的全链路，提示词和踩坑经验可直接复用。

**核心逻辑**:

- **先验证核心交互再做美术**：整个项目从"音频驱动角色运动"这一最高风险点切入，先用静帧+最简代码跑通MVP，确认可行后才投入美术资源，避免返工。
- **音频驱动策略选型是关键决策**：初版用"频谱能量绝对值"驱动位移，导致角色高频抽搐；正确策略是"能量相对变化量（delta）"，只在音乐重拍时触发"冲出去"的位移脉冲，其余时间保持平稳飞行。经过5轮迭代才找到正确方向。
- **物理弹簧模拟让运动自然**：找到正确驱动策略后，运动仍然生硬，需叠加弹簧物理模型（惯性+阻力），再经4次参数微调，才达到自然的飞行手感。
- **序列帧动画速度必须写死时间而非绑定帧率**：AI默认写法是"每渲染N帧切换一张动作图"，在60fps设备上0.1s/帧，在120fps设备上变成0.05s/帧，节能模式下全站动画会极度卡顿。正确做法：写死`0.1s`换一张动作图，与设备帧率解耦。
- **序列帧制作要保证单图同一画布尺寸+同一对齐基线**：裁切多帧时若画布尺寸或基线不一致，帧动画会产生抖动和闪烁。
- **无缝循环背景的三层视差方案**：远景（深蓝夜空+楼影）、中景（城市建筑+圣诞装饰）、前景（雪地）分别生成，以不同速度向左滚动。提示词中直接写"水平无缝循环"即可让Nano Banano生成首尾相接图片。
- **中景多图衔接用"积木拼接法"而非AI直出**：AI无法直接生成比例细长的多图无缝拼接，解决方案是：先让AI把参考图拆成单体建筑零件，再生成更多同风格建筑，最后手动排布图1/2/3，在接缝处放置同一栋建筑保证无缝循环。
- **带时间戳歌词难获取，可用Manus辅助**：大部分歌曲的LRC时间戳歌词难以直接找到，可让Manus搜索或给它歌曲视频让其边听边记（精度有限，需人工校准）。
- **歌词漂浮动画一次成功的关键**：提供示意图+文字描述"像在水里自由漂浮"，AI可理解并实现；后续迭代细节包括：漂浮位移幅度、抖动频率、防重叠最小间距、渐隐渐显、最大宽度及折行规则。
- **部署后性能劣化需专项优化**：本地丝滑但部署后小女警主题加载需数十秒，AI排查出11个性能问题并按P0/P1/P2优先级列表，核心优化项包括将PNG全部转为WebP格式，共花3天完成P0+P1优化。
- **国内访问稳定性**：Cloudflare Pages默认域名（`.pages.dev`）在国内不稳定，绑定自定义域名（如腾讯云购买）可显著改善；商业项目国内用户必须ICP备案+境内服务器（腾讯云/阿里云）。

### 🎯 关键洞察

**音频驱动视觉的策略选型矩阵**（作者与AI总结）：

| 视觉表现 | 适合的驱动方式 | 说明 |
|---------|-------------|------|
| 频谱柱状图 | 能量绝对值（实时） | 连续伸缩，适合频谱可视化 |
| 角色"冲出去"位移 | 能量相对变化量（delta） | 只在能量突变时触发脉冲 |
| 平滑飞行/弹跳 | delta + 弹簧物理（惯性+阻力） | 自然手感，避免生硬 |
| 节拍闪光/粒子爆发 | 能量绝对值阈值触发 | 超过阈值才触发，非连续 |

**为什么5轮迭代才找对方向**：
1. 作者只知道"频谱"这一种音频驱动方式，初始指令就锁定了错误策略
2. AI在收到"效果不自然"的反馈后，持续在频谱逻辑内微调参数，而非质疑策略本身
3. 直到作者明确要求AI"全面深刻地复盘这个问题"，AI才跳出原有框架，识别出应换用delta策略

→ **操作建议**：遇到多轮微调仍无改善时，直接让AI"复盘问题根因"而非继续调参。

### 🛠️ 操作流程

**阶段1：MVP验证——音频驱动角色运动**

1. 准备角色静帧（从参考素材抠图）+ 背景图 + 歌曲文件，放入项目文件夹
2. 向AI描述需求：角色随音乐"冲出去"的位移效果，提供参考示意图
3. 初版跑通后观察效果，若角色高频抽搐 → 明确告知AI"要的是重拍时冲出去，不是连续伸缩"
4. 若多轮调参无改善 → 让AI"全面深刻地复盘这个问题"，引导其切换到delta能量策略
5. 确认delta策略后，叠加弹簧物理模型（惯性+阻力），再微调4次参数至自然

**阶段2：序列帧制作**

1. 用角色静帧作为参考图，输入以下提示词结构（以蓝色角色为例）：

```
这个8bit像素风格的角色正在摆出"飞行"的姿态，两个肉色的是手臂，两个白色的是脚。
衣服和眼睛有蓝色的部分，头发是灰色。请你帮我修改它手脚的动作以拓展成一套连续
飞行动作的序列帧，一共6张图片。
要求：
1、保持角色形象一致性，角色朝向一致性。
2、动作首尾相接不断循环，我给你的图作为动作1，最后的动作6也循环回动作1。
3、只稍微修改手脚的动作，不要增加任何新元素。
4、背景必须是纯白色。
```

2. 多次抽卡，常见问题：配色错位、多余肢体、比例不合理 → 调整提示词+人工微调
3. 裁切单帧时：统一画布尺寸 + 统一对齐基线（防抖动）
4. 生成圣诞配件（帽/围巾/蝴蝶结）叠加到角色上
5. 将静帧替换为序列帧，切换速度写死`0.1s`，不绑定设备帧率

**阶段3：三层视差背景制作**

1. 远景生成（提示词）：
```
像素艺术游戏背景纹理，2D横版游戏。深蓝色的夜空渐变，天空中有乌云漂浮。
远处是非常模糊的摩天大楼剪影，使用深海军蓝和紫色调来表现深度。
极简细节，强调氛围感，8位复古风格。水平无缝循环，比例是16:9
```

2. 前景生成（提示词）：
```
像素艺术游戏背景纹理，2D横版游戏。沿着画面底部水平排列的雪，白色，
带有冷蓝色的夜间阴影。极简细节，强调氛围感，8位复古风格，纯白背景。
水平无缝循环，比例是16:9。
```

3. 中景（积木拼接法）：
   - Step 1：让AI把参考图拆成单体建筑：
     ```
     把这张图里的建筑，拆分成单独的一个个建筑，彼此之间要有留白，背景纯白
     ```
   - Step 2：生成更多同风格建筑：
     ```
     参考这张图的设计风格，生成一系列类似设计风格和色调的2D，8bit复古风格的建筑素材，
     每个建筑之间留有足够的空隙以便我之后单独取用。请注意，不要添加任何深色描边，
     保持这种简约色块的风格。建筑也不要太复杂了，就类似参考图例这种简单的构成。
     ```
   - Step 3：手动在Figma排布图1/2/3，接缝处（图1末尾/图2开头、图2末尾/图3开头、图3末尾/图1开头）放置同一栋建筑

4. 用Nano Banano给建筑叠加积雪效果
5. 导出：中景和前景必须是透明底PNG，再统一转WebP
6. 让AI实现三层图片以不同速度向左滚动，描述需求后一次成功，微调速度+修复接缝处细线

**阶段4：歌词漂浮动画**

1. 准备带时间戳的LRC歌词（Manus搜索或听写，人工校准）
2. 提供示意图+描述"歌词像在水里自由漂浮"，让AI实现
3. 迭代细节参数：漂浮位移幅度、抖动频率、防重叠最小间距、渐隐渐显时长、最大宽度及折行规则

**阶段5：部署与性能优化**

1. 代码推送GitHub，关联Vercel/Cloudflare/Netlify，开启自动部署
2. 国内用户：绑定自定义域名（腾讯云购买）
3. 部署后测试加载速度，若慢 → 让AI排查性能问题并按P0/P1/P2优先级列表
4. 优先处理图片资源：PNG全部转WebP
5. 逐一处理P0、P1问题（作者共花3天）

### 📝 避坑指南

- ⚠️ **序列帧切换速度绑定帧率**：必须写死时间值（如`0.1s`），否则120fps设备动作过快，节能模式全站卡顿
- ⚠️ **序列帧裁切基线不统一**：每帧必须用同一画布尺寸+同一对齐基线，否则动画抖动
- ⚠️ **音频驱动用绝对值而非delta**：绝对值导致角色持续高频抽搐，必须换用能量变化量
- ⚠️ **AI在错误策略内反复微调**：多轮调参无效时，主动让AI"复盘问题根因"而非继续调参
- ⚠️ **Figma AI提示词不保留**：每次生成后提示词消失，必须提前另存一份
- ⚠️ **Figma多图只能生成在一张大图里**：需手动裁切，但一张大图内角色一致性优于多张独立图
- ⚠️ **Cloudflare默认域名国内不稳定**：必须绑定自定义域名才能保证国内访问
- ⚠️ **本地性能不代表部署后性能**：本地直接打开丝滑，部署后加载可能需数十秒，需专项优化
- ⚠️ **中景AI无法直出多图无缝拼接**：必须用积木拼接法手动排布，接缝处放同一建筑
