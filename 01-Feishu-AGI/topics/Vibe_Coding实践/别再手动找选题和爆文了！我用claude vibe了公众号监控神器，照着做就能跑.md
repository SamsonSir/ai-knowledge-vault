### 📋 核心分析

**战略价值**: 用 Vibe Coding（零代码 AI 驱动开发）搭建一套公众号文章自动监控系统，实现每日自动采集数据 → 热度算法筛选 → 生成选题报告 → 邮件推送，彻底替代手动找选题的低效工作流。

**核心逻辑**:

- **痛点驱动**: 关注账号太多导致信息过载，手动逐一查看分析效率极低，需要一个可定制化的自动降噪工具
- **开发工具链**: Claude Code（opus 4.5）负责开发，Codex（gpt 5.2）负责自动化测试，人工介入仅在测试验收和最终部署阶段
- **需求沟通方式**: 用 Claude Code 的 Plan 模式，以问答形式让 AI 追问需求细节，由 AI 生成 PRD 和 Spec 文档，而非自己写需求文档
- **启动 Prompt 模板**: `"我在运营一个ai领域的公众号，经常花大量时间去找选题，所以我准备做个工具去监测文章热度，帮助找选题，我需要你用问答形式和我一起梳理出这个工具的需求文档，让ai根据这个文档来开发"`
- **数据采集三件套**: wechat2rss（RSS订阅采集账号）+ 极致了API（获取阅读/转发/评论等互动数据）+ Supabase（云端数据库存储，降低重复调用API成本）
- **热度算法三维度**: 综合指数 + 互动指数 + 超常指数，核心是互动数而非绝对阅读量，解决不同账号基数差异问题（小账号互动好也能被识别为潜力选题）
- **超常指数的价值**: 专门识别某篇文章远超该账号正常数据的情况，捕捉黑马爆款选题
- **算法权重可配置**: 所有排行规则做成可调参数，后续可根据需求动态调整权重，无需重新开发
- **邮件推送工具**: Resend（https://resend.com/login），AI 主动建议引入，申请一个 key 即可实现每日定时推送选题报告 + 热榜到邮箱
- **Key 获取全程 AI 协助**: 整个开发过程中所有第三方工具的 key 申请和环境变量配置，均由 AI 一步步指导完成，无需技术背景

### 🎯 关键洞察

**为什么选私有部署 wechat2rss 而非免费服务**:
免费服务内置账号库是固定的，AI 领域账号覆盖不足。私有部署后可自由添加任意账号，灵活性完全不同。部署平台用的是作者文档推荐的 Sealos 云服务，AI 协助下不到 3 分钟完成部署。

**为什么引入 Supabase 做中间层存储**:
极致了 API 按次计费（每次约 0.04-0.06 元），如果每次展示都重新调用 API，成本会持续累积。Supabase 免费版足够用，数据短期存储后直接从数据库取数，大幅降低 API 调用频次和成本。

**Vibe Coding 的人机分工边界**:
人负责：① 开头用 Plan 模式沟通需求 ② 按 AI 提示申请各类 key ③ 人工验收测试 ④ 最终部署确认。AI 负责：生成 PRD/Spec → 分任务分步骤开发 → 自动化测试 → 部署指导。

### 🛠️ 操作流程

**1. 准备阶段 — 需求文档生成**

打开 Claude Code，切换到 **Plan 模式**，输入以下 Prompt：

> `我在运营一个ai领域的公众号，经常花大量时间去找选题，所以我准备做个工具去监测文章热度，帮助找选题，我需要你用问答形式和我一起梳理出这个工具的需求文档，让ai根据这个文档来开发`

按照 AI 的追问逐一回答需求细节（账号范围、筛选维度、推送频率、页面功能等），AI 自动生成 PRD 和 Spec 文档。

**2. 核心执行 — 分任务开发**

切换到**执行模式**，让 CC 分任务分步骤开发。过程中 AI 会提示需要哪些第三方 key，按提示依次完成：

- 私有部署 wechat2rss（Sealos 平台，AI 全程指导，约 3 分钟）
- 购买 wechat2rss 服务 key，后台添加要监控的账号
- 注册极致了 API，获取 API key
- 注册 Supabase，新建项目，将项目 URL 和 2 个 key 发给 AI，让 AI 生成建表 SQL，自己复制粘贴执行
- 注册 Resend，获取 API key 用于邮件推送

所有 key 按 AI 指示配置到环境变量中。

**3. 测试阶段**

在项目目录文件夹下打开 **Codex**，让 gpt 自动测试。AI 测试完成后，人工介入进行功能验收测试。

**4. 部署上线**

测试通过后，在 AI 指导下完成部署上线。

### 💡 具体案例/数据

- **热度算法三维排行**（均可调整权重）：
  - **综合指数**：综合多维度数据的整体热度排名
  - **互动指数**：以转发、评论等互动行为为核心权重，解决大账号阅读基数碾压小账号的问题
  - **超常指数**：识别某篇文章相对该账号历史均值的超常表现，捕捉潜力爆款

- **页面四大功能模块**：
  1. **排行榜** — 结合算法做文章主题归类 + 热度排行
  2. **标题分析** — 从数据侧对标题进行归类分析
  3. **选题报告** — 自动生成，当前为数据驱动分析，后续计划接入 AI 提升报告质量
  4. **邮件推送** — 每日自动将选题报告 + 热榜发送到指定邮箱

- **极致了 API 来源**: 由博主 @饼干哥哥 的文章中发现，是目前能获取公众号文章互动数据的可用 API 方案

### 📝 避坑指南

- ⚠️ **wechat2rss 不要选免费服务**：免费版账号库固定，垂直领域（如 AI）账号覆盖严重不足，必须私有部署才能自定义添加账号
- ⚠️ **Supabase 建表不要自己手写 SQL**：直接把项目 URL 和 key 发给 AI，让 AI 生成建表语句，自己只需复制粘贴执行，避免字段设计遗漏
- ⚠️ **极致了 API 需配合数据库缓存使用**：每次调用有成本，不做缓存会导致费用持续累积，务必先存 Supabase 再取数
- ⚠️ **Plan 模式和执行模式要分开**：Claude Code 先在 Plan 模式下跑完需求文档，再切换执行模式开发，混用会导致 AI 边规划边执行，质量下降
- ⚠️ **私有部署不要自己硬上**：直接把 wechat2rss 作者文档链接发给 AI，让 AI 读文档后一步步指导操作，无技术背景也能 3 分钟完成
