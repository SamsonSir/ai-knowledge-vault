### 📋 核心分析

**战略价值**: Agent 并发时代，用 zoxide 做终端快速跳转 + 多文件夹复制策略，把"切换项目"的摩擦降到 1 秒，让单人同时跑 5 个 Claude Code 会话成为可能。

**核心逻辑**:

- **并发能力决定产出上限**：Agent 时代不是你写多快，而是你能同时跑多少个 Agent 会话。人的角色从"写代码"变成"审核结果"。
- **Claude Code 作者 Boris 的方法**：用 `git worktree` 同时开 3-5 个并行会话，每个 worktree 是独立工作目录，互不干扰。
- **更简单的平替方案（宝玉翻译版）**：直接复制多个项目文件夹，命名为 `project-1`、`project-2`、`project-3`……哪个空闲用哪个，做完合并推回主文件夹。无需学 git worktree，复制粘贴即可。
- **范式转移：终端为核心，文件为辅助**：传统开发（VS Code 时代）以文件为核心，人频繁看代码；Agent 时代以终端为核心，人只审结果，简单任务不需要 IDE。
- **旧工作流的摩擦点**：用 Cursor 类产品要先打开文件夹 → 逐级找 → 选中 → 打开，同时开 5-6 个项目极其繁琐。
- **zoxide 解决的问题**：记一个关键词，终端 1 秒直达对应文件夹，彻底消除"找文件夹"的摩擦。
- **wave 工具的使用**：作者实际在用名为 wave 的编程工具，支持 N×N 多窗口并排，配合 zoxide 做多项目并发。
- **映射表机制**：zoxide-skills 安装后在 Skills 文件夹下生成一个关键词→路径的对照表，`z 关键词` 即可跳转，维护成本极低。
- **核心心智模型转变**：以前 = 打开桌面 → 一个个文件夹找 → 点进去再打开；现在 = 记个关键词 → 一步到位。

### 🛠️ 操作流程

**第1步：下载 zoxide-skills**

把以下提示词发给 Claude，让它自动新建文件夹并下载 GitHub 内容：

```
给用户新建一个文件夹，下载GitHub内容
https://github.com/Ceeon/zoxide-skills
```

**第2步：安装插件**

在 Claude Code 中执行：

```
/zoxide-install
```

**第3步：管理目录（添加路径映射）**

```
/zoxide-manage
```

执行后，在 Skills 文件夹下会生成一个关键词 → 路径的对照表（映射表），后续所有跳转依赖这张表。

**第4步：日常使用**

```bash
z 关键词    # 跳转到对应目录
```

示例：

```bash
z 写作      # 直接跳转到"写作"项目文件夹
```

跳转后直接输入 Claude Code 启动指令，开始工作。

### 📦 配置/工具详表

| 模块/功能 | 关键设置/代码 | 预期效果 | 注意事项/坑 |
|---|---|---|---|
| zoxide-skills 下载 | 发 Prompt 给 Claude，指向 `https://github.com/Ceeon/zoxide-skills` | 自动建文件夹并拉取 | 需要 Claude Code 环境 |
| 安装 | `/zoxide-install` | 插件就绪 | 安装后才能用 `/zoxide-manage` |
| 目录管理 | `/zoxide-manage` | 在 Skills 文件夹生成关键词→路径映射表 | 每次新增项目都要更新映射表 |
| 跳转 | `z 关键词` | 1 秒进入目标文件夹 | 关键词需与映射表一致 |
| 多项目并发（简单版） | 复制文件夹为 `project-1/2/3` | 多个 Claude Code 会话并行 | 做完需手动合并推回主分支 |
| 多项目并发（进阶版） | `git worktree`（Boris 原版方法） | 3-5 个会话并行，共享 git 历史 | 学习成本较高 |
| 多窗口工具 | wave（编程工具，支持 N×N 窗口） | 多项目并排可视化管理 | 作者实测在用 |

### 📝 避坑指南

- ⚠️ `git worktree` 对新手不友好，直接用"复制多个文件夹"的平替方案，功能等价，零学习成本。
- ⚠️ 映射表需要手动维护，新增项目后记得跑 `/zoxide-manage` 更新，否则 `z 关键词` 找不到路径。
- ⚠️ 并发会话数量受限于"脑子跑不过 AI"的审核瓶颈，不是工具瓶颈——同时开太多会话，人工审核反而成为新的卡点。
