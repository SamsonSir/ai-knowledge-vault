### 📋 核心分析

**战略价值**: 一条全自动流水线（播客链接 → 下载 → 转写 → 搜索补充 → 提炼坐标体系 → 生成4K信息图 → 推送飞书群），全程约3分钟，零人工干预，解决播客"听完即忘"问题。

**核心逻辑**:

- 播客三大痛点：时长40-60分钟（时间成本高）、音频线性无法快速定位（信息提取难）、听完留存率不足10%（知识留存差）。现有工具（Podwise、AI好记、通义听悟）只做到"转文字"，未做到"转知识图"。
- 整条流水线共8步：接收链接 → 下载音频 → 语音转写 → 保存飞书文档 → Exa搜索补充 → AI提炼坐标体系 → 生成4K信息图 → 发送飞书群。
- 音频下载核心：小宇宙音频URL藏在页面源码 `NEXT_DATA` JSON中，需用iPhone UA伪装请求，正则提取 `props.pageProps.episode.enclosure.url`，保存为 `/data/podcast.m4a`。
- 转写前必做FFmpeg预处理：`ffmpeg -i podcast.m4a -ac 1 -ar 16000 -b:a 64k podcast.mp3 -y`，三参数缺一不可（单声道、16kHz采样率、64k比特率），跳过此步识别率明显下降。
- 转写双方案：SiliconFlow（免费，模型 `FunAudioLLM/SenseVoiceSmall`，40分钟播客实测可用）vs 阿里云百炼FunASR（精度更高，支持说话人分离，模型 `fun-asr-flash-8k-realtime-2026-01-28`）。两者输出格式一致，切换成本极低。
- 知识图坐标体系固定7模块：A-01核心概念、B-05技术原理、C-09应用场景、D-12优势特点、E-15局限挑战、F-17发展趋势、G-20实践建议。一期播客可生成最多6张图，每张对应独立主题，内容不重复。
- 生图风格强制规范：深蓝背景 `#0a1628`，荧光粉 `#ff6b9d`，荧光绿 `#00ff88`，荧光黄 `#ffeb3b`，赛博朋克+实验室美学。配色必须用十六进制色值，"蓝色背景"和 `#0a1628` 效果天差地别。Prompt不能简化，少一段就掉品质，每个要点控制在15字以内。
- 生图API优先级：Google Gemini（免费300刀额度）→ 302.ai（付费备选）。模型：`gemini-3-pro-image-preview`，responseModalities需同时包含 `["image", "text"]`。
- 飞书推送两步走：先POST上传图片获取 `image_key`，再POST发送消息。图片上传必须用Python requests的multipart，curl方式会报 `234001` 错误；文档写入频率上限每秒3次，必须加 `time.sleep(0.35)`。
- Exa语义搜索补充播客中提到的论文/概念/人物背景，type用 `neural`（语义搜索），比关键词匹配更智能，每条结果限制 `maxCharacters: 2000`。

### 🎯 关键洞察

**为什么要建坐标体系而不是直接生图？**
AI生图模型在没有结构约束时会自由发挥布局，导致信息密度不均、模块缺失。固定7个坐标编号（A-01到G-20）相当于给AI一张"地图"，强制它在每个坐标位置填充对应内容，保证每张4K图信息完整且可复现。

**为什么Prompt不能简化？**
生图模型（尤其Gemini）对Prompt长度和细节敏感。视觉风格描述越模糊，模型自由度越高，输出越不稳定。具体到色值、布局网格、字体大小，是保证"工业级复刻"的关键。作者特别标注这是"坑一"。

**为什么选SiliconFlow而不是直接上百炼？**
SiliconFlow完全免费，跑通流程零成本。百炼精度更高但按量计费。建议策略：先用SiliconFlow验证整条流水线，确认效果后再按需切换百炼，两者接口格式一致，改一行代码即可。

### 🛠️ 操作流程

1. **准备阶段**
   - 安装FFmpeg：`yum install -y ffmpeg` / `apt install ffmpeg` / `brew install ffmpeg`
   - 安装Python依赖：`pip install requests dashscope`
   - 可选安装Playwright（网页抓取）：`pip install playwright && playwright install chromium`
   - 配置 `.env` 文件，填入：SiliconFlow API Key、阿里云百炼API Key、Exa API Key、飞书App ID、飞书App Secret、Gemini API Key

2. **核心执行**
   - Step 1：接收播客链接，确认生成图数量（最多6张），每张对应独立主题
   - Step 2：调用 `download_xiaoyuzhou(url)` 下载音频到 `/data/podcast.m4a`
   - Step 3：FFmpeg预处理 → `ffmpeg -i /data/podcast.m4a -ac 1 -ar 16000 -b:a 64k /data/podcast.mp3 -y`
   - Step 4：调用SiliconFlow或百炼ASR，获取完整文字稿
   - Step 5：调用飞书API写入知识库（每次写入间隔0.35秒）
   - Step 6：调用Exa搜索补充播客中提到的论文/概念/人物（num_results=5，type=neural）
   - Step 7：AI大模型按7模块坐标体系（A-01到G-20）结构化提炼内容
   - Step 8：构建完整Prompt（含色值、布局、坐标体系），调用Gemini生图API，保存PNG到 `/data/infographic.png`
   - Step 9：上传图片获取image_key，推送到飞书群

3. **验证与优化**
   - 检查转写文字稿完整性，若识别率低先检查FFmpeg预处理参数
   - 检查生图质量：Prompt是否完整、色值是否用十六进制、每要点是否≤15字
   - 若生图效果差，换302.ai备用API重试
   - 若飞书推送失败，检查是否用了curl（改用Python requests）

### 📝 避坑指南

- ⚠️ 音频必须先FFmpeg转码（`-ac 1 -ar 16000 -b:a 64k`），跳过此步ASR识别率明显下降
- ⚠️ 飞书图片上传必须用Python requests multipart，curl方式报 `234001` 错误
- ⚠️ 飞书文档写入每秒最多3次，必须加 `time.sleep(0.35)`，否则被限流
- ⚠️ 飞书批量删除block用 `batch_delete`（DELETE方法带JSON body）
- ⚠️ 百炼异步转写需传文件URL，本地文件必须用流式接口（Recognition类）
- ⚠️ 生图Prompt不能简化，少一段就掉品质，这是最高频踩坑点
- ⚠️ 配色必须指定十六进制色值，"蓝色背景"和 `#0a1628` 效果天差地别
- ⚠️ 每个知识点控制在15字以内，超出会导致AI排版混乱
- ⚠️ 4K分辨率要在Prompt里明确要求，不写默认输出低分辨率
