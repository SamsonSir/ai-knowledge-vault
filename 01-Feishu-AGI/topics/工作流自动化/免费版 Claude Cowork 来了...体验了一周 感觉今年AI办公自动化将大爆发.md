### 📋 核心分析

**战略价值**: 阶跃星辰「小跃」是一款比 Claude Cowork 早几个月上线（2024年9月）、功能对标 Claude Code 的免费 Mac 桌面 AI 自动化助手，支持本地文件操作、MCP 工具调用、Skill 复用，可直接跑通无人驾驶式工作流。

**核心逻辑**:

- **上线时间早于 Claude Cowork 数月**：小跃 2024年9月上线，Claude Cowork 之后才发布，但国内知名度远低于前者，属于被低估的工具。
- **定位是「Claude Code 小白版」而非纯聊天 AI**：通过调用 Mac 系统工具和终端，可直接读写本地文件、执行 shell 命令，能力边界接近 Claude Code，而非普通对话 AI。
- **五大核心能力**：① 读取/操作本地文件；② 联网 + 自动执行复杂任务（如抓取小红书内容）；③ 定时任务（每天自动重复劳动）；④ 妙计系统（Prompt 模板复用，类似 Claude Skill）；⑤ 多任务并行执行，互不干扰。
- **全局记忆 = 可回溯的个人工作记忆层**：自动记录电脑活动轨迹（包括聊天内容、浏览网页），生成每日复盘日报。官方声明所有记忆数据存储在本地设备，不上传云端。可随时开启/关闭。
- **同步应用「所见即所问」**：悬浮球常驻桌面任意位置，点击后自动识别当前窗口（文件/网页/应用）作为上下文，无需手动复制粘贴。
- **妙计广场 = 公共 Skill 市场**：可保存、一键调用、分享 Prompt 模板。目前以飞书文档形式运营：https://wvixbzgc0u7.feishu.cn/docx/FaYvdiwRuoijGDxaNkicDjlpnjc
- **支持直接导入 Claude Skills**：@宝玉 在 GitHub 发布的 skill 包（https://github.com/JimLiu/baoyu-skills）可直接导入阶跃使用，且支持多 skill 同时调用，实现模块化解耦工作流。
- **MCP 工具生态已接入 16 款**：包括 Excel、QQ邮箱、飞书、钉钉、Notion、高德等，用户可自行扩展更多 MCP 工具，实现跨应用操控。
- **任务执行为「无人驾驶」模式**：任务启动后可最小化为悬浮窗，完成后自动弹窗提醒，后台不干扰当前工作，支持同时运行多个任务。
- **当前短板**：底层模型为阶跃自研，终端编码能力和复杂任务处理与 Claude 有差距，暂不支持接入第三方模型（作者希望后续开放）。

### 🎯 关键洞察

**Skill 解耦设计是核心方法论**：不要把整个工作流做成一个大 Skill，而是拆成独立模块，每个模块一个 Skill。工作流变化时只需修改对应模块，其余不动。这是软件工程「解耦」原则在 AI 工作流中的直接应用。

**实测案例：DAN KOE 爆款文章 → 漫画全自动生成**
- 输入：DAN KOE《How to fix your entire life in 1 hour》（X 平台阅读量破亿）
- 调用链：`/漫画生成 skill` → 自动识别需要生图 → 自动调用 `Gemini 画图 skill` → 调用 Gemini API 生成图片 → 自动下载到本地
- 全程无人工干预，输出多张完整漫画图片

**关键发现**：小跃在执行漫画生成 skill 时，自动发现生图环节需要另一个 skill，主动查找并调用了 Gemini 生图 skill，体现了一定的自主任务规划能力。

### 🛠️ 操作流程：导入 Claude Skill 到阶跃

1. **下载 Skill 包**
   - 访问 https://github.com/JimLiu/baoyu-skills
   - 点击 `Code` 按钮 → 最下方 `Download ZIP` → 解压

2. **查看 skill.md**
   - 每个 skill 文件夹内有 `skill.md`
   - 记录文件顶部的 `name` 和 `description` 字段

3. **在阶跃中创建妙计**
   - 进入设置 → 新建妙计
   - 妙计名称：自定义（如「漫画生成」）
   - Prompt 指令填写格式：
     ```
     ---
     name: baoyu-comic
     description: 知识漫画创作器，支持多种风格（Logicomix/Ligne Claire、欧姆社漫画指南）。创建具有详细分镜布局和连续图像生成的原创教育漫画。当用户要求创建"知识漫画"、"教育漫画"、"传记漫画"、"教程漫画"或"Logicomix风格漫画"时使用。
     ---
     当需要阅读Skill.md时，请查阅Skill.md全部内容，并基于Skill.md的指引查看其余文件
     下面是我的指令：
     ```
   - 点击「添加 skill 文件目录」，选择对应 skill 文件夹路径

4. **调用验证**
   - 悬浮球展开 → 输入框输入 `/` → 看到刚注册的 skill 名称 → 选择调用
   - 输入任务指令，观察是否自动执行并输出文件

5. **多 Skill 联动验证**
   - 同时注册「漫画生成」和「Gemini 画图」两个 skill
   - 执行漫画任务，观察是否自动跨 skill 调用生图流程

### 💡 具体案例/数据

**文件整理案例**
- 指令：「把桌面上所有发票（图片或PDF）按开票日期重命名，并按月份分文件夹整理」
- 执行链：自动扫描 → 分类 → 整理 → 打包成新文件夹

**表格比价案例**
- 指令：「根据最新汇率，把1688商品的美元价和亚马逊同款比价，生成差价表」
- 执行链：读取 Excel → 联网获取汇率 → 计算差价 → 输出新表格

**播客转录案例**
- 指令：`/播客音频下载` + 播客链接
- 执行链：下载音频 → 调用终端安装语音识别工具 → 转录为文字稿 → 整理输出

**漫画生成全流程**
- 输入文章：DAN KOE《How to fix your entire life in 1 hour》
- 自动完成：理解文章 → 制作脚本 → 写生图 Prompt → 调用 Gemini API → 下载图片到本地
- 输出：9 张完整漫画图片，全程无人工干预

### 📝 避坑指南

- ⚠️ **全局记忆隐私风险**：会记录你与他人的聊天内容，官方称存本地，但使用前需自行评估风险，可随时在设置中关闭。
- ⚠️ **妙计广场未整合进客户端**：目前只是飞书文档页面，添加他人 skill 需手动操作，不如 Claude Skill 市场流畅。
- ⚠️ **底层模型能力有上限**：终端编码和复杂任务处理弱于 Claude，不适合高复杂度编程任务，适合内容处理、文件管理、信息整理类工作流。
- ⚠️ **Skill Prompt 必须加引导语**：导入外部 skill 时，prompt 中必须包含「当需要阅读Skill.md时，请查阅Skill.md全部内容」，否则 skill 文件内容可能不被正确读取。
- ⚠️ **Skill 设计原则**：不要做大而全的单一 skill，要拆成独立模块，否则工作流变更时维护成本极高。
