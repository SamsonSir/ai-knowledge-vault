### 📋 核心分析

**战略价值**: 用 Chatbot → n8n 自动化工作流 → Claude Code Skill 三级递进方案，将亚马逊竞品分析从30分钟/个压缩到批量100个仅需几分钟，决策周期从1-2周压缩到1小时内。

**核心逻辑**:

- **效率基准线**：手工分析单个亚马逊竞品至少30分钟，新类目需看前20-50名Best Seller，传统方式耗时1-2周才能形成类目认知
- **三阶段演进路径**：Chatbot（1分钟/个，适合少量）→ n8n工作流（100个/几分钟，适合批量）→ Claude Code Skill（灵活可进化，适合长期复用）
- **Gemini优先选型理由**：多模态能力处理文本+图像+视频处于断档级领先，可直接发送亚马逊链接进行分析，无需额外处理
- **Chatbot方案的致命缺陷**：上下文丢失问题严重——作者实测Gemini在大量文件输入后出现幻觉，翻出10天前的listing做分析；且每次分析都要重新向AI解释背景
- **n8n爬取方案选型**：放弃网页截图（识别准确度低、抓不到缩略图、无法覆盖评论），改用 Olostep API（每月1000次免费额度），抓取内容转为Markdown，覆盖全量文案+图片链接+最新100条评论
- **差评是最高价值信源**：1-3星差评直接反映用户真实感受，外国用户会写详细使用过程+问题描述+优化建议，是找竞品弱点和弯道超车机会的核心数据
- **n8n工作流的局限**：固定流程灵活性差，换平台或改爬取参数需重构整个工作流配置；搭建调试过程复杂，改一个参数可能触发多个节点BUG
- **Skill vs n8n的核心差异**：Skill用自然语言修改功能逻辑，可处理复杂数据分析，支持随时扩展平台（速卖通/Shopee/Temu等），可与其他Skill组合形成依赖链，且会根据用户反馈持续进化
- **Skill-First思维框架**：判断标准是"以后还会不会做？流程是否固定？其他人是否会遇到类似问题？"——满足则抽象成Skill，沉淀可复用的最佳实践
- **从0到1构建Skill的耗时**：把n8n的JSON代码+口喷需求文档喂给Claude Code，生成Skill构建方案，整个过程最多15分钟

### 🎯 关键洞察

**为什么Skill比n8n更优越的底层逻辑**：

n8n是"死板的管道"——数据流向固定，节点配置硬编码，每次需求变化都要人工重构。Skill是"有记忆的专家"——它读工作文档、吸收用户反馈、自我迭代，且能生成subagent并行处理多任务。

**差评分析的价值密度远高于好评**：好评告诉你用户为什么买，差评告诉你用户为什么退货/不复购。后者直接指向产品改进方向和竞品的市场空白。100条最新评论中的1-3星内容，是整个分析流程中含金量最高的信源。

**Google Sheets作为轻量RAG的用法**：分析结果写回Sheets后，可直接用Gemini in Google Sheets对100个ASIN的缺陷做二次批量处理，生成类目级痛点报告；或将整张表丢进NotebookLM，作为新品上架时写标题/定价/图文方案的辅助参考库。

### 🛠️ 操作流程

#### 方案一：Chatbot快速分析（适合1-5个竞品）

1. **准备阶段**: 打开Gemini，准备目标ASIN的亚马逊链接；可选用卖家精灵插件下载主图、A+图片、完整评论内容、详情页截图
2. **核心执行**: 将链接（或文件）+ 以下V4.0提示词一起发给Gemini，在`[输入数据]`处填入链接/文案/图片，在`[我的产品核心优势]`处填入自身优势以获取针对性反击建议
3. **验证与优化**: 检查输出的4部分报告是否完整（文案逻辑/视觉设计/评论分析/市场动态），重点核查差评穿透分析中的3条核心痛点和3条改进建议

#### 方案二：n8n自动化工作流（适合10-100个竞品批量分析）

1. **准备阶段**: 注册Olostep账号获取API Key（每月1000次免费）；准备Google Sheets，A列填入待分析ASIN列表（每行一个）；安装n8n（自托管或云端），导入从公众号获取的JSON代码
2. **核心执行**:
   - Step 1: n8n从Google Sheets读取ASIN列表
   - Step 2: 调用Olostep API爬取每个ASIN的页面，获取全量文案+图片链接+最新100条评论，转为Markdown
   - Step 3: 将Markdown内容喂给Gemini（或其他模型），使用V4.0提示词生成分析报告
   - Step 4: 结构化提取节点拆分标题/价格/差评总结/关键词等字段，写回Google Sheets对应列
3. **验证与优化**: 在Google Sheets中用Gemini in Google Sheets对所有ASIN的差评列做二次汇总，生成类目级痛点报告；或将整张表导入NotebookLM作为新品上架的参考RAG

#### 方案三：Claude Code Skill（适合长期复用、多平台扩展）

1. **准备阶段**: 安装Claude Code；从 `https://github.com/buluslan/ecommerce-competitor-analyzer` clone仓库；配置Olostep API Key和Google Sheets凭证
2. **核心执行**: 在Claude Code中直接输入自然语言指令，例如：`帮我针对亚马逊平台的ASIN B08XXX, B09XXX, B0AXXX进行竞品分析`；系统自动执行：detect-platform.js识别平台→scrape-amazon.js爬取数据→batch-processor.js并行处理（错误隔离，单个失败不影响整批）→analysis-prompt模板生成报告→输出Markdown+写入Google Sheets
3. **验证与优化**: 检查Markdown报告的4大分析模块；如需修改分析维度，直接编辑`prompts/analysis-prompt-*.md`；如需新增平台（速卖通/Shopee/Temu），在`platforms.yaml`添加配置并创建对应爬虫脚本；将此Skill设为listing生成Skill的前置依赖，形成分析→生成的完整链路

### 💡 具体案例/数据

- 手工分析单个竞品：30分钟起步
- 新类目调研（看前20-50名Best Seller）：传统方式1-2周
- AI工作流方案：跑Deepsearch报告+竞品分析工作流，整个分析决策过程约1小时
- n8n方案：100个ASIN，几分钟内完成全部分析
- Skill构建耗时：从0到1最多15分钟（vs n8n搭建需要反复调试，改一个参数触发多个节点BUG）
- Olostep API免费额度：每月1000次抓取
- 每次评论抓取量：最新100条

### 📝 V4.0 提示词完整模板

```
# Role / 身份角色
你是一位拥有 10 年经验的"亚马逊顶级运营总监"和"品牌战略官"。你不仅精通 A9和rufus算法，更擅长解析品牌背后的营销心理学与视觉工业设计逻辑。你的任务是透过 Listing 表面现象，还原对手的战略布局、设计方法论以及运营套路。

# Goal / 工作目标
对 [提供的ASIN/商品链接/文本/图片] 进行深度扫描，输出一份能够指导产品迭代和营销升级的结构化报告。

# Output / 强制输出结构

第一部分：文案构建逻辑与词频分析 (The Brain)
- 构建逻辑与方法论：拆解标题、五点描述、详情描述的文本构建策略。基于"痛点触发"、"场景驱动"还是"参数压制"？使用了什么叙事模板？
- 词频情报：提取 Listing 全文中的 Top 10 核心关键词，分析这些词如何在文案中进行权重分配与埋点。

第二部分：视觉资产设计思路 (The Face)
- 设计方法论：分析主图与 A+ 图片的整体设计风格与视觉定位（如：极简工业风、居家生活风等）。
- 视觉动线拆解：逐一说明不同图片（功能图、场景图、对比图等）分别表达了什么核心内容，视觉上有哪些抓人眼球的特点。
- 设计逻辑：分析构图、色彩心理学应用以及字体选型，识别视觉上的差异化"钩子"。

第三部分：评论定量与定性分析 (The Voice)
- 量化数据概览：明确分析的样本量及当前总评分；统计好评（4-5星）与差评（1-3星）的数量及百分比。
- 定性穿透分析：
  - 优势聚类：用户评论中反复提到的优点及其触发场景。
  - 差评穿透：差评主要体现的核心问题（区分产品缺陷、描述不符或体验感差）。
- 核心总结 (Top 3)：
  - 3 条核心优势（用户为何买他）
  - 3 条核心痛点（用户为何退货/差评）
  - 3 条改进建议（我该如何做得更好）

第四部分：市场动态与盲区扫描 (The Pulse)
- 市场表现：分析价格波动规律、评分稳定性以及在类目中的排名变动趋势。
- QA 价值挖掘：从问答区提取用户下单前的"最后一道心理防线"。
- 盲区扫描：识别任何尚未察觉但具有威胁或机会的内容（如：特定背书、捆绑策略等）。

# Analysis Content / 输入数据
[在此处粘贴竞品链接 / 复制粘贴文案文本 / 上传图片]
我的产品核心优势是：[填入你的产品优势，以便 AI 生成更有针对性的反击建议]
```

### 📝 避坑指南

- ⚠️ Chatbot上下文幻觉：长时间向Gemini塞大量文件后，会翻出历史数据做分析，竞品数量超过5个建议直接上n8n
- ⚠️ 网页截图爬取方案不可用：识别准确度低，抓不到商品缩略图，且无法覆盖评论内容，必须用API方案
- ⚠️ n8n参数联动BUG：修改单个节点参数可能触发多个下游节点报错，调试前先备份整个工作流JSON
- ⚠️ Olostep免费额度限制：每月1000次，100个ASIN一次跑完消耗100次，注意规划用量
- ⚠️ Skill新增平台需双文件操作：仅修改`platforms.yaml`不够，必须同时创建对应平台的爬虫脚本，否则抓取会失败
- ⚠️ 分析结果二次处理别跳过：单个ASIN报告价值有限，用Gemini in Google Sheets对100个ASIN的差评做汇总，才能看清整个类目最痛的问题
