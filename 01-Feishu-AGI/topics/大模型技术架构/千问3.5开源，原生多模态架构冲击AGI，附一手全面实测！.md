### 📋 核心分析

**战略价值**: Qwen3.5-Plus 以 MoE 架构实现原生多模态能力，在参数更少、推理更快的前提下超越 Qwen3-Max-Thinking，多模态理解局部场景已追平 Gemini 3.0 Pro。

**核心逻辑**:

- **除夕开源节点**：阿里 Qwen 团队在 2026 年除夕（2月16日）发布 Qwen3.5-Plus 开源版，HuggingFace 地址：https://huggingface.co/Qwen/Qwen3.5-397B-A17B
- **MoE 架构规格**：总参数 397B，激活参数仅 17B；专家配置为 1 个共享专家 + 512 个路由专家，每次激活 10 个路由专家
- **Attention 结构**：采用 Linear Attention Gated DeltaNet + Gated Attention 混合机制，与此前 Qwen3-next 的 Attention 结构基本一致，同时提供 Dense 和 MoE 两个版本
- **原生多模态设计**：使用 M-RoPE（多模态旋转位置编码）+ 3D 位置编码，将时空信息原生融入 LLM 的位置嵌入，而非外挂视觉模块
- **上下文与词表扩展**：上下文窗口扩展至 1M tokens；词表从 150K 扩展至 250K，支持语言/方言扩展至 201 种
- **性能定位**：榜单指标比 Qwen3-Max-Thinking 更强，但参数更少、速度更快；API 价格每百万 Token 低至 0.8 元，仅为 Gemini 3 Pro 的 1/18
- **多模态能力跃升**：相较 Qwen3-VL，目标识别与空间排序有显著提升，网页复刻、细粒度图像识别效果明显更好，部分场景已堪比 Gemini 3.0 Pro
- **已知短板**：空间推理（迷宫路径、六面体展开）仍不可靠；时钟读取依然出错；泛化能力不足（同类视觉幻觉题换 case 后易出错）；极端 OCR（复杂手写/变形字体）仍属长尾问题；文本推理稳定性有待提高
- **后续规划**：按惯例后续会推出若干 Dense 小模型，全尺寸开源

### 🎯 关键洞察

**原生多模态 vs 外挂视觉模块的本质差异**：传统 VLM 是在 LLM 基础上外接视觉编码器（如 CLIP），图文信息在特征层面拼接，位置感知割裂。Qwen3.5 用 M-RoPE + 3D 位置编码将视觉 token 的空间坐标直接编码进位置嵌入，模型在 attention 计算时天然感知图像的行列关系，这是其细粒度目标定位（行列序号返回）和网页复刻精度提升的根本原因。

**MoE 激活效率**：512 路由专家中每次只激活 10 个，激活参数仅 17B，推理成本接近一个中型 Dense 模型，但总参数容量达 397B，知识容量远超同等推理成本的 Dense 模型。这是"更强但更便宜"的结构性原因。

### 🛠️ 操作流程

1. **获取模型**
   - HuggingFace：https://huggingface.co/Qwen/Qwen3.5-397B-A17B
   - 或通过 API 调用（每百万 Token 0.8 元）

2. **多模态目标识别（推荐 Prompt 格式）**
   ```
   找到图片中[目标描述]，并返回行列序号，比如：几行几列。
   ```
   - 目标描述越具体越好（如"奔跑的人"、"磨损的眼镜"）
   - 语义模糊目标（如"不开心的冰块"）准确率下降

3. **网页/UI 复刻**
   ```
   根据图片内容，1:1复刻出网页
   ```
   或
   ```
   复刻一下[内容]，要保持一模一样，画一个web页面出来
   ```

4. **图像事件排序**
   ```
   根据图中显示的多个场景，将最有可能发生的事件按顺序排列。
   ```
   实测两组均回答正确。

5. **OCR 识别**
   ```
   识别图片中的文字内容
   ```
   或
   ```
   识别图中所有字母
   ```

6. **避开已知坑点**：空间路径规划、时钟读取、复杂3D变换题不要依赖此模型，当前版本不可靠。

### 📝 避坑指南

- ⚠️ **空间路径规划不可用**：迷宫"上下左右"步骤类任务，Qwen3.5 和 Gemini 均失败，当前所有 VLM 都不可靠，勿用于此类场景
- ⚠️ **时钟读取持续出错**：从 Qwen3-VL 到 Qwen3.5 均未解决，不要依赖模型读取模拟时钟
- ⚠️ **视觉泛化能力不足**：同类视觉幻觉题换 case 后易出错，推测与激活参数量（17B）有限相关，生产环境需多 case 验证
- ⚠️ **极端 OCR 仍是长尾**：复杂变形、艺术字体识别失败，不要用于高精度 OCR 场景
- ⚠️ **3D 空间变换理解弱**：六面体展开图选择题出错，涉及复杂空间变换的推理不可靠
- ⚠️ **文本推理稳定性**：部分推理题（如老鹰题）表现不稳定，重要场景需多次采样验证
- ⚠️ **Dense 版本尚未发布**：当前只有 MoE 版（397B-A17B），小参数 Dense 版本等后续发布
