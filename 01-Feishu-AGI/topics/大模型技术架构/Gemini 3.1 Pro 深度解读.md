### 📋 核心分析

**战略价值**: Gemini 3.1 Pro 以 ARC-AGI-2 推理分数翻倍、价格不变的组合拳，在推理/编程/Agent 三个维度重新定义旗舰模型的性价比基准线。

**核心逻辑**:

- **ARC-AGI-2 跳跃式突破**：Gemini 3.1 Pro 得分 77.1%，前代 Gemini 3 Pro 仅 31.1%，增幅超过 2 倍；Claude Opus 4.6 为 68.8%，GPT-5.2 为 52.9%。ARC-AGI-2 专门考察模型面对从未见过的逻辑题时的真实推理能力，不可靠死记硬背，是目前公认最难的推理基准之一（ARC-AGI-1 已被刷爆后推出的升级版）。
- **16 项基准赢 13 项**：推理与科学知识、竞赛编程、Agent 工作流是优势最集中的三个维度；唯一明显落后的推理类项目是含工具版 Humanity's Last Exam（Opus 4.6 以 53.1% 小胜 Gemini 的 51.4%）。
- **编程基准双标问题**：在统一 Terminus-2 框架下 Gemini 3.1 Pro 以 68.5% 领先；但 GPT-5.3-Codex 在各家自选框架的最佳成绩对比里报出 77.3%——两组数字测试条件不同，不可直接比较大小。
- **Agent 类存在短板**：GDPval-AA 专家任务测试中，Claude Sonnet 4.6 以 1633 大幅领先所有模型，Gemini 3.1 Pro 仅 1317，甚至低于 Opus 4.6 的 1606；t2-bench Retail 上 Opus 4.6 也以 91.9% 小胜 Gemini 的 90.8%。
- **四项明确弱点**：① Arena 平台用户投票中文本对话体验不如 Claude Opus 4.6；② 工具使用流畅度不如 Claude 和 GPT；③ 审美与创意写作"品味"弱于 Claude；④ 首次响应延迟平均 29 秒，Claude 和 GPT 通常几秒即开始输出。
- **开发者侧实质性升级**：输出 token 上限提升至 65,000；文件上传限制从 20MB 扩至 100MB（5 倍）；支持直接传 YouTube 链接由模型自行处理；新增专属"干活模式"端点 `gemini-3.1-pro-preview-customtools`，专门优化工具调用稳定性。
- **API 字段变更**：`total_reasoning_tokens` 字段已改名为 `total_thought_tokens`，正在使用 API 的开发者必须检查并更新代码。
- **技术路径推断**：官方定性为"改进核心推理能力"，外部分析认为背后是训练方法的重大升级——谷歌找到了更有效的"教 AI 解题"方式，进步最明显的方向（编程、逻辑推理、多步任务执行）与训练方向高度吻合。
- **战略意图清晰**：BrowseComp、MCP Atlas、APEX-Agents 这些新加入的测试项目专门考察 AI 自主上网、调用工具、完成复杂任务的能力；Google Antigravity Agent 开发平台与 customtools 端点同步上线，谷歌在搭建"让 AI 真正干活"的完整生态，而非只卖更聪明的模型。
- **现状仍是 Preview**：谷歌明确表示等各方面验证后才会全面发布，当前可先行试用。

### 🎯 关键洞察

**推理能力跳跃的结构性意义**

ARC-AGI-2 的 77.1% 不只是一个数字，它意味着 Gemini 3.1 Pro 在"真实推理"这个维度已经与竞品拉开代差。这个测试的设计逻辑是：题目全部是模型训练集里不存在的新题型，必须靠逻辑推导而非模式匹配。31.1% → 77.1% 的跳跃，说明谷歌在训练方法层面发生了质变，而不是简单的数据堆叠。

**空间推理能力的具体体现**

以往 AI 用纯代码生成动画时，空间位置经常出错（如骑自行车的人手脚与车把踏板对不上）。Gemini 3.1 Pro 在"鹈鹕骑自行车"SVG 动画测试中，鸟的姿态自然，自行车车架、链条、脚踏板细节完全符合常识——这是推理能力提升在创意编程场景的直接映射。

**三国格局的底层逻辑**

| 实验室 | 核心押注 | 护城河来源 | 当前优势 | 当前短板 |
|--------|---------|-----------|---------|---------|
| 谷歌 | 消费端 Agent 场景入口 | 搜索+Android+云基础设施 | ARC-AGI-2 第一、推理/编程基准领先、性价比最高 | 工具调用流畅度、对话体验、响应延迟 |
| Anthropic | 开发者端 Agent 基础设施 | MCP 协议标准制定者、Claude Code 市场占有率 | 绝大多数 AI Agent 系统底层跑 Claude、专家任务对话质量 | 消费端品牌认知弱 |
| OpenAI | 用户入口 + 品牌心智 | ChatGPT 先入为主、周活跃用户超 8 亿 | 消费市场"AI=ChatGPT"认知绑定、GPT-5.3-Codex 纯终端编程 | 模型分数不再领先 |

**MCP 的隐性控制力**：MCP（Model Context Protocol，AI 调用外部工具的通用语言）由 Anthropic 制定，谷歌、微软、各大云厂商均已接入。这意味着即便用户在非 Anthropic 平台上让 AI 干活，底层工具调用标准仍是 Anthropic 定义的——这是一种不显眼但极其深层的生态控制力。

### 🛠️ 操作流程

**1. 准备阶段：选择接入渠道**

根据身份选择对应入口：

- 开发者：Gemini API / Google AI Studio / Gemini CLI / Google Antigravity / Android Studio
- 企业用户：Vertex AI / Gemini Enterprise
- 普通用户：Gemini App（Google AI Pro/Ultra 用户有更高用量上限）；NotebookLM（仅限 Pro/Ultra）

**2. 核心执行：API 代码迁移检查清单**

```python
# ❌ 旧写法（已废弃）
reasoning_tokens = response.usage.total_reasoning_tokens

# ✅ 新写法
reasoning_tokens = response.usage.total_thought_tokens
```

- 切换模型标识符为 `gemini-3.1-pro-preview`
- 工具调用密集型任务改用 `gemini-3.1-pro-preview-customtools` 端点
- 文件上传逻辑检查：100MB 以内无需切割，直接上传
- YouTube 视频处理：直接传链接字符串，无需预处理

**3. 验证与优化**

- 用 ARC-AGI-2 类型的多步逻辑任务验证推理能力提升是否符合预期
- 对比 `customtools` 端点与标准端点在复杂工具调用任务上的稳定性差异
- 监控首次响应延迟（基准约 29 秒），对延迟敏感的场景需评估是否可接受
- 长文档/大代码仓库场景：测试 65,000 token 输出上限是否满足需求

### 📝 避坑指南

- ⚠️ **API 字段必须更新**：`total_reasoning_tokens` 已改名为 `total_thought_tokens`，不更新会导致字段读取失败，影响 token 用量监控和计费核对。
- ⚠️ **编程基准数字不可直接比较**：Gemini 3.1 Pro 的 68.5% 和 GPT-5.3-Codex 的 77.3% 来自不同测试框架，拿来直接对比会得出错误结论。
- ⚠️ **延迟敏感场景需提前评估**：首次响应延迟平均 29 秒，对实时交互体验要求高的产品（如客服机器人、实时助手）需在上线前做延迟压测。
- ⚠️ **当前仍是 Preview 版**：接口、参数、端点名称在正式版发布前均可能变动，生产环境接入需关注官方更新公告。
- ⚠️ **Agent 专家任务不是强项**：GDPval-AA 专家任务上 Gemini 3.1 Pro（1317）明显弱于 Claude Sonnet 4.6（1633），高复杂度专家对话 Agent 场景建议对比测试后再决策。
- ⚠️ **工具调用流畅度仍有差距**：复杂任务中工具选择稳定性不如 Claude 和 GPT，`customtools` 专属端点是当前的缓解方案，但不是根治。
