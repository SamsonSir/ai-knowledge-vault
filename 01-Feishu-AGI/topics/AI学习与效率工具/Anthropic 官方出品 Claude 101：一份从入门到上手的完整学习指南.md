### 📋 核心分析

**战略价值**: Anthropic Academy 的 Claude 101 免费课程（13节）系统覆盖 Projects / Skills / Artifacts / Connectors / Enterprise Search / Research Mode 六大模块，是目前最完整的官方 Claude 上手路径。

**核心逻辑**:

- **提示词三步法（Set the Stage → Define the Task → Specify Rules）**：先给背景（你的角色/行业/前提），再定任务（写作/分析/构建，越具体越好），最后指定风格或直接给样例参考。上下文越充分，Claude 理解需求越快。
- **迭代思维是核心心智模型**：第一个提示词只是对话起点，不追求一次完美。正确姿势：审阅初稿 → 找出有效/无效部分 → 针对性反馈（"删掉前两段，让结论更具行动导向"比"缩短篇幅"有效得多）→ 必要时直接开新对话而非硬扭。
- **4D AI 素养框架**（Delegate / Describe / Discern / Due Diligence）：① Delegate：判断哪些任务给 AI、哪些留给人，需理解 AI 能力边界；② Describe：精确定义输出、流程、行为方式；③ Discern：批判性评估输出的质量、准确性、适用性；④ Due Diligence：对 AI 辅助成果承担责任，保持透明度。
- **Evals 评估方法论**：收集 5–10 个高频任务样本 → 用样本测试 Claude 输出 → 与自己的成果对比（关键信息是否抓住、语气风格是否对、缺什么）→ 根据对比结果迭代提示词。这是判断 Claude 是否适合某个特定工作流的唯一可靠方法。
- **Projects = 知识中心**：每个 Project 有独立聊天记录 + 知识库 + 自定义说明。知识库用 RAG 技术处理，容量几乎无限制，支持各种文件格式及 Google Drive 连接（单文件最大 30MB）。注意：项目内各聊天之间上下文默认不共享，需手动加入知识库才能跨聊天复用。
- **Skills = 流程编码包**：Skills 文件夹包含指令、脚本、资源，Claude 动态加载后以可重复方式执行特定任务。Anthropic 官方 Skills 覆盖 Excel / Word / PowerPoint / PDF 创建，付费用户自动可用。自定义 Skills 通过对话创建：描述需求 → 回答 Claude 的流程问题 → 上传参考资料（模板/品牌素材）→ Claude 生成 ZIP → 上传到设置启用。
- **Artifacts = 可交互交付物**：满足"内容超过 15 行 + 自成一体 + 可能被复用"时自动生成，支持 Markdown / 代码片段 / HTML 页面 / SVG / Mermaid 图表 / React 组件六类。可在预览和代码视图间切换，支持复制、下载、发布链接、remix。
- **模型选择策略**：Opus → 复杂任务；Sonnet → 日常任务；Thinking 模式 → 需要深度推理；Research 模式 → 多角度调研。
- **Enterprise Search（企业搜索）**：同时搜索 SharePoint / Slack / Gmail / Google Drive，整合成统一答案并标注来源。权限隔离：只显示你在原有工具中已有访问权限的内容，数据不被单独索引或存储。设置分两步：管理员配置连接器 → 各用户用自己账号分别认证。
- **Research Mode 工作机制**：Claude 独立决定下一步调查方向，同时进行多项相互衔接的搜索，从不同角度探讨问题，输出带引文的综合报告。可与 Extended Thinking 组合使用（规划方法 + 信息收集双管齐下），界面有计时器 + 可展开的思考过程面板。

### 🎯 关键洞察

**Projects vs Skills vs Artifacts 三者关系**：

- Projects 提供"内容"（知识库里的客户资料、品牌文档、历史数据）
- Skills 提供"方法"（如何执行"客户通话准备"这个流程的具体步骤）
- Artifacts 是"结果"（执行完流程后生成的那份 PPT / 报告 / 代码）

三者组合逻辑：Skills 从 Projects 知识库中提取信息 → 按编码好的流程执行 → 输出 Artifact。

**Artifacts 提示词精度决定输出质量**：
- 低精度："创建一个预算追踪器"
- 高精度："创建一个月度预算追踪器，按类别输入支出，查看饼图明细，超支时提醒"
- 另一个关键变量：告诉 Claude 最终用户是谁（"此流程图适用于新员工" vs "此流程图适用于工程团队"，产出完全不同）

### 🛠️ 操作流程

**1. 准备阶段：建立第一个 Project**
- 新建 Project，撰写项目目标（说清楚这个项目是干什么的、面向谁）
- 上传相关文档（支持各种格式 + Google Drive，单文件 ≤30MB）
- 写好项目说明：语气要求 + 聚焦领域 + 禁止事项
- 适合场景：产品上市计划、内容创作工厂、教育课程开发、金融预算计划、家庭装修改造

**2. 核心执行：用三步法写提示词**
- Set the Stage：我是[角色]，在[行业/公司]，背景是[前提条件]
- Define the Task：我需要你[写作/分析/构建]，具体是[详细描述]
- Specify Rules：用[风格/语气]，参考[样例]，避免[禁忌]

**3. 创建自定义 Skill**
- 直接跟 Claude 对话："我想创建一个 Skill，用于[具体工作流]"
- 回答 Claude 关于流程细节的追问
- 上传参考资料（模板、风格指南、品牌素材）
- 下载 Claude 生成的 ZIP 文件
- 进入设置上传 ZIP，启用 Skill

**4. 验证与优化：Evals 流程**
- 收集 5–10 个高频任务的真实样本
- 用样本测试 Claude 输出
- 对比维度：关键信息覆盖率 / 语气风格匹配度 / 缺失内容
- 根据对比结果修改提示词，重复测试直到满意

### 💡 具体案例/数据

- **想法落地案例**：在办公桌前跟 Claude 讨论一个计划 → 离开时想法已同步到手机 → 日历记录时间、地图标注地点、提醒事项安排待办 → 当晚事情推进完毕
- **反馈精度对比**：
  - 低效反馈："缩短篇幅"
  - 高效反馈："删掉前两段，让结论更具行动导向"
- **Artifacts 触发条件**：内容超过 15 行 + 自成一体 + 可能被复用或独立存在
- **Research Mode 耗时**：几分钟到几小时，取决于任务复杂度
- **Enterprise Search 典型问法**：
  - "昨天我不在的时候发生了什么"
  - "公司远程办公政策是什么"
  - "身份验证系统是怎么运作的"

### 📝 避坑指南

- ⚠️ 项目内各聊天之间上下文默认不共享，跨聊天复用信息必须手动加入项目知识库
- ⚠️ Artifacts 不会在内容少于 15 行时自动创建，需手动说"将其创建为 Artifact"
- ⚠️ 自定义 Skill 质量高度依赖参考资料的完整性，模板/品牌素材越详细，生成的 Skill 越精准
- ⚠️ Connectors 连接后务必验证权限：直接问 Claude "你有没有权限访问 [工具名]"
- ⚠️ Research Mode 耗时不可控（几分钟到几小时），适合异步场景，不适合需要即时结果的场合
- ⚠️ 对话偏离方向时，开新聊天比硬扭回来更高效，不要在一个跑偏的对话里反复纠正
