### 📋 核心分析

**战略价值**: 通过 PDF→Markdown 转换流水线，将任意扫描/文字 PDF 喂入 Cherry Studio 知识库，实现 AI 秒级精准检索。

**核心逻辑**:

- **根本原因**：扫描版 PDF 每页是图片，AI 无法提取文字，导入知识库后嵌入失败，查询返回空。
- **三种文档格式的检索效果差异**：扫描版 PDF（图片）= 完全无法识别；可搜索版 PDF（WPS OCR 转换）= 部分识别，有错字；Markdown 文档 = 完整识别，效果最佳。
- **核心转换路径**：扫描版 PDF → WPS OCR → 可搜索 PDF → Trae/扣子空间 pdf-2-markdown → Markdown → Cherry Studio 知识库。
- **pdf-2-markdown 技能**：基于开源项目 `markitdown` 二次迭代，集成本地 OCR + LLM 大模型识别兜底，并对输出执行格式清洗（去噪、段落拼接、结构修复）。
- **WPS OCR 优先于本地 OCR**：本地 OCR 效果尚可但速度慢（低配电脑尤甚），WPS 会员 OCR 速度更快，适合扫描件批量处理。
- **格式清洗是关键步骤**：即使 OCR 识别准确，原始文本仍有噪声、段落断裂、结构错乱，必须经过清洗才能让知识库嵌入质量达标。
- **扣子空间技能入口**：`https://www.coze.cn/?skills=7598110618176028723`（审核中，可能需等待），单文件限制 50M。
- **技能源码开源地址**：`https://github.com/shaoyi1991/pdf-2-markdown`，可自行部署或二次迭代。
- **SOP 四步标准流程**（技能内部逻辑）：① 识别输入格式选转换路径（文本提取/本地 OCR/LLM）→ ② 执行内容提取得原始文本 → ③ 格式清洗（去噪、段落拼接、结构修复）→ ④ 写入 Markdown，头部标注"生成方式 + 自动清洗"。
- **知识库对话时强制只走知识库查询**，避免 AI 用自身训练数据回答，确保答案来源可溯。

### 🎯 关键洞察

**为什么 Markdown 是知识库的最优格式**：
- TXT 是纯文本，无结构标记，AI 难以区分标题/正文/列表层级。
- PDF（尤其扫描版）本质是图像容器，嵌入模型处理的是向量化文字，图片无法向量化。
- Markdown 是计算机可结构化解析的标记语言，标题层级（`#`/`##`）直接对应知识库的分块逻辑，嵌入质量显著高于 TXT 和 PDF。

**原因 → 动作 → 结果链**：
- 扫描件清晰度低 → OCR 错字多 → 知识库嵌入噪声大 → AI 查询命中率低
- 解法：WPS OCR（高质量识别）+ 格式清洗（去噪）→ 干净 Markdown → 嵌入质量高 → AI 精准命中

### 🛠️ 操作流程

**以 176 页《民法典》扫描版 PDF 为例**

1. **准备阶段**
   - 判断文档类型：文字可选中 → 直接跳到第 3 步；文字不可选中（扫描版）→ 进入第 2 步。
   - 工具准备：Cherry Studio（已安装配置）、WPS 会员版、Trae 或扣子空间账号。

2. **核心执行**
   - **Step 1 - OCR 识别**（扫描版必做）：WPS 打开扫描版 PDF → 顶部菜单「识别文字」→「多文档」→「开始转换」→ 得到可搜索版 PDF。验证：用鼠标能选中文字即成功。
   - **Step 2 - 转换为 Markdown**（二选一）：
     - 快速版（10 分钟，小白）：访问扣子空间技能链接 `https://www.coze.cn/?skills=7598110618176028723`，上传 PDF（≤50M），直接下载 MD 文件。
     - 完整版（30 分钟，进阶）：在 Trae 中配置好 `pdf-2-markdown` 技能，直接对话"请将 xxx.pdf 转换为 Markdown"，AI 自动执行并生成 MD 文件。转换完成后随机抽查几段与原文比对。
   - **Step 3 - 导入 Cherry Studio 知识库**：Cherry Studio → 知识库 → 新建/选择知识库 → 上传 MD 文件 → 等待嵌入完成（状态显示"已完成嵌入"）。

3. **验证与优化**
   - 新建对话，挂载对应知识库，设置强制只走知识库查询。
   - 提问知识库内具体内容（如某条法律条文），验证 AI 返回内容与原文一致。
   - 若有格式错误（段落断裂、乱码），在 MD 文件中手动修正后重新导入。

### 💡 具体案例/数据

- 测试文档：《民法典》176 页，先人为转成扫描版，再走完整流程。
- 三格式对比测试：同一问题"东汉王朝有几个皇帝，存在多久？"分别用三种格式的知识库查询：
  - 扫描版 PDF → AI 无法从知识库找到答案
  - 可搜索版 PDF → 部分识别，答案不完整或有错字
  - Markdown 文档 → 精准返回，与原文一致
- pdf-2-markdown 技能迭代周期：作者花费 3-4 天，经多次迭代，集成 OCR + LLM 兜底 + 格式清洗，才达到知识库可用标准。

### 📝 避坑指南

- ⚠️ 扫描件分辨率低或含大量图片时，WPS OCR 仍会有错字，转 MD 后需人工抽查关键段落。
- ⚠️ 扣子空间技能单文件限制 50M，超大 PDF 需先拆分再分批上传。
- ⚠️ 知识库对话时若不强制指定"只从知识库查询"，AI 会混用自身训练数据，导致答案来源不可控。
- ⚠️ 可搜索版 PDF 直接导入知识库效果仍差于 Markdown，不要跳过 MD 转换步骤。
- ⚠️ 本地 OCR 在低配电脑上速度极慢，优先用 WPS 会员 OCR 处理扫描件。
- ⚠️ 扣子空间技能审核中，访问前确认是否已上线，否则需改用 Trae IDE 方式。
