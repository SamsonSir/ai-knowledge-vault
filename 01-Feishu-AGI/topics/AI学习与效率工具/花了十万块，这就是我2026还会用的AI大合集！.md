### 📋 核心分析

**战略价值**: 一位重度AI用户（年消费近十万）跨模型、跨场景的真实选品清单，覆盖从生图/视频/编程/知识管理到工作流自动化的完整AI工具栈，可直接作为个人/团队AI工具选型参考。

**核心逻辑**:

- **大模型分工明确**：Gemini 3.0 Pro 主攻多模态（Excel数据+截图+视频上传）；GPT 5.2 Thinking 主攻写作与深度推理；DeepSeek 专攻论文级文本处理、AI翻译、文本纠错，API便宜大碗持续首选。
- **信息搜索三角分工**：OpenAI Deep Research 覆盖中英文全网、幻觉极低、无需二次核实；Grok 定向监控 Reddit + X 最新动态，用 Task 功能设定每天早8点自动抓取特定话题前10条热帖；元宝专盯公众号+视频号生态。
- **生图首选 Nano Banana（Pro版）**：支持"用嘴改图"（语音指令修图），中文海报场景可用即梦/豆包的 Seedream 4.5 补位。
- **AI视频核心指标是声画同步**：Veo 3.1 声画同步已达可接受价格区间，实测一条咬香蕉ASMR视频获12万围观、半年持续涨赞；Sora 2 Pro 主打人物合拍与趣味性。
- **编程组合拳四件套**：AI Studio + Gemini 3.0 Pro 做快速原型验证 → Claude Code（默认 Claude Sonnet 4.5，可切换 Kimi K2 / GLM 4.7 / MiniMax M2.1）搭框架 → CodeX + GPT 5.2 补Bug与细节 → Cursor 做精细修改与进度管理。Claude Code 使用的是 guizang.ai 的 GitHub 项目 `ai-claude-star`。
- **声音克隆从11labs全面切换到MiniMax**：支持方言，有MCP接口，可在音频/视频/图片场景直接调用，API形式灵活接入。
- **AI音乐 Suno 实测数据**：用 Suno 制作的大东北系列古风纯音乐多次登上B站热搜。
- **AI输入法双端方案**：手机端豆包输入法（识别精准度测评第一，但键盘不支持emoji）；电脑端闪电说+麦克风，按住 fn 语音转文字，接 DeepSeek 纠错，用于回微信消息，从三天处理一次提升到每天可处理。
- **AI浏览器核心价值**：Perplexity Comet + Dia，浏览器本身即大上下文，Agent模式可直接完成"看完B站教程视频→提取关键步骤和代码→更新到Notion页面"的跨网页复合任务，AI分组解决几十个标签页的信息过载。
- **知识管理双引擎**：NotebookLM 支持 Flash Research/Deep Research/网页/音视频/图片等几乎所有格式导入，输出方式最多（PPT、信息图等）；腾讯 IMA 自带50GB存储，支持知识库嵌套（知识库A可将B/C/D作为信息源）。
- **工作流自动化双路径**：n8n 作为万能连接器打通不相干App；飞书多维表格做轻量批量出图——在表格第一列放图，飞书自动读取生成 Nano Banana 产品图，后续列批量生成周边图，可扩展到批量出视频。

### 🎯 关键洞察

**Grok Task 功能具体用法**：设置每天早上8点，定向抓取某个特定关键词的前10条热门帖子。比自己手动换搜索词的准确度高，相当于一个定时信息专员。（作者提示后续会单独写一篇）

**Claude Code + ai-claude-star 项目**：默认模型 Claude Sonnet 4.5，启动时可额外切换 Kimi K2、GLM 4.7、MiniMax M2.1，实现多模型协作而不需要分别开多个工具。项目地址来自 guizang.ai（归藏）的 GitHub。

**飞书多维表格批量出图 SOP 逻辑**：
- 第一列：放基础图（产品图素材）
- 飞书自动读取第一列 → 调用 Nano Banana 生成完整产品图
- 后续列：等产品图生成完毕后，批量生成周边图
- 可扩展：同逻辑批量出视频

**AI浏览器改变工作习惯的本质**：不再需要"复制粘贴给AI"，浏览器本身就是持续的AI上下文，跨网页任务在Agent对话中一步完成。

### 🛠️ 操作流程

**飞书多维表格批量出图**

1. 准备阶段：在飞书多维表格新建表格，第一列填入所有想要的画面元素和产品名字，上传基础素材图。
2. 核心执行：飞书自动读取第一列图，调用 Nano Banana 生成完整产品图，等待生成完毕。
3. 验证与优化：产品图生成完毕后，在后续列批量触发周边图生成；如需视频，同逻辑扩展到批量出视频。

**Grok Task 定时信息抓取**

1. 准备阶段：进入 Grok，找到 Task 功能入口。
2. 核心执行：设置触发时间为每天早上8点，输入目标关键词，设定抓取数量为前10条热门帖子，指定来源为 Reddit 或 X。
3. 验证与优化：对比手动搜索结果，确认准确度，按需调整关键词。

**Claude Code 多模型切换（ai-claude-star）**

1. 准备阶段：从 guizang.ai 的 GitHub 克隆 `ai-claude-star` 项目，按说明配置。
2. 核心执行：启动 Claude Code，默认使用 Claude Sonnet 4.5；需要时在启动时切换为 Kimi K2、GLM 4.7 或 MiniMax M2.1。
3. 分工执行：Claude Code 搭框架 → CodeX + GPT 5.2 补Bug → Cursor 精细修改与进度管理。

### 📝 避坑指南

- ⚠️ Gemini 3.0 Pro 写作能力一般，不要用它做纯文字创作，它的优势在多模态输入处理。
- ⚠️ 豆包输入法键盘不支持 emoji，群聊场景不便，需搭配其他输入法使用。
- ⚠️ n8n 上手难度较高，不适合零基础直接上手，建议先用飞书多维表格跑通批量出图逻辑再迁移。
- ⚠️ Sora 2 Pro 初期生成清晰度一般，适合趣味性/人物合拍内容，不适合对画质要求高的商业项目。
- ⚠️ 夸克S1 AI眼镜（方框）目前购买渠道受限，需要线下渠道获取。
- ⚠️ 豆包手机已被下架，无法购买使用。
- ⚠️ AI浏览器（Comet/Dia）目前功能尚未完全覆盖 Arc，部分工作流需要手动补位。
