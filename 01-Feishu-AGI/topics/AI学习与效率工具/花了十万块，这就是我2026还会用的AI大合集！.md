### 📋 核心分析

**战略价值**: 一位重度 AI 内容创作者（年度 353 篇内容）经过真实高频使用后，按场景筛选出的 2026 年核心 AI 工具栈，覆盖模型/搜索/生图/视频/编程/音乐/浏览器/工作流全链路。

**核心逻辑**:
- **大模型分工明确**：Gemini 3.0 Pro 主攻多模态（Excel 数据+截图+视频上传），GPT 5.2 Thinking 主攻写作输出，DeepSeek 主攻论文级 AI 翻译与文本纠错（API 便宜大碗）
- **信息搜索三角覆盖**：OpenAI Deep Research 覆盖中英文全网（幻觉极低，无需二次确认信源）；Grok 盯 Reddit+X 动态（Task 功能定时抓取，每天 8 点获取特定话题前 10 条热帖）；元宝盯公众号+视频号
- **生图首选 Nano Banana**：无论 Pro 还是初版，直接用自然语言"嘴改图"；中文海报场景可用即梦/豆包的 Seedream 4.5
- **AI 视频核心是声画同步**：Veo 3.1 做声画同步内容（做的咬香蕉 ASMR 视频获 12 万围观，半年后仍持续涨赞）；Sora 2 Pro 主攻人物合拍与娱乐向玩法
- **编程用组合拳**：AI Studio+Gemini 3.0 Pro 快速原型验证 → Claude Code（基于 guizang.ai 的 ai-claude-star 项目，默认 Claude Sonnet 4.5，可切换 Kimi K2/GLM 4.7/MiniMax M2.1）搭框架 → CodeX+GPT 5.2 补 Bug 和细节 → Cursor 做精细修改和进度管理
- **声音合成全面转向 MiniMax**：替代原 ElevenLabs，支持方言，有 MCP 接口（可生成音频/视频/图片），适合 API 化接入任意工作流
- **AI 音乐 Suno 做配乐**：专用于古风纯音乐和短视频配乐，大东北系列多次登 B 站热搜
- **知识管理双轨并行**：NotebookLM 做专项知识库（支持 Flash Research/Deep Research/网页/音视频/图片，输出形式最丰富，PPT 和信息图直接产出）；腾讯 IMA 做知识库嵌套（知识库 A 可以把 B/C/D 作为信源，自带 50GB 存储）
- **自动化工作流核心是 n8n + 飞书多维表格**：n8n 做跨 App 服务连接；飞书多维表格做批量出图（第一列传参 → 自动调 Nano Banana 生成产品图 → 后续列批量生成周边图）
- **AI 浏览器彻底改变工作流习惯**：Perplexity Comet + Dia，核心价值是「浏览器即大 AI 上下文」——看完 B 站教程视频后直接 Agent 对话，自动总结关键步骤并更新到 Notion；几十个标签页可 AI 分组或按主题分开总结

### 🎯 关键洞察

**Grok Task 功能实操**：设置每天早上 8 点定向抓取某特定信息前 10 条热门帖子，比手动换关键词搜索准确率高很多。适合需要持续监控 Reddit/X 特定话题的人，作者单独标注后续会写专篇拆解。

**Claude Code 模型切换方案**：使用 GitHub 项目 `ai-claude-star`（来自 guizang.ai），默认 Claude Sonnet 4.5，启动时可额外切换 Kimi K2、GLM 4.7、MiniMax M2.1，实现多模型协同而不绑死单一 API。

**飞书多维表格批量出图工作流逻辑**：
1. 第一列：填入所有想要的画面元素 + 产品名
2. 飞书自动读取第一列图 → 调用 Nano Banana 生成完整产品图
3. 后续列：等产品图完成后 → 批量生成周边图
4. 进阶：同样逻辑可批量出视频

**MiniMax MCP 能力**：一个 MCP 接口可同时生成音频、视频、图片，作者在工作流中音频场景（AI 翻译、声音克隆）全部接 MiniMax，替代此前的 ElevenLabs，且支持中文方言。

### 🛠️ 操作流程：Grok Task 定时信息抓取

1. **准备**：开通 Grok，进入 Task 功能模块
2. **设置定时任务**：配置每天早上 8:00 执行
3. **指定抓取目标**：填入特定信息关键词/话题方向
4. **设置抓取范围**：定向 Reddit 和/或 X 平台
5. **设置返回数量**：前 10 条热门帖子
6. **验证**：次日 8 点后查看推送，对比手动换关键词的搜索结果

---

### 🛠️ 操作流程：飞书多维表格批量生图

1. **建表**：飞书多维表格新建数据表
2. **第一列**：填入所有目标产品名称 + 画面元素描述
3. **接入 Nano Banana**：通过飞书自动化/API 接入 Nano Banana 图像生成
4. **触发生成**：飞书自动读取第一列内容 → 调用 Nano Banana → 生成完整产品图
5. **后续列扩展**：产品图生成完毕后，后续列自动触发批量生成周边图
6. **进阶**：同样框架可替换视频生成模型，批量出视频

---

### 🛠️ 操作流程：Claude Code 多模型切换配置

1. **获取项目**：前往 guizang.ai 或 GitHub 找 `ai-claude-star` 项目
2. **安装配置**：按项目文档完成安装
3. **默认模型**：Claude Sonnet 4.5（框架搭建主力）
4. **启动时切换**：可在启动 Claude Code 时切换为：
   - Kimi K2
   - GLM 4.7
   - MiniMax M2.1
5. **分工执行**：Claude Code 搭框架 → CodeX+GPT 5.2 补 Bug → Cursor 精细修改+进度管理

### 📝 避坑指南

- ⚠️ Gemini 3.0 Pro 写作能力一般，不适合作为写作主力，核心价值在多模态输入处理
- ⚠️ 豆包输入法的键盘目前不支持打 emoji，如果频繁用 emoji 找群聊会不便
- ⚠️ n8n 上手难度较高，不适合零基础用户直接上手复杂工作流（作者正在准备真小白版教程）
- ⚠️ 夸克 S1 AI 眼镜目前渠道稀缺，购买前需确认货源
- ⚠️ 豆包手机已被下架，无法购买使用
- ⚠️ Claude Code 的 Skill 功能作者尚未拆解，MCP 功能已有多篇文章（包括空间 Agent/家具定制场景）
