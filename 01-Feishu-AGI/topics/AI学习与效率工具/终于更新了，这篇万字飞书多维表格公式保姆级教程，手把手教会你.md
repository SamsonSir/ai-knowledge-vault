### 📋 核心分析

**战略价值**: 飞书多维表格公式完整操作手册，覆盖文本/时间/逻辑/聚合/清洗全链路，让业务人员无需编程背景即可构建自动化数据系统。

**核心逻辑**:

- **公式字段 ≠ 单元格公式**：多维表格不能在单个单元格写公式，只能在字段（列）级别定义规则，规则自动作用于整列所有记录。新建方式：点击表头最右侧 `+` → 选择「公式」字段类型。
- **文本拼接用 `&` 而非 CONCATENATE**：`[部门] & [编号]` 比 CONCATENATE 更简洁；加分隔符写法：`[部门] & "-" & [编号]`；Emoji 输入：Windows 按 `Win + .`，Mac 右键选「表情符号」，可直接嵌入公式。
- **时间计算三件套**：`DATEDIF(起始, 结束, "Y"/"D"/"YD")` 算年/天/忽略年份的天数；`YEAR()`/`MONTH()`/`DAY()` 提取具体数值；`WEEKDAY()` 返回星期数字，配合 IF 自动标记周末。
- **工作日计算用 NETWORKDAYS 双表法**：`NETWORKDAYS([开始日期], [结束日期], [2026年节假日].[法定假期日期])`，先建独立节假日表录入法定假日，主表公式引用该表，自动剔除周末+法定假日，算出真实工作日。反向推算用 `WORKDAY()`。
- **空值处理两件套**：`ISBLANK` 是判断者（空→True，非空→False），配合 IF 做视觉标记：`IF(ISBLANK([交付日期]), "❌", "✅")`；`IFBLANK` 是补救者，`IFBLANK([交付日期], NOW())` 等价于 `IF(ISBLANK([交付日期]), NOW(), [交付日期])`，但更简洁。
- **多条件判断用 IFS 而非嵌套 IF**：`IFS([销售额]>=800,[销售额]*0.2,[销售额]>=500,[销售额]*0.1,TRUE(),0)`；最后必须加 `TRUE()` 作兜底，否则前面条件都不满足时会报错。
- **报错处理用 IFERROR 遮瑕**：`IFERROR([总价]/[数量], "数据有误")`，计算出错时显示自定义文本而非 `#DIV/0!`；`ISERROR` 用于诊断（返回 True/False），`IFERROR` 用于直接替换错误结果。
- **两种数据引用模式决定能力边界**：引用 `[字段名]` = 当前行数据；引用 `[数据表].[字段名]` = 整列集合，可跨表。百分比公式：`[电脑销量] / SUM([A组销售数据].[电脑销量])`，结果为小数时在字段格式里改为百分比显示。
- **FILTER + CurrentValue 是核武器**：`[销售总表].FILTER(CurrentValue.[销售人员]="销售001").[销售额]`；多条件用 `&&`（且）或 `OR()`（或）；不等于用 `!=`；区间筛选：`CurrentValue.[销售额]>100 && CurrentValue.[销售额]<500`。
- **多条件求和用 FILTER+SUM 替代 SUMIFS**：多维表格无 SUMIFS，用 `[销售总表].FILTER(CurrentValue.[销售人员]="周北北" && CurrentValue.[商品类型]="食材").[销售额].SUM()`；筛选非空：`CurrentValue.[销售人员]!=""`。

### 🎯 关键洞察

**集合引用是理解多维表格公式的核心心智模型**：
- 当前行引用（`[字段]`）→ 一个值
- 全表引用（`[表].[字段]`）→ 一组值（集合）
- FILTER 在集合上打筛子 → 子集合
- SUM/COUNT/COUNTA/UNIQUE 在集合上做聚合 → 结果值

这个链路打通后，所有复杂公式都是这条链的组合变体。

**AI 生成公式的计费逻辑**：每次点击生成消耗 3 个 AI 点数；AI 字段捷径按行收费，1000 行 = 单次消耗 × 1000，刷新单元格会重新扣点；火山引擎捷径默认无免费额度，需在配置面板关联自己的火山引擎账号才能用其免费额度；自动化工作流 AI 对话基础消耗 3 点 + 模型思考费用，AI 节点运行 0.1 点/次，仪表盘 AI 智能总结 2 点/次，智能分析按钮 0.5 点/次。

### 📦 配置/工具详表

| 函数/功能 | 关键公式/写法 | 预期效果 | 注意事项/坑 |
|----------|-------------|---------|-----------|
| 文本拼接 | `[部门] & "-" & [编号]` | 合并字段内容 | 用 `&` 而非 CONCATENATE |
| 工龄/天数 | `DATEDIF([入职日期], TODAY(), "Y")` | 返回整年数 | 单位："Y"年/"D"天/"YD"忽略年份天数 |
| 净工作日 | `NETWORKDAYS([开始], [结束], [节假日表].[日期])` | 剔除周末+法定假日 | 需单独建节假日表 |
| 空值判断 | `IF(ISBLANK([字段]), "❌", "✅")` | 视觉标记空值 | ISBLANK 返回 True/False |
| 空值补救 | `IFBLANK([字段], NOW())` | 空时填默认值 | 等价于 IF+ISBLANK 嵌套 |
| 多条件判断 | `IFS(条件1,结果1,条件2,结果2,TRUE(),"兜底")` | 阶梯逻辑 | 最后必须加 TRUE() 兜底否则报错 |
| 报错遮瑕 | `IFERROR([总价]/[数量], "数据有误")` | 替换错误显示 | 不影响其他行计算 |
| 条件筛选 | `[表].FILTER(CurrentValue.[字段]="值").[目标字段]` | 跨表取数 | CurrentValue 代表逐行检查的当前值 |
| 条件计数 | `[表].COUNTIF(CurrentValue.[分数]>80)` | 统计满足条件的行数 | 多条件用 AND() 包裹 |
| 查重标记 | `IF([表].COUNTIF(CurrentValue.[货品名称]=[货品名称])>1, "重复", "")` | 自动标记重复录入 | 出现次数>1即为重复 |
| 多条件求和 | `[表].FILTER(条件1 && 条件2).[字段].SUM()` | 替代 SUMIFS | 多维表格无 SUMIFS |
| 信息聚合 | `ARRAYJOIN([表].FILTER(...).[字段], CHAR(10))` | 多值合并到一格 | CHAR(10) 为换行符 |
| 打包列表 | `LIST([字段1], TEXT([日期字段], "YYYY/MM/DD"), [字段3])` | 临时打包多列 | 日期字段必须用 TEXT() 转换，否则变序列号 |
| 批量加工 | `[表].FILTER(...).[字段].MAP("《" & CurrentValue & "》")` | 对列表每项执行操作 | MAP 禁止嵌套 MAP |
| 去重 | `UNIQUE([表].[字段])` | 去除重复值 | 不排序，只去重 |
| 去重计数 | `UNIQUE([表].FILTER(条件).[字段]).COUNTA()` | 统计不重复的数量 | 三步：FILTER→UNIQUE→COUNTA |
| 字符提取 | `LEFT([邮箱], FIND("@", [邮箱], 1) - 1)` | 提取@前的用户名 | FIND 返回位置数字 |
| 模糊计数 | `[表].COUNTIF(FIND("螺栓", CurrentValue.[货品名称]) > 0)` | 统计包含关键字的行 | FIND 找到返回>0，找不到报错需配合 IFERROR |
| 关联字段取值 | `[关联字段].[目标字段]` | 直接拿关联表数据 | 打点后系统弹出字段菜单，无需建 Lookup 字段 |
| 地理位置提取 | `[地址字段].[省]` / `[地址字段].[市]` | 提取省市区 | 暂不支持海外地址 |
| 距离计算 | `DISTANCE([地址1], [地址2])` | 返回直线距离（千米） | 两个地理位置字段 |
| 日期格式化 | `TEXT([日期], "YYYYMMDD")` / `TEXT([日期], "YYMMDD")` | 生成合同编号等 | TEXT 结果是文本，不能直接 SUM，需 VALUE() 转回数字 |
| 时间差格式化 | `TEXT([下班]-[上班], "hh:mm")` | 显示工时 | 跨天/跨24h用 `"[h]:mm"` |
| 工号补零 | `TEXT([编号], "0000")` | 5 → 0005 | 结果为文本 |
| 消除时间尾巴 | 双击列表头 → 字段格式 → 纯日期 | 去掉 00:00:00 | 不改公式，改字段格式更简单 |

### 🛠️ 操作流程

**1. 净工作日计算（双表法）**

1. 新建独立数据表，命名「2026年节假日」，录入全年法定假日日期（只录需额外扣除的日期）
2. 在主表（如「净工作日计算」）建好「开始日期」和「结束日期」字段
3. 主表新建公式字段，输入：
   ```
   NETWORKDAYS([开始日期], [结束日期], [2026年节假日].[法定假期日期])
   ```
4. 反向推算（10个工作日后是哪天）：`WORKDAY([开始日期], 10)`

**2. 去重计数（统计今日不重复公司数）**

```
UNIQUE(
  [客户信息管理].FILTER(CurrentValue.[录入日期]=[日期]).[公司名称]
).COUNTA()
```
逻辑链：筛今日记录 → 取公司名列 → 去重 → 计数

**3. 质检合格判断（LIST + IFS + COUNTIF）**

```
IFS(
  [设备状态汇总].COUNTIF(CurrentValue="是")=3, "合格",
  [设备状态汇总].COUNTIF(CurrentValue="")=3, "未质检",
  TRUE, "不合格"
)
```
其中 `[设备状态汇总]` 是一个 LIST 字段：`LIST([外观], [控制器], [按钮])`

**4. 样本查漏（LIST + CONTAIN）**

```
IF(CONTAIN(LIST([质量报告], [评级报告]), "N"), "❌", "✅")
```
任意一项为"N"即打叉

**5. 使用 AI 生成公式**

1. 新建公式字段 → 点击「编辑公式」
2. 切换到「AI 生成公式」标签页
3. 用自然语言描述需求，如「统计本月商品的销售量」
4. 点击生成（消耗 3 个 AI 点数）
5. 确认解释无误后点「采纳」，代码自动填入编辑框

### 💡 具体案例/数据

**阶梯提成计算**：
```
IFS(
  [销售额]>=800, [销售额]*0.2,
  [销售额]>=500, [销售额]*0.1,
  TRUE(), 0
)
```

**成绩评级**：
```
IFS(
  [成绩]>=85, "优秀",
  [成绩]>=75, "良好",
  [成绩]>=60, "及格",
  TRUE(), "不及格"
)
```

**提取邮箱用户名**：
```
LEFT([邮箱地址], FIND("@", [邮箱地址], 1) - 1)
```

**提取「部门-姓名」中的姓名**：
```
MID([人员信息], FIND("-", [人员信息]) + 1, LEN([人员信息]))
```

**生成合同编号（日期格式）**：
```
TEXT([销售日期], "YYMMDD")
```

**生成业务陈述句**：
```
TEXT([购入日期], "YYYY/MM/DD") & "购入" & "-" & [库存情况]
```

**计算工时**：
```
TEXT([下班时间]-[上班时间], "hh:mm")
```
跨24小时：`TEXT([日期1]-[日期2], "[h]")`

**给书名批量加书名号**：
```
[书籍清单].FILTER(...).[书名].MAP("《" & CurrentValue & "》")
```

**字母批量转大写**：
```
MAP(LIST("a", "b"), UPPER(CurrentValue))
```

**多字段合并为摘要**：
```
ARRAYJOIN(LIST([设备], [设备系统], [问题类型], [优先级]), " - ")
```

**多选字段自定义拼接**：
```
ARRAYJOIN([擅长领域], " & ")
```

### 📝 避坑指南

- ⚠️ **IFS 必须加 TRUE() 兜底**：前面所有条件都不满足时，IFS 会报错，最后一个条件必须写 `TRUE(), "默认值"`
- ⚠️ **TEXT 函数结果是文本，不是数字**：`TEXT([编号], "0000")` 得到的 "0005" 不能直接 SUM，需用 `VALUE()` 转回数字再计算
- ⚠️ **LIST 里的日期字段必须用 TEXT() 包裹**：直接放日期字段会变成序列号（如 44567），正确写法：`LIST([产品名称], TEXT([日期], "YYYY/MM/DD"), [类别])`
- ⚠️ **MAP 禁止嵌套**：MAP 内部不能再写 MAP，否则直接报错
- ⚠️ **日期带时间尾巴（00:00:00）**：优先在字段格式里改（双击表头→字段格式→纯日期），而非改公式；公式强制改法：`TEXT([结果], "YYYY-MM-DD")`
- ⚠️ **AI 字段捷径按行计费**：1000 行数据触发一次 = 单次消耗 × 1000；刷新单元格会重新扣点，避免手抖
- ⚠️ **火山引擎捷径无默认免费额度**：需在配置面板关联自己的火山引擎账号才能使用其免费额度
- ⚠️ **地理位置字段提取省市区暂不支持海外地址**
- ⚠️ **UNIQUE 不排序**：只去重，不改变顺序，需排序另用 SORT 函数
- ⚠️ **百分比显示问题**：公式算出 0.05 是正常的，不要改公式，在字段格式里切换为百分比格式即可
