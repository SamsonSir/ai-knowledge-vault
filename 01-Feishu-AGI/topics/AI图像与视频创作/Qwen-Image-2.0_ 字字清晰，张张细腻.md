### 📋 核心分析

**战略价值**: Qwen-Image-2.0 将生图与编辑两条技术支线合并为单一 omni 模型，在文字渲染精度（1k token 指令）、2K 写实质感、图像编辑一致性三个维度同步突破，且以 7B 轻量架构实现。

**核心逻辑**:

- **技术路线合并**：2025年5月项目启动后分两条支线并行——生图支线（8月 Qwen-Image 文字渲染 → 12月 Qwen-Image-2512 细节质感）、编辑支线（8月单图编辑 → 9月多图编辑 → 12月 Qwen-Image-Layered 图层拆分 → 12月 Qwen-Image-Edit-2511 一致性提升），2026年2月10日 Qwen-Image-2.0 将两条支线合二为一。
- **文字渲染特性①「准」**：支持复杂"画中画"渲染，例如在 PPT 时间轴节点内嵌套组图（戴帽子小狗 → 去帽小狗），模型不仅完成渲染还保持了图内角色一致性。
- **文字渲染特性②「多」**：支持 1k token 指令长度，可直出包含多栏数据表、流程图、统计显著性检验（p<0.05, 95% CI, Cohen's d=0.32）等复杂信息图表；复杂 Prompt 可通过 LLM 改写获得（用户输入简短意图 → LLM 扩写为详细描述 → 喂给 Qwen-Image-2.0）。
- **文字渲染特性③「美」**：生成图文混排时，模型自动将文字渲染在画面空白区域，不遮盖图像主体；支持多种书法字体，包括瘦金体（宋徽宗赵佶风格）、王羲之小楷、田英章硬笔、赵孟頫楷书等，且能在水墨长卷中完整渲染《兰亭集序》全文（极个别字有误差，整体准确）。
- **文字渲染特性④「真」**：可同时渲染多种介质上的文字——玻璃白板上的手写体、衣服上的 Logo 印刷体、杂志封面的排版字体，各介质材质、空间斜度、光影反射均有差异化处理，增强真实感；电影海报场景中，密集文字元素（出品方、主演、制作信息）与画面光影自然融合。
- **文字渲染特性⑤「齐」**：漫画对话框内文字自动居中对齐；信息图中相似段落自动对齐；日历格内多行文字（农历日期 + 公历日期 + 节假日标注）排布整齐。
- **写实质感提升**：2K 分辨率（2048×2048）原生支持；森林场景可区分 23 种以上不同明度/饱和度/冷暖倾向/材质（蜡质、绒面、革质、胶质）的绿色，丁达尔光束、苔藓露珠、叶脉半透明感均有精细刻画。
- **omni 模型增益传导**：生图能力的文字渲染和写实质感提升，直接传导给图像编辑能力——可在已有图片上题写书法诗词（指定字体、排列方向）、九宫格姿势组图生成、双图合成自然合照（指定镜头参数：等效50mm, f/4.0, 1/160s, ISO 200）、跨次元编辑（真实城市照片 + 卡通形象融合，保持照片真实性）。
- **模型规模**：7B 参数，架构为 8B Qwen3-VL Encoder → 7B Diffusion Decoder → 2048×2048 像素输出，在速度与画质间取得平衡。
- **AI Arena 盲测**：在文生图和图生图两个基准中均取得优越性能（具体数值见原文图表）。

### 🎯 关键洞察

**LLM 改写 Prompt 的工作流是实用关键**：用户无需手写千字 Prompt，只需输入简短意图（如"帮我生成一个手绘风格的杭州两日禅意人文之旅双语海报"），交由任意 LLM 扩写为包含具体时间、地点、人物、色调、排版、双语对照的详细描述，再输入 Qwen-Image-2.0。这一工作流将模型的 1k token 指令上限真正用满，是解锁复杂渲染能力的核心路径。

**omni 架构的核心价值**：单模型同时覆盖文生图 + 图像编辑，避免了多模型切换的工程复杂度；更重要的是，生图侧的能力提升（文字渲染精度、写实质感）自动惠及编辑侧，无需分别优化两套模型。

### 🛠️ 操作流程

1. **准备阶段 — Prompt 构建**
   - 输入简短意图到任意 LLM（如 Qwen Chat 对话模式）
   - 要求 LLM 扩写为包含：背景色调、构图方式、文字内容（含双语对照）、字体风格、排版方向、光影描述的完整 Prompt
   - 控制扩写后 Prompt 在 1k token 以内

2. **核心执行 — 生图/编辑**
   - 文生图：将扩写后 Prompt 输入 chat.qwen.ai "生成图像" 功能或百炼 API
   - 图像编辑（题词）：上传原图 + 指令，例如：`在图片的左上角加上从右到左，从上到下写着的赵孟頫楷书"[诗词全文]"`
   - 双图合成：上传图1、图2，Prompt 中明确指定：两人站位关系、服装细节、背景统一来源（取图1或图2）、镜头参数（焦距/光圈/快门/ISO）
   - 跨次元编辑：上传真实照片（图1）+ 卡通形象（图2），Prompt 中明确：`请勿更改照片中的真实建筑、街道、车辆或人物，保持照片的真实性`，再描述卡通形象的位置关系

3. **验证与优化**
   - 检查文字渲染准确性（重点核查生僻字、长文本末尾部分）
   - 检查多介质场景中各介质的光影/透视是否自然
   - 若文字对齐/排版不理想，在 Prompt 中补充明确的对齐指令（如"居中对齐"、"自右向左竖排"）

### 💡 具体案例/数据

**案例1 — PPT 时间轴生成**：完整 Prompt 约 500 字，包含深蓝渐变背景、发光时间轴、两条支线（生图支线/编辑支线）、各节点日期（2025-05-06 至 2026-02-10）、节点内嵌套组图描述。模型输出的 PPT 直接用于官方发布文章，无人工后期。

**案例2 — A/B 测试信息图**：Prompt 包含三栏布局、具体数据（转化率 4.2% vs 5.1%，+21.4%；点击率 12.7% vs 14.9%，+17.3%；跳出率 58.1% vs 52.6%，-5.5pp；p=0.003；Cohen's d=0.32；n=42,500/组；28天；ROI 1:4.8），模型完整渲染所有数字、箭头、颜色标注。

**案例3 — 小楷压测**：《兰亭集序》全文（约 324 字）以王羲之小楷竖排渲染，结论：除极个别字外基本准确。

**案例4 — 双图合成**：两张同一东亚男性不同场景照片（米白衬衫+黑框眼镜+米色斜挎包 vs 北航学士服+毕业证书），合成为并肩站立于常春藤石墙前的自然合照，指定 50mm f/4.0 1/160s ISO 200 参数，无拼接痕迹。

### 📝 避坑指南

- ⚠️ 长文本渲染（如《兰亭集序》全文）存在极个别字错误，重要场景需人工校对
- ⚠️ 复杂 Prompt 直接手写效率低，务必先用 LLM 改写扩充，再输入模型
- ⚠️ 双图编辑时若不明确指定背景来源，模型可能自行混合两张图的背景，需在 Prompt 中显式声明"背景统一为图X中的[具体场景]"
- ⚠️ 跨次元编辑需显式声明"请勿更改照片中的真实元素"，否则模型可能对真实照片部分进行风格化处理
- ⚠️ API 当前为邀测阶段，生产环境接入需提前申请百炼资格
