### 📋 核心分析

**战略价值**: Seedance 2.0 将视频生成从"文字驱动"升级为"图+视频+音频+文本"四模态自由组合，核心突破在于「参考复刻」能力——可精准还原运镜、动作、特效、音色，让普通用户具备导演级创作控制力。

**核心逻辑**:

- **四模态输入上限**：图片 ≤9 张、视频 ≤3 个（总时长 ≤15s）、音频 ≤3 个（总时长 ≤15s）、文本自然语言；混合输入总文件上限 12 个
- **生成时长自由选**：4–15s 可选，视频延长时"生成时长"填写的是「新增部分」时长（如延长 5s 就选 5s，不是总时长）
- **两个入口区别明确**：仅首帧图+prompt → 走「首尾帧」入口；多模态组合输入 → 必须进「全能参考」入口；智能多帧和主体参考当前不可选
- **@素材指定语法是核心操作**：在 prompt 中用 `@图片1`、`@视频1`、`@音频1` 精确指定每个素材的用途，例如：`@图片1 为首帧，参考 @视频1 的打斗动作，@音频1 用于配乐`
- **参考视频可复刻运镜/动作/特效**：无需描述专业术语，直接上传参考视频 + 写清楚"参考 @视频1 的运镜"即可，模型识别镜头语言、动作节奏、视觉结构并精准复刻
- **视频延长支持连续叙事**：可在 prompt 中描述延长段落的具体画面内容，模型按提示生成衔接镜头，不只是机械续帧
- **视频编辑支持定向修改**：已有视频可直接作为输入，指定角色替换、动作调整、片段增删，无需重头生成
- **音频参考支持音色克隆**：上传参考视频后可在 prompt 中写"音色参考 @视频1"，模型复刻说话音色和语气
- **音乐卡点能力**：上传音频/含音乐的视频，prompt 中说明"参考 @视频1 的画面节奏进行卡点"，模型自动对齐关键帧节奏
- **有参考视频会额外计费**：使用视频参考功能比纯文/图生成成本更高，需合理规划素材使用

### 🎯 关键洞察

**@语法的精度决定生成质量**：素材越多，@指定越要清晰。混淆图、视频、角色是最常见失误。建议每次生成前检查 prompt 中每个 @对象是否都明确标注了用途（首帧/参考运镜/参考动作/配乐/音色等）。

**"参考"与"编辑"要在 prompt 中区分**：参考 = 模仿某素材的风格/动作/运镜；编辑 = 对已有视频进行修改。两者混用会导致模型理解偏差。

**一镜到底的实现逻辑**：在 prompt 中加入"全程不要切镜头，一镜到底"并配合多张场景图（@图片1~@图片5）按顺序描述镜头路径，模型会尝试连续追踪镜头穿越多个场景。

**视频融合的 prompt 结构**：`"我要在 @视频1 和 @视频2 之间加一个场景，内容为 xxx"` — 这是官方推荐的多视频合成写法。

### 🛠️ 操作流程

**1. 准备阶段**
- 确认素材类型和数量：图片（≤9）、视频（≤3，总时长≤15s）、音频（≤3，总时长≤15s），总文件≤12
- 判断入口：有多模态素材 → 全能参考；仅首帧图+prompt → 首尾帧
- 进入全能参考入口，点击上传按钮唤起本地文件弹窗，选定文件添加至输入框

**2. 核心执行（@素材指定两种方法）**
- 方法1：在输入框中输入 `@` 唤起参考调用弹窗 → 选择对应素材 → 素材名落入输入框 → 继续写 prompt
- 方法2：点击参数工具栏中的 `@` 按钮 → 选择参考 → 落入输入框 → 写 prompt
- Prompt 结构模板：
  ```
  @图片1 为首帧，@图片2 为尾帧，参考 @视频1 的运镜效果和动作节奏，
  @音频1 用于配乐。[具体画面描述，包含镜头语言、动作、场景、情绪]
  ```
- 选择生成时长（4–15s），延长场景选新增部分时长

**3. 验证与优化**
- 上传后悬停素材可预览图片/视频/音频，确认素材正确
- 生成后检查：运镜是否复刻准确、角色一致性、音画同步
- 若效果偏差：检查 @对象是否标注清晰，是否有图/视频/角色混淆；细化 prompt 中的镜头描述（景别、运镜方向、动作节点）

### 📝 避坑指南

- ⚠️ **延长视频时生成时长填错**：延长 5s 就选 5s，不是原视频+5s 的总时长
- ⚠️ **素材多时 @对象混淆**：图、视频、角色必须逐一标注用途，"@图片1 为首帧"和"@图片2 参考服装"要分开写清楚
- ⚠️ **有参考视频额外计费**：使用视频参考功能比纯文/图生成贵，合理规划是否必须用视频参考
- ⚠️ **全能参考入口才支持多模态**：误用首尾帧入口会导致视频/音频素材无法被正确解析
- ⚠️ **参考视频总时长限制**：3 个视频加起来不超过 15s，超出部分可能被截断或报错
- ⚠️ **"参考"和"编辑"语义要区分**：prompt 中要明确写"参考 @视频1 的运镜"（模仿）还是"将 @视频1 中的人物替换为..."（编辑），混用导致模型理解偏差
- ⚠️ **总文件上限 12 个**：优先上传对画面或节奏影响最大的素材，次要素材可用文字描述替代
