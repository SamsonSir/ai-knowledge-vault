### 📋 核心分析

**战略价值**: 用云雾 AI 的 Imagen API，自建 Streamlit 工具，彻底绕开 Gemini/NotebookLM 的梯子+账号+次数三重限制，实现本地/云端可控的生图与 PDF 幻灯片生成。

**核心逻辑**:

- **痛点根源**：Gemini Gems 生图、NotebookLM 幻灯片底层都是调用 Imagen（香蕉 Pro）API，谷歌只是套了一层 UI，限制了免费次数。有 API Key 就能绕过所有限制。
- **替代路径**：云雾 AI 平台提供 OpenAI 标准接口的 Imagen API，直接用 API Key 调用，无需梯子、无需谷歌账号、无次数上限（按量计费）。
- **工具选型**：前端用 Streamlit（Python Web UI 框架），AI 编程工具用 Trae，全程零手写代码，只需提需求 + 配置 API 参数。
- **单图生成**：直接在网页输入提示词，选手绘风格，调用 API 生成单张图，对标 Gemini Gems 生图效果。
- **PPT 生成两步走**：① 用 Cherry Studio + AI 员工生成 PPT 内容大纲；② 用另一个 AI 员工（PPT 大纲提示词）将大纲转换为结构化视觉提示词；③ 粘贴到工具网页，生成 PDF 版幻灯片。
- **提示词架构**：PPT 提示词采用"架构师（The Architect）"人设，强制 16:9 比例、三联画/网格布局、玻璃拟态风格，调色板固定为 `#F0F4F8` 背景 + `#6C5CE7` 主色 + `#FD79A8` 辅色。
- **本地项目结构**：两个 Python 脚本（单图脚本 + PPT 脚本），本地约 2 小时用 AI 编程完成；云端部署额外 2 小时，AI 全程指导。
- **API 配置极简**：云雾 AI 采用 OpenAI 标准，只需填入 API Base URL + API Key + 模型名称，AI 编程工具（Trae）无需额外说明即可理解并生成调用代码。
- **成本结构**：按张计费，5 张 PPT 约 1 元人民币，相比谷歌免费额度耗尽后的等待成本，灵活度极高。
- **迭代节奏**：先做最小可用版（本地单图）→ 追加 PPT 需求（30 分钟出第一版）→ 再追加云端部署需求，每次迭代都是独立对话驱动。

### 🎯 关键洞察

**为什么能替代官方工具**：
谷歌 Gemini 生图和 NotebookLM 幻灯片的核心能力来自 Imagen 模型，谷歌的 UI 层只是封装，并非独家优化。只要 API 能力相同，自建工具可以 100% 复刻效果。云雾 AI 提供的正是同源 API，且遵循 OpenAI 标准，接入成本极低。

**为什么用 Streamlit**：
Streamlit 是 Python 生态中最快的 Web UI 框架，无需前端知识，AI 生成的代码可以直接 `streamlit run app.py` 运行，适合 AI 编程场景下的快速验证和部署。

**PPT 提示词的关键设计逻辑**：
每张幻灯片提示词包含三层结构：
1. `NARRATIVE GOAL`（叙事目标）：告诉 AI 这张图要传达的情感和认知目的
2. `KEY CONTENT`（关键内容）：具体文字内容
3. `VISUAL + LAYOUT`（视觉+布局）：精确描述构图、色彩、元素位置

三层缺一不可，缺少叙事目标会导致图像风格漂移，缺少布局描述会导致排版混乱。

### 🛠️ 操作流程

**单图生成**

1. 准备阶段：注册云雾 AI，获取 API Key（官网有免费试用额度）
2. 核心执行：打开工具网页 `https://8.138.201.223:8501/`，输入 API Key，填写提示词，选择手绘风格，点击生成
3. 验证与优化：查看生成结果，不满意则修改提示词重新生成，无次数焦虑

**批量幻灯片生成**

1. 准备阶段：准备好 Cherry Studio 及两个自定义 AI 员工（参谋长 + PPT 大纲提示词）
2. 第一步：在 Cherry Studio 与参谋长对话，输入主题（如"AI 是什么？"），输出结构化 PPT 大纲（含每页标题、内容、emoji）
3. 第二步：将大纲完整粘贴给 PPT 大纲提示词 AI 员工，自动转换为包含 `NARRATIVE GOAL / KEY CONTENT / VISUAL / LAYOUT` 四层结构的视觉提示词
4. 第三步：将生成的完整提示词粘贴到工具网页，检查文字内容（可直接修改），点击生成
5. 验证与优化：下载 PDF，检查每张幻灯片效果，如需微调则修改对应页的提示词后重新生成

**本地自建项目**

1. 准备阶段：安装 Python + Streamlit，准备云雾 AI 的 API Base URL、API Key、模型名称
2. 核心执行：打开 Trae，描述需求（"用 Streamlit 做一个调用 Imagen API 的生图工具，API 采用 OpenAI 标准"），粘贴 API 配置信息，根据 AI 反馈多轮迭代
3. 追加 PPT 需求：在同一对话中追加"增加 PPT 幻灯片生成功能，对标 NotebookLM 效果"，约 30 分钟出第一版
4. 本地运行：`streamlit run app.py`（单图）或对应 PPT 脚本
5. 云端部署（可选）：继续在 Trae 中提云端部署需求，AI 全程指导，额外约 2 小时

### 📝 避坑指南

- ⚠️ 云雾 AI 模型名称必须从官网确认，不能随意填写，否则 API 调用报错
- ⚠️ PPT 提示词中每页的 `NARRATIVE GOAL` 不能省略，省略后图像风格会漂移，失去叙事连贯性
- ⚠️ 生成 PDF 前务必在网页检查文字内容，标题文字会直接渲染到幻灯片中，错别字无法事后修改
- ⚠️ 本地运行需要 Python 环境，Trae 生成代码后直接问 AI "如何运行"，它会给出完整的环境配置步骤
- ⚠️ 云端部署涉及服务器配置，建议先跑通本地版本再提云端需求，避免调试成本叠加
- ⚠️ 不同充值方案折扣不同，按量计费，PPT 生成成本取决于页数和重新生成次数
