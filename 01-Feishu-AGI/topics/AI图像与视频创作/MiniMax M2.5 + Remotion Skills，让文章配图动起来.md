### 📋 核心分析

**战略价值**: 用 MiniMax M2.5 + Remotion Skill，三步将文章静态逻辑图升级为可导出 GIF 的动态配图，全程无需手写代码。

**核心逻辑**:

- **动画比静态图多两个维度**：①先后顺序（节点依次弹入，读者无需猜起点）；②重点引导（放大/高亮/弹入效果主动抓注意力），静态箭头图做不到这两点。
- **技术路线选型**：Remotion 是代码渲染动画框架，输出结果是真实 GIF，不是 PPT 动画，可直接嵌入文章。
- **Skill 机制**：将 Remotion 操作规范封装成 `.claude/skills/remotion` 目录下的 Skill 文件，Agent 读取后自动知道弹性动画、循环播放、防闪烁缓冲等设计规范，无需每次重复指令。
- **M2.5 规划能力提升**：旧模型「想到哪写到哪」，M2.5 能先输出完整任务拆解 + 技术方案，再逐步执行，整个项目（拆任务→定方案→实现）全由它自主规划。
- **M2.5 调试能力提升**：旧模型遇报错盲目乱改，M2.5 能准确定位问题根源并给出有效修复方案，不再反复试错。
- **M2.5 代码质量**：一次生成即可跑通的概率高；对 Remotion 这类小众框架也能输出符合最佳实践的代码结构。
- **工作流完全由 Agent 驱动**：不只写代码，连 bug 定位、渲染、导出 GIF 全链路均由 M2.5 完成，人只需描述需求 + 确认 ASCII 草图。
- **ASCII 草图作为中间确认层**：Agent 先画 ASCII 草图让人确认节点内容和关系是否正确，再进入渲染阶段，避免方向跑偏后再返工。
- **一次性安装，永久复用**：Skill 只需克隆一次，后续所有项目直接调用 `/remotion` 指令即可。
- **适用场景**：循环流程图、卡片网格、线性流程图等逻辑图类型均已验证可行。

### 🛠️ 操作流程

**第 1 步：安装 Skill（一次性）**

发给 Agent：
```
帮我把 https://github.com/Ceeon/remotion-skill 克隆到 .claude/skills/remotion 目录下
```
Agent 自动执行，完成后无需再操作。

---

**第 2 步：描述目标图形**

发给 Agent（示例）：
```
画一个循环流程图：AI识别 → 你审核 → 反馈学习，三个节点形成循环
```

Agent 返回 ASCII 草图：
```
      🤖 AI识别
       ╱    ╲
      ╱      ╲
  👤 你审核 ── 📋 反馈学习
```
确认节点内容和关系无误后进入下一步。

---

**第 3 步：生成 GIF**

发给 Agent：
```
/remotion 把上面的 ASCII 做成动画 GIF
```

Agent 自动完成：生成动画代码 → 渲染 → 导出 GIF，全程无需手动干预。

### 📝 避坑指南

- ⚠️ ASCII 草图确认步骤不要跳过，节点关系错了进渲染阶段再改成本高。
- ⚠️ Skill 目录路径 `.claude/skills/remotion` 必须严格对应，路径错误导致 `/remotion` 指令失效。
- ⚠️ 旧模型（非 M2.5）调试能力弱，遇到 Remotion 报错容易越改越乱，建议锁定 M2.5。
