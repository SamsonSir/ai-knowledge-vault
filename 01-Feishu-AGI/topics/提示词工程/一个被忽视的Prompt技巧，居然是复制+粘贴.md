### 📋 核心分析

**战略价值**: Google论文实证：对非推理模型，把Prompt原文重复一遍（Ctrl+C/V），可将准确率从21%飙升至97%，70组测试47胜0负，零成本、零门槛。

**核心逻辑**:

- **论文出处**: Google新论文《Prompt Repetition Improves Non-Reasoning LLMs》，核心结论：重复输入提示词，显著提升非推理类大模型准确率。
- **测试规模**: 7个主流非推理模型（Gemini 2.0 Flash、Flash Lite、GPT-4o、GPT-4o-mini、Claude 3 Haiku、Claude 3.7 Sonnet、DeepSeek V3），全部使用官方API，覆盖ARC、OpenBookQA、GSM8K、MMLU-Pro、MATH等标准测试集，外加自研NameIndex、MiddleMatch两个测试。
- **核心数据**: 70组「原始Prompt vs 重复一遍Prompt」对比，重复版赢47次，平23次，0次失败。某些任务准确率从21.33%→97.33%。
- **操作方法极简**: 原来发 `<问题>`，现在发 `<问题><问题>`，中间不加任何解释、不加please、不加think step by step、不贴示例，就是原文再粘一遍。
- **为什么有效（机制）**: 大模型是"因果语言模型"，从左到右逐token预测，当前token只能看到之前的内容，无法回头。第一遍读到选项A/B时，还不知道后续的场景条件，表征很弱。把整段重复后，第二遍的A/B在生成时，已经能"看到"第一遍的完整上下文（包括场景条件），等于给模型一次「回头重读」的机会，选项表征更准确，输出更正确。
- **适用范围：非推理模型专属**: DeepSeek V3（非推理）有效，DeepSeek R1（推理模型，有思维链）无效。原因：推理模型在RL微调阶段已自学了这个技巧——它们回答时会自动先复述题目（"题目问的是……"、"我们需要求解的是……"），本质上已经在自动重复一遍Prompt。
- **重复次数可扩展**: 论文衍生实验显示，重复三遍效果同样提升，并非只限于重复一次。
- **自研测试设计**: NameIndex——给模型50个名字的列表，问第25个是谁；MiddleMatch——给含随机重复名字的列表，问两个字符之间的名字是什么。这两个测试专门考验模型对列表中间位置信息的定位能力，正是"回头看"能力最弱的场景。
- **论文指出的未来方向**: ①将重复提示写入模型预训练/微调流程；②推理阶段只在KV cache保留第二遍提示，性能不受影响；③只重复Prompt的一部分而非全文；④多模态场景（图像、视频）中的重复实验。

### 🎯 关键洞察

**为什么非推理模型在"中间信息"上特别容易出错？**

因果语言模型的注意力机制决定：越靠前的token，在后续生成时被"稀释"得越厉害。当题目结构是「选项在前、条件在后」时，模型读到选项时没有条件信息，初始表征质量低；读到条件时，选项的表征已经固化，无法更新。重复整段后，第二遍选项的每个token在计算attention时，可以直接attend到第一遍的完整条件，相当于强制做了一次「带条件的二次编码」。

**推理模型为何免疫？**

推理模型（如R1、o1系列）通过RL训练，已经内化了「先复述题目再作答」的行为模式。它们的思维链开头几乎必然包含对题目的重述，这在功能上等价于Prompt重复，所以外部再重复一遍收益边际极低。

### 🛠️ 操作流程

1. **判断模型类型**: 确认你用的是非推理模型（无思维链、无"深度思考"开关）。如果模型回答前会出现"正在思考…"或明显的推理过程，跳过此技巧。

2. **核心执行**: 将你的完整Prompt原文复制一遍，直接拼接在原Prompt末尾，中间无需任何分隔符、说明或额外指令。
   ```
   # 原来的写法
   梵蒂冈圣伯多禄大教堂门口有几根柱子？

   # 使用重复技巧后
   梵蒂冈圣伯多禄大教堂门口有几根柱子？梵蒂冈圣伯多禄大教堂门口有几根柱子？
   ```
   对于长Prompt（含选项+条件的结构），整段复制：
   ```
   选项：
   A. 把蓝色方块放到红色方块左边
   B. 把红色方块放到蓝色方块左边
   场景说明：现在红色在左，蓝色在右。
   问题：哪一个选项会改变画面？只输出A或B

   选项：
   A. 把蓝色方块放到红色方块左边
   B. 把红色方块放到蓝色方块左边
   场景说明：现在红色在左，蓝色在右。
   问题：哪一个选项会改变画面？只输出A或B
   ```

3. **进阶尝试**: 重复三遍（Q1Q2Q3）效果同样提升，可在准确率要求极高的场景下使用。不需要修改任何其他Prompt结构。

### 💡 具体案例/数据

- **NameIndex测试**: 给模型50个名字的列表，问第25个是谁。这类"中间位置定位"任务是非推理模型的典型弱点，重复后准确率提升最为显著。
- **MiddleMatch测试**: 给含随机重复名字的列表，问两个特定字符之间的名字。同样考验中间信息提取，重复后效果明显改善。
- **极端案例**: 某任务准确率从21.33%→97.33%，提升幅度76个百分点，仅靠Ctrl+C/V实现。
- **整体统计**: 70组测试，47胜23平0负，胜率67%，无一失败。

### 📝 避坑指南

- ⚠️ **推理模型上用了没效果不要怀疑自己**：R1、o1、Claude Thinking等推理模型已内化复述机制，重复Prompt对它们基本无收益，这是正常现象。
- ⚠️ **不要在重复之间加任何额外文字**：论文的测试是纯粹的`<Q><Q>`结构，加了"请再想想"、"重新考虑"等文字后，变成了另一种Prompt技巧，效果不可类比。
- ⚠️ **速度不受影响，但token消耗翻倍**：重复一遍意味着输入token数×2，在API调用量大的场景下需注意成本。
- ⚠️ **对短问题效果最直接**：越是「选项/条件分离」、「关键信息在中间」的结构，收益越大；纯粹的开放式长文生成任务收益相对有限。
