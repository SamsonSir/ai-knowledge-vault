### 📋 核心分析

**战略价值**: 用 TELOS 10文件体系 + 三层记忆架构 + Hooks/Skills，把 AI 从无状态对话工具升级为持久化、可编程的个人基础设施。

**核心逻辑**:

- **范式转变的本质**：当前 AI 是"自动售货机"（无状态、问答即止）；PAI 要把它变成"操作系统"（持久运行、记住设置、管理文件）。两者的根本差异在于：有无跨会话的持久状态。
- **TELOS = 10个 Markdown 文件构成的数字身份档案**，覆盖：核心身份(00)、价值观(01)、背景(02)、技能(03)、沟通风格(04)、目标(05)、知识领域(06)、工作方式(07)、关系与背景(08)、兴趣(09)。每个文件独立维护，持续迭代，不是一次性问卷。
- **数据主权设计**：TELOS 全部用 Markdown 存本地，可用 Git 做版本控制（`git diff` 追踪自我认知演变），不锁定任何平台，随时迁移。
- **三层记忆架构模拟人脑**：① 短期记忆（当前对话上下文，实时动态）；② 情景记忆（具体对话记录和项目历史，可精确回溯）；③ 语义记忆（从经验提炼的模式和偏好，如"你通常用函数式风格"）。检索机制用向量数据库 + 相似度搜索，实现联想式调取，而非线性翻记录。
- **Hooks = 给 AI 装传感器**：在特定事件（提问前、回复后、工具调用时）自动触发脚本，让 AI 主动感知工作环境，而非被动等待输入。三类典型 Hook 见下方配置表。
- **Skills = 可组合的命令式能力插件**：每个 Skill 是命名化、参数化的 Prompt 模板，通过 `/skill-name` 调用，Skills 之间可相互调用构建复杂工作流，可在团队间共享复用，类比 VS Code 扩展市场。
- **工程三原则**：① 确定性（所有配置文件化，行为可追溯可回滚）；② 模块化（TELOS/Memory/Hooks/Skills/Tools 各自独立可替换）；③ 可追溯性（Markdown + YAML + JSON Lines 全开放格式，`grep` 可搜，任何文本工具可处理）。
- **数据流是明确的单向管道**：用户输入 → Hooks预处理 → TELOS加载 → Memory检索相关上下文 → 构建完整Prompt → 调用LLM → Skills执行 → Memory存储新交互 → Hooks后处理 → 返回响应。每步可配置、可观察。
- **存储方案全部开放格式**：TELOS 用 Markdown、对话历史用 JSON Lines、向量库用 SQLite+向量扩展或专用向量库、配置用 YAML。
- **冷启动是最大门槛**：需要填写10个 TELOS 文件（深度自我反思）+ 积累交互历史（时间投入）+ 配置 Hooks/Skills（技术门槛）。类比搭建个人知识管理系统：前期痛苦，长期回报。

### 🎯 关键洞察

**为什么 TELOS 用10个独立文件而不是一个大文档？**
模块化设计：每个维度独立更新，不互相污染。你的技能(03)提升了，不需要重写价值观(01)。Git 追踪时 diff 粒度更细，能清晰看到"这个月我的目标(05)发生了什么变化"。

**为什么三层记忆比单一聊天记录强？**
原因：人类大脑不是录像机，是压缩机。逐字存储所有对话会导致上下文爆炸（更慢、更贵）且信噪比极低。
动作：语义记忆层做提炼压缩，只保留模式和偏好；情景记忆保留关键事件；短期记忆处理当下。
结果：检索时精准调取相关片段，而非加载全部历史，在"完整上下文"和"效率"之间取得平衡。

**为什么选 CLI 优先而非 GUI？**
可脚本化 → 便于与 Hooks 集成；本地存储 → 隐私和控制权；模型无关 → 支持 OpenAI、Anthropic、本地模型，不绑定单一厂商。

### 🛠️ 操作流程

1. **准备阶段**:
   - 安装工具：Claude Code（或国内可用 AI）+ Obsidian（免费）
   - 克隆开源项目：`https://github.com/danielmiessler/Personal_AI_Infrastructure`
   - 在本地创建 TELOS 目录，初始化 Git 仓库（`git init`）
   - 依次创建并填写 10 个 TELOS 文件（`00` 至 `09`），每个文件聚焦单一维度，用 Markdown 自由书写，不求完美，求真实

2. **核心执行**:
   - 将 TELOS 文件路径配置到 AI 工具的系统提示或 `Claude.md` 中，使每次对话自动加载
   - 根据实际工作流编写 Hooks 脚本（Shell 脚本或可执行文件），挂载到对应事件节点
   - 从社区获取或自行编写常用 Skills（Markdown 文件定义），放入 Skills 目录
   - 配置向量数据库（SQLite+向量扩展起步），开启情景记忆和语义记忆的自动存储

3. **验证与优化**:
   - 进行几次真实工作对话，验证 AI 是否正确加载 TELOS 上下文
   - 用 `git diff` 检查 TELOS 文件变更，确认版本控制正常
   - 检查 `~/logs/ai-activity.log` 确认 Hooks 正常触发
   - 定期（建议每月）回顾并更新 TELOS 文件，特别是 `05-goals.md` 和 `03-skills.md`
   - 制定记忆管理策略：决定什么该长期保留、什么该归档、如何处理过时信息

### 💡 具体案例/数据

**案例1：长期项目管理**
- `05-goals.md` 记录项目目标 → Memory 系统记住所有设计决策 → `/project-status` Skill 快速生成进度报告 → Hooks 自动同步 Git 状态和 TODO 列表
- 效果：AI 成为真正的项目成员，无需每次重新解释背景

**案例2：学习新技术栈**
- 每次学习对话记录进 Episodic Memory → 重要概念自动提炼到 Semantic Memory → `06-knowledge-domains.md` 随学习更新 → `/review-learning` Skill 定期复习
- 效果：AI 从"答题机器"变成"学习伙伴"

**案例3：团队协作**
- 导出脱敏版 TELOS 配置供团队成员导入 → Skills 在团队间共享 → Memory 系统记录团队共识
- 效果：AI 从个人工具升级为团队基础设施

**案例4：创意工作（作家/设计师/创业者）**
- `04-communication-style.md` 定义语言风格 → `01-values.md` 确保建议符合价值观 → Memory 记住创意演化过程 → `/brainstorm` Skill 基于历史产生新想法
- 效果：AI 理解创意意图，不再生成通用内容

### 📝 避坑指南

- ⚠️ **冷启动成本被低估**：10个 TELOS 文件不是填表，是深度自我反思。建议分批完成，先填 `00`（核心身份）和 `05`（目标），其余逐步补充，不要试图一次性完成。
- ⚠️ **记忆无限增长问题**：向量库和对话历史会持续膨胀，导致检索变慢、Prompt 变长变贵。必须提前设计遗忘/归档策略，例如超过90天的情景记忆自动降级为摘要。
- ⚠️ **TELOS 信息过时**：目标变了、技能提升了，但文件没更新，会导致 AI 基于错误画像给建议。建议每月 `git diff` 一次，强制回顾。
- ⚠️ **多设备同步是技术门槛**：本地存储保证隐私，但跨设备同步需要自行解决（Git 私有仓库 + 加密是推荐方案）。
- ⚠️ **Hooks 脚本路径硬编码**：示例中 `~/work/status.json` 等路径需按实际环境调整，否则 Hook 静默失败，AI 拿不到上下文。
- ⚠️ **Skills 质量参差不齐**：社区 Skills 需要审查后再用，Prompt 模板质量直接决定输出质量，建议先在沙盒环境测试。
