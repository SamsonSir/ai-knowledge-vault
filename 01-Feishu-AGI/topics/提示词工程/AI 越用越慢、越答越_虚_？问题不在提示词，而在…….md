### 📋 核心分析

**战略价值**: AI 变慢变笨的根因是上下文膨胀，解法是把"聊天"改造成"阶段交付系统"。

**核心逻辑**:

- **根因不是提示词，是上下文成本**：每发一条新消息，AI 要重新读取整段历史。对话越长，处理成本越高，速度越慢，准确率越低。
- **"Lost in the Middle"效应**：研究证明，当关键信息处于长文本中间位置时，模型更容易忽略它——即便内容确实存在于对话里。结果是：你给得越多，它越抓不住重点。
- **症状识别三连**：回复变慢 + 细节开始丢 + 容易答非所问 = 上下文已过载，必须交接。
- **安全字数阈值约 2～3 万字**：官方说百万 Token，但实际深度使用中，超过这个量就会出现质量下滑，这是经验值而非官方硬规定。
- **分阶段开新对话**：每个阶段只做一个交付物，完成即开新对话，避免"对话垃圾堆"。
- **交接摘要是核心资产**：把对话内容转化为结构化交付物，而不是让它停留在聊天记录里。
- **AI 分工协作**：不同工具按特长分配任务，减少主对话膨胀，避免查资料污染写作风格。
- **查证隔离原则**：查资料单开对话，只把"结论 + 一句话解释 + 来源"带回主对话。
- **一次生成多版本**：让 AI 一次输出 A/B/C 三版，避免反复重读历史对话。
- **个人设定档**：每次开新对话时贴入"AI 使用说明书"，让 AI 无需依赖旧对话记忆就能迅速进入状态。

### 🎯 关键洞察

**为什么对话越长越出错？**

原因：AI 每次响应都要重新处理整段历史上下文，长度越长，注意力越分散，中间段内容被忽略概率越高（Lost in the Middle 研究支撑）。

动作：主动在 2～3 万字前做交接，开新对话。

结果：AI "恢复智商"，速度和准确率立刻提升。

**为什么要把 AI 当"工作系统"而不是"聊天工具"？**

聊天模式 → 对话越来越长 → 成本高、质量低、无法复用。
系统模式 → 任务拆分 + 阶段交付 + 交接摘要 + 资产沉淀 → 同样额度产出翻倍。

### 🛠️ 操作流程：四段式汇报材料生产 SOP

**对话 A：材料清洗（只做"去噪"）**

```
我现在有一批非常杂乱的原始材料，包括会议录音转文字、群聊记录和个人笔记。
这一步请你只做一件事：材料清洗，不要写方案、不要总结成文章。
请帮我完成以下输出：
1、去掉口语、重复、无关内容
2、提取清晰的【事实信息】和【已发生的动作】
3、标出【关键结论】（如果有）
4、列出【待解决问题 / 待决策事项】
输出请用条列式，保持"干净、客观、不加发挥"。
附件是原始材料：（上传附件文件、图片等）
```

输入：会议录音文字、聊天记录、散乱笔记
输出：干净纪要、关键结论、待办清单

---

**对话 B：结构搭建（只做"骨架"）**

```
我将给你一份已经清洗过的材料摘要。
这一步请你只做一件事：结构搭建，不要写成完整文章。
任务目标：这是一次给领导的「项目阶段汇报 + 下阶段建议」
请你输出：
1、一个清晰的汇报大纲（3–5 个模块）
2、每个模块下，只列【要点标题 + 支撑要点】
要求：结构优先，不展开成段落，不润色语言
下面是材料摘要：（把对话 A 的回答粘贴在这里）
```

输入：A 的结果
输出：汇报大纲（3-5块）、每块观点与支撑点

---

**对话 C：表达成稿（只做"写得好"）**

```
我现在有一份已经确认的大纲结构。
这一步请你只做一件事：把它写成一份可以直接发给领导的成稿。
写作要求：
语气：专业、克制、偏汇报
风格：结论前置，逻辑清晰
不夸张、不口号化
输出目标：一份无需我再改结构、可以直接发送的汇报稿
下面是大纲：（粘贴对话 B 的答案）
```

输入：B 的结果
输出：可直接发给领导/客户的成稿

---

**对话 D：外部核对（只做"查证"）**

```
我有一份即将提交的汇报稿。
这一步请你只做"核对与查证"，不要改写全文。
请你重点检查：
1、是否存在表述过于绝对的判断
2、是否有需要补充依据的数据或案例
3、是否有容易被领导质疑的逻辑跳跃
输出要求：用「问题点 + 建议修正方式」列出，不要直接重写正文
下面是成稿内容：（粘贴对话 C 答案）
```

输入：需要核实的术语、数据、案例
输出：核对后的信息，再带回 C 修改

### 💡 核心 Prompt 模板库

**万能交接摘要（对话快到 2～3 万字时触发）**

```
我要准备开启一个新的对话了。请把我们当前工作整理成一份【交接摘要】，必须包含：
1）当前进度：已经完成了什么（条列）
2）关键结论：最重要的 5 条判断
3）待办事项：下一步行动项（按优先级排序）
4）输出规范：格式、语气、受众、禁忌点
5）可复用内容：可以直接复制的段落/模板
请用清晰条列式输出，确保我能粘贴到新对话继续工作。
```

---

**个人设定档（每次开新对话时贴入）**

```
我是一名职场人，常用 AI 帮我写：会议纪要、方案、汇报稿、邮件。
输出要求：逻辑清晰、条理化、尽量短句、少废话，语气专业不油腻。
格式要求：先给结论，再给步骤，最后给可复制模板。
禁忌：不要夸张、不喊口号、不要过度鸡汤。
目标读者：25-35岁上班族。
以后我发材料，请你按这个标准输出。
```

---

**一次生成多版本（避免反复重读历史）**

```
请一次性给我 A/B/C 三个版本：分别偏正式、偏简洁、偏有说服力。每版不超过 120 字。
```

### 📝 避坑指南

- ⚠️ 不要在同一对话里从背景聊到结案，全程一个对话是最大的效率杀手
- ⚠️ 官方说的百万 Token 上下文是理论值，实际 2～3 万字后质量就开始下滑
- ⚠️ 让 AI 改某一处时，它可能悄悄改动其他地方——对话越长这个问题越严重
- ⚠️ AI 对话不是存储系统，是临时草稿纸，不备份就是在赌运气
- ⚠️ 查资料和写作混在同一对话，会污染写作风格，且让上下文无谓膨胀
- ⚠️ 感觉 AI 开始变慢时，先别继续问——先输出交接摘要、备份文档，再开新对话
