### 📋 核心分析

**战略价值**: 用一套经过实战验证的 Bento Grid 提示词 + Gemini 2.5 Pro，将任意文档（如财报）一键生成高颜值动态网页，并可进一步转为 PPT 设计稿。

**核心逻辑**:

- **触发场景**：用 Gemini Deep Research 生成特斯拉 Q1 财报分析文档 → 直接喂给提示词 → 生成完整动态网页，全程无需手写代码
- **模型选择关键**：Claude 3.7 能生成卡片但不能完整理解 Bento Grid 核心理念（多模态理解弱 + 知识截止问题）；Gemini 2.5 Pro 多模态能力强，一个词即可准确还原风格，无需垫图
- **Bento Grid 定义**：源自日语「弁当」（便当盒），所有内容装入不同大小的卡片，卡片相互组合排列，苹果 PPT 常用此风格，参考站：`https://bentogrids.com/`
- **提示词核心设计逻辑**：黑底 + 主题高亮色 + 超大字体数字形成视觉反差 + 中英文混排 + Apple 官网滚动动效 + 勾线图形数据可视化
- **技术栈固定组合**：HTML5 + TailwindCSS 3.0（CDN）+ Framer Motion（CDN）+ Font Awesome / Material Icons（CDN）+ 在线图表组件
- **垫图技巧与代价**：可附设计稿图片让模型学习更细分的 Bento 子风格，但图片会消耗上下文窗口，导致生成代码量和内容量减少，需权衡
- **最佳使用姿势**：将所有内容整理进一个文档，文档 + 提示词同时发给模型，不要分开发送
- **Gemini App 的坑**：Gemini App 内模型会「偷懒」，读取信息不完整；推荐用 AI Studio 或通过 API Key 接入 Cursor / Chatwise 使用
- **网页 → PPT 转化路径**：生成网页后用 Figma 插件转设计稿，Bento Grid 天然适配 PPT 版式，稍作调整即可直接使用

---

### 🎯 关键洞察

Gemini 2.5 Pro 在网页生成场景的核心优势不是代码能力，而是**多模态理解能力**——它能从语义层面理解「Bento Grid」这个设计概念并正确还原，而 Claude 只能理解「有卡片」但无法把握「全卡片 + 大小反差」的核心要素。这意味着：**设计类提示词的效果上限，取决于模型对视觉概念的理解深度，而非单纯的代码生成能力。**

### 🛠️ 操作流程

1. **准备阶段**:
   - 准备好内容文档（可用 Gemini Deep Research 生成，也可自己整理）
   - 访问 `https://aistudio.google.com/prompts/new_chat`，选择 Gemini 2.5 Pro 模型
   - 或点击「Get API Key」创建密钥，接入 Cursor / Chatwise 使用

2. **核心执行**:
   - 将内容文档 + 以下提示词一起发给模型：

```
帮我将这个[主题]文档生成网页，不要遗漏信息
根据上面内容生成一个 HTML 动态网页
1. 使用Bento Grid风格的视觉设计，纯黑色底配合[主题色]颜色作为高亮
2. 强调超大字体或数字突出核心要点，画面中有超大视觉元素强调重点，与小元素的比例形成反差
3. 中英文混用，中文大字体粗体，英文小字作为点缀
4. 简洁的勾线图形化作为数据可视化或者配图元素
5. 运用高亮色自身透明度渐变制造科技感，但是不同高亮色不要互相渐变
6. 模仿 apple 官网的动效，向下滚动鼠标配合动效
8. 数据可以引用在线的图表组件，样式需要跟主题一致
9. 使用 Framer Motion （通过CDN引入）
10. 使用HTML5、TailwindCSS 3.0+（通过CDN引入）和必要的JavaScript
11. 使用专业图标库如Font Awesome或Material Icons（通过CDN引入）
12. 避免使用emoji作为主要图标
13. 不要省略内容要点
```

   - 如需学习特定 Bento 子风格，可额外附上 2-3 张参考设计图（注意：图片越大，生成内容越少）

3. **验证与优化**:
   - 检查网页是否完整呈现文档所有要点（提示词第13条约束）
   - 如内容缺失，重新发送时减少附图或拆分文档分批生成
   - 需要转 PPT：将生成的 HTML 网页用 html.to.design 或腾讯 CoDesign 插件导入 Figma，微调后直接用作 PPT

### 📝 避坑指南

- ⚠️ **Gemini App 会偷懒**：直接在 Gemini App 对话框使用，模型读取文档信息不完整，生成网页内容会缺失，优先用 AI Studio 或 API
- ⚠️ **垫图会压缩代码量**：附参考图片会占用上下文，图片越多越大，生成的 HTML 代码越短、内容越少，非必要不垫图
- ⚠️ **高亮色渐变规则**：只允许同一高亮色做透明度渐变（如红色从 100% 到 0%），严禁两种高亮色互相渐变，否则破坏科技感
- ⚠️ **Claude 不适合此提示词**：Claude 3.7 对 Bento Grid 理解不完整，生成结果「有卡片但不全是卡片」，效果明显弱于 Gemini 2.5 Pro
- ⚠️ **内容必须提前整合**：不要边聊边发内容，把所有素材整理成一个完整文档后一次性提交，避免模型遗漏信息
