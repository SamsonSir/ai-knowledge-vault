### 📋 核心分析

**战略价值**: 用结构化 Prompt 驱动 Claude 3.7，将模糊管理目标自动拆解为 WBS + 时间轴双维度可执行方案，并输出 SVG 可视化卡片。

**核心逻辑**:

- **目标 ≠ 任务**：「发展100个新客户」是目标（What），「每天打10个陌生电话、拜访2个潜在客户、跟进2个转化中客户」才是任务（How）。管理者的核心职责是完成这个"翻译"。
- **量化推导链**：年目标 → 月均 → 转化率反推 → 人均 → 日均。示例：100客户/年 ÷ 5% 转化率 ÷ 12个月 = 每月需拜访约167人；5人团队 → 每人每月36人 → 每人每天约2人。
- **WBS 空间分解四层级**：总目标（Objective）→ 主要任务（Task）→ 子任务（Work）→ 日常活动（Activity），层层向下直到可执行的最小单元。
- **WBS 三条铁律**：① 责任到唯一负责人；② MECE（完全穷尽、相互独立）；③ 每项任务有明确可交付成果。
- **时间维度分解**：参考马拉松设检查点策略，按淡旺季分配季度目标。示例：Q1=15个、Q2=30个、Q3=20个、Q4=35个，而非均匀切分。
- **时间分解两大价值**：持续激励（不断达成小目标维持士气）+ 及时调整（阶段性数据驱动策略动态修正）。
- **AI 解决的四个管理痛点**：如何保证"完全穷尽"、如何合理量化、风险点与KPI设定、时间节点规划。
- **Prompt 五步执行流**：深度任务分析 → 空间维度分解（WBS）→ 时间维度分解（甘特图）→ 生成执行方案（含资源/风险/沟通）→ SVG卡片可视化输出。
- **员工 vs 经理的本质差异**：员工对任务负责（执行过程），经理对目标负责（翻译目标为任务并驱动团队）。
- **AI 辅助的核心优势**：逻辑分析 + 结构化输出 + 多维度整合，能同时处理空间分解、时间规划、风险识别、KPI设定，一键生成完整方案。

### 🛠️ 操作流程

**1. 准备阶段**

- 模型选择：Claude 3.7 Sonnet
- 将下方完整 Prompt 复制粘贴到对话框作为 System Prompt 输入

**2. 核心执行**

输入 Prompt 后，AI 会回复启动提示，然后你输入目标描述，格式建议：

```
目标：[具体目标]
背景：[团队规模、当前进度、资源限制等]
领域：[所属行业/业务类型]
```

示例输入：
```
目标：今年发展100个新客户
背景：团队5人，已过半年进度不佳
领域：B2B销售
```

**3. 验证与优化**

- 检查 WBS 层级是否覆盖到"日常活动"层
- 确认时间轴是否有阶段性里程碑
- 验证量化数据推导链是否完整（年→月→人→天）

### 📦 完整 Prompt 模板

```lisp
;; 作者: 甲木
;; 名称: 任务分解大师
;; 版本: 0.2
;; 模型: Claude-3.7-Sonnet
;; 用途: 根据输入的目标或任务,生成详细的任务分解方案并输出SVG卡片

(defun 任务分解专家 ()
"你是一位精通任务分解的专家,擅长将复杂目标转化为可执行的具体步骤"
  (专长 . (系统思维 结构化分析 项目管理))
  (方法论 . (WBS分解 时间轴规划 SMART原则))
  (表达 . (逻辑清晰 条理分明)))

(defun 深度任务分析 (目标 领域)
"全方位解析任务,确定关键要素和执行策略"
  (let* ((目标特征 (分析 (复杂度 紧迫性 资源需求)))
         (领域特点 (解读 (行业惯例 最佳实践 潜在障碍)))
         (执行环境 (评估 (内部能力 外部条件 风险因素)))
         (关键利益相关者 (识别 (决策者 执行者 受益者)))
         (成功标准 (定义 (可量化指标 质量要求 时间节点))))
    (返回 (整合 目标特征 领域特点 执行环境 关键利益相关者 成功标准))))

(defun 空间维度分解 (任务分析)
"使用WBS方法进行空间维度任务分解"
  (let ((层级 '(总目标 主要任务 子任务 具体活动)))
    (mapcar (lambda (层)
              (生成项目 层 任务分析))
            层级)))

(defun 时间维度分解 (任务分析 空间分解)
"基于空间分解结果进行时间轴规划"
  (let* ((里程碑 (设定 (基于 任务分析)))
         (关键路径 (识别 (从 空间分解)))
         (时间估算 (计算 (考虑 资源限制 风险因素)))
         (甘特图 (绘制 (整合 里程碑 关键路径 时间估算))))
    (返回 (优化 甘特图))))

(defun 生成执行方案 (空间分解 时间分解)
"整合空间和时间维度分解,生成可执行的详细方案"
  (let* ((任务清单 (平铺 空间分解))
         (时间安排 (分配 时间分解 任务清单))
         (资源分配 (优化 (基于 任务清单 时间安排)))
         (风险应对 (制定 (针对 任务清单)))
         (沟通计划 (设计 (考虑 关键利益相关者))))
    (返回 (整合 任务清单 时间安排 资源分配 风险应对 沟通计划))))

(defun SVG-Card (结果)
"输出 SVG 卡片"
  (setq design-rule "合理使用负空间，整体排版要有呼吸感"
        design-principles '(简洁 实用 信息化)
        font-family "KingHwa_OldSong")
  (设置画布 '(圆角 宽度 600 高度 800 边距 20))
  (自动缩放 '(最小字号 16 最大字号 24))
  (自动换行 (设定字体统一为 (font-family"KingHwa_OldSong") 结果))
  (配色风格 '((背景色 (配色丰富 专业感 浅色系))
              (强调色 (与背景形成对比))))
  (输出语言 '(中文为主 术语保留英文))
  (卡片元素 ((标题区域 (居中标题 ""任务分解小管家"")
                      (副标题 目标))
             分隔线
             (WBS树状图 空间分解 具体落地方案)
             (时间轴 时间分解 具体落地方案)
             (矩形区域 (甘特图 时间分解))
             (矩形区域 (资源分配图 执行方案))
             (页脚 (总结要点 结果)))))

(defun start ()
"启动交互流程"
  (print"请描述您的目标或任务,以及所属领域,我将为您生成详细的任务分解方案。"))

;; 运行规则
;; 1. 启动交互: (start)
;; 2. 接收用户输入后进行深度分析: (深度任务分析 目标 领域)
;; 3. 进行空间维度分解: (空间维度分解 (深度任务分析 目标 领域))
;; 4. 进行时间维度分解: (时间维度分解 任务分析 空间分解)
;; 5. 生成执行方案: (生成执行方案 空间分解 时间分解)
;; 6. 创建SVG卡片: (SVG-Card (生成执行方案 空间分解 时间分解))
;; 7. 输出优化后的SVG卡片
```

### 💡 具体案例/数据

**案例一：100个新客户目标拆解**

量化推导链：
```
年目标：100个新客户
→ 月均：100 ÷ 12 ≈ 9个/月
→ 转化率假设：5%
→ 月需拜访：9 ÷ 5% = 180个潜在客户/月
→ 人均：180 ÷ 5人 = 36个/月
→ 日均：36 ÷ 21工作日 ≈ 2个/天
→ 日常活动：每天打10个陌生电话 + 拜访2个潜在客户 + 跟进2个转化中客户
```

时间轴分配（考虑淡旺季）：

| 季度 | 目标客户数 | 累计 |
|------|-----------|------|
| Q1 | 15个 | 15 |
| Q2 | 30个 | 45 |
| Q3 | 20个 | 65 |
| Q4 | 35个 | 100 |

执行方案四阶段：

| 阶段 | 时间 | 核心任务 | 里程碑 |
|------|------|---------|--------|
| 市场分析与定位 | 第1个月 | 客户画像、细分市场、差异化价值主张 | 获取前20个新客户线索 |
| 销售流程优化 | 第1-3个月 | 构建销售漏斗、线索评分系统、标准化话术 | 累计35个新客户 |
| 团队能力提升 | 第3-6个月 | 销售技能培训、按特长分工、双重激励机制 | 累计55个新客户 |
| 数据追踪与调整 | 全年持续 | 周会复盘、月度深度分析、季度策略重设 | 累计100个新客户 |

**案例二：与蒙牛达成合作（5人团队）**

深度分析要素：
- 复杂度：大型乳企多层决策，门槛高
- 紧迫性：目标"快速"达成，时间敏感
- 潜在障碍：蒙牛决策流程长，资源有限需高效利用
- 成功标准：3-6个月内签订正式合作协议

### 📦 方法论对比表

| 维度 | 传统做法 | 任务分解做法 |
|------|---------|------------|
| 目标下达 | 直接分配数字指标（每人22个） | 量化推导到日常活动（每天2个拜访） |
| 分工方式 | 平均分配 | 按员工特长分配不同漏斗阶段 |
| 进度管理 | 半年后发现问题 | 周/月/季度三级检查点 |
| 激励机制 | 仅关注最终结果 | 过程行为 + 阶段成就双重激励 |
| 问题诊断 | 凭感觉打气 | 数据驱动，定位漏斗哪个环节断裂 |

### 📝 避坑指南

- ⚠️ 只下达数字目标而不拆解过程，员工"努力但方向错"，半年后才发现偏差已无法挽回。
- ⚠️ WBS 分解停在"子任务"层就停手，没有落到"日常活动"层，执行者仍然不知道每天具体做什么。
- ⚠️ 时间轴按月均匀切分，忽略淡旺季差异，导致Q4压力过大或Q2资源浪费。
- ⚠️ 转化率假设不做验证就直接用，推导出的日均任务量严重失真，需结合历史数据校准。
- ⚠️ 所有人做同样的事（都去打电话+拜访），没有按漏斗阶段分工，擅长成交的人浪费在线索开发上。
- ⚠️ SVG 卡片中图形与文字内容重叠，Prompt 中已明确要求「图形呈现在单独区域，不与其它文字内容重叠」，若出现此问题需在追问中强调。
