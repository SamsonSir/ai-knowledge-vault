### 📋 核心分析

**战略价值**: BabyVision 是首个系统量化多模态大模型「纯视觉原子能力」的评测集，用 22 个子任务揭示当前顶尖模型的视觉能力尚不及 3 岁幼儿，为具身智能与世界模型的下一步突破提供可诊断的靶点。

**核心逻辑**:

- **语言捷径是核心陷阱**：若视觉题可被完整语言化，模型会绕过视觉能力靠文本推理通关，看起来「会看」实则走语言捷径。BabyVision 的设计原则就是强制排除这条捷径。
- **BabyVision-Mini 对比实验**：20 道视觉中心任务，同时交给 3/6/10/12 岁儿童和顶尖多模态模型作答。结果：大多数模型得分低于 3 岁儿童基线；Gemini 2.0 Flash Preview 是唯一稳定超过 3 岁基线的模型，但仍比 6 岁儿童低约 20 个百分点。
- **BabyVision Full 人类基线**：16 位本科及以上背景测试者完成全量 388 题，人类准确率 94.1%；闭源最强 Gemini 2.0 Flash Preview 仅 49.7%；开源最强 Qwen2.5-VL-72B-Thinking 仅 22.2%；多数开源模型集中在 12–19% 区间。
- **差距是系统性的，非单点缺陷**：四大类能力（精细辨别、视觉追踪、空间感知、视觉模式识别）全线下滑，说明模型缺的是视觉地基，而非某个局部能力。
- **「Unspeakable」现象**：BabyVision 中的题目无法在不损失信息的情况下被完整语言化。模型把视觉压缩成 token 时，细节在压缩中消失，这是根本性的结构缺陷。
- **数据构建严苛**：参考儿童认知教材和视觉发育测验 → 4-5 个种子示例 → 逆向图像搜索爬取约 4000 张候选图 → 人工标注 + 双盲质检（两位独立专家交叉审核，异议退回修改，无法达成一致则弃用）→ 最终产出 388 道题，覆盖 22 种子任务。
- **BabyVisionGen 新方向**：从 388 题中重新标注出 280 道适合「生成式作答」的题，要求模型输出图像/视频表达解题过程，自动评测工具与人工评测一致性达 96%。测试模型包括 NanoBanana Pro、QwenImage、Veo3、Sora2，结论：生成式推理在视觉追踪、精细辨别等任务上出现「更像人类」的行为，但整体仍无法稳定达到完全正确解。
- **具身智能的必修课**：「很难想象一个视觉能力低于 3 岁孩子的机器人，能够可靠地在真实物理世界里帮助人类。」BabyVision 的 22 个原子能力直接对应具身智能落地的基础需求。
- **Count 3D Blocks 是最典型的全员翻车子任务**：多模型在该子任务上普遍偏低，暴露模型在结构化三维场景理解上的系统性不足。

### 🎯 关键洞察

**4 类「Unspeakable」挑战的具体机制**：

**挑战 1：非语言细节（Fine-grained Discrimination）**
- 典型题型：拼图/补全题，选项差别仅为微小边界、局部凸起、像素级错位。
- 人类机制：凭几何直觉「对齐边界」秒选。
- 模型失败机制：把形状语言化为「像钩子、两个腿、差不多七八个六边形」后，细节被抹平，选项在 token 空间里变得「几乎一样」，无法区分。

**挑战 2：追线追丢（Manifold Understanding / Visual Tracking）**
- 典型题型：连线/绕线/轨迹题，答案编码在「连通性」里。
- 人类机制：锁定一条线 → 穿过交叉 → 一路追到终点（3 岁幼儿用手指沿线划即可完成）。
- 模型失败机制：把线翻译成「左/右/上/下」的离散步骤，遇到交叉点出现分叉爆炸，容易「换轨」追错线。
- 具体案例：垃圾分类连线题（A-蓝、B-黄、C-绿），Gemini 2.0 Flash Preview 答成 A-绿、B-黄、C-蓝，写了一大段「逐段追踪」推理，最终仍把两条路径接反。

**挑战 3：缺少真正的空间想象（Spatial Imagination）**
- 典型题型：三维方块计数、视角投影、遮挡下的结构判断。
- 人类机制：把结构在脑中「立起来」，换角度看再数，不依赖语言描述。
- 模型失败机制：两类典型错误——漏掉隐藏块、投影关系搞错。根因是缺少稳定的 3D 内部表征与变换能力，而非逻辑推理差。

**挑战 4：图形规律归纳（Visual Pattern Induction）**
- 典型题型：从少量视觉示例中抽象规则，迁移到新图。
- 人类机制：做关系映射，关注「发生了什么变化」而非「那里有什么」，形状/颜色/绝对位置可变，变换中的「身份」不变。
- 模型失败机制：盯着表面属性（颜色、形状），把「结构规则」误读成「外观统计」，迁移时产生幻觉规则。

### 🛠️ 数据构建流程

1. **需求定义**：参考儿童认知教材和视觉发育测验，梳理出 4 大类共 22 种基础视觉子任务。
2. **种子采集**：每个子技能挑选 4–5 个种子示例（种子图片）作为典型代表。
3. **大规模爬取**：基于种子示例，利用逆向图像搜索 + 关键词搜索，从互联网爬取约 4000 张候选图片；严格遵守版权规范，仅选非商业/学术用途素材；过滤含大量文字说明或需要文化常识的图片。
4. **人工标注**：多名专业人员逐一检查，筛除不适合出题的样本；对保留图片精心设计问题和标准答案；每个问题附详细「解题过程」说明，证明答案可由视觉推理得出。
5. **双盲质检**：两位独立专家交叉审核，双方均认可才收录；出现异议退回修改；反复无法达成一致则弃用。
6. **最终产出**：388 道高质量视觉题目，覆盖 22 种子任务。

### 📋 核心分析

**战略价值**: Niji V7 于2026年1月9日正式上线，在线条质量、提示词遵循度、材质光泽感三个维度有肉眼可见的代际提升，是动漫/插画方向 AI 绘图工具链的新首选节点。

**核心逻辑**:

- **图像质量全面升级**：连贯性大幅增强，眼睛反射细节、背景飘落花瓣等微观元素精准呈现，相当于一次"高清升级"，近景 Waifu 眼睛光晕效果显著改善。
- **提示词遵循能力增强**：模型更偏向字面理解，可准确区分"左边红色立方体/右边蓝色立方体"的空间位置，可绘制"四只手臂各拿冰淇淋"等高度具体的请求。注意：原先强调氛围感的提示词需重新调整，因为模型现在更倾向字面执行。
- **线条质量突破**：杂乱无序的"线头"大幅减少，更接近手绘/板绘质感，简单插画线条勾勒更精确。可用 `anime screenshot` 提示词专门体验线条优化效果。
- **简约风格支持**：Niji V7 有意识追求简洁，支持大面积留白，可用 `minimalist graphic logo` 提示词体验。AI 艺术中"简单画面最难"的悖论在此版本有所突破。
- **默认风格平面化**：默认降低了渲染量（减少3D感），呈现更平面化外观，以展示底层绘图的连贯性，线条与大面积平涂区域结合效果更出色。
- **sref（风格参考）表现出色**：Niji V7 与 Midjourney 的 sref code 可以共用，且在 Niji V7 上体验感优于 Midjourney 本体，加持下提示词随便写也能出好图。
- **cref（角色参考）暂不支持**：官方表示正在开发"超级神秘惊喜"替代方案，目前该功能缺失，需注意工作流规避。
- **即将上线功能**：个性化（Personalization）与情绪板（Moodboards）功能即将推出。
- **局限性依然存在**：大全景/复杂场景线条仍会碎/糊在一起；中式东方风格学得杂，效果不如即梦，中文提示词在中式场景效果更好但时不时男女不分；首饰等细节仍有瑕疵；材质/形体/物种要求叠加多了出图质量下降。
- **手部细节改善**：手指细节有明显进步，但仔细看仍有瑕疵，可结合 NBP（Negative Prompt Booster）修复。

### 🛠️ 操作流程

1. **准备阶段**
   - 访问官方网页端：`https://www.midjourney.com/imagine`
   - 在界面切换模型至 **Niji V7**（界面有专属切换入口）
   - 确认当前不支持 cref，规避角色参考需求

2. **核心执行**
   - 单角色中景构图优先（方便查看细节，后续扩图/放大均方便）
   - 基础参数模板：`--chaos 5 --ar 16:9 --stylize 400~800 --niji 7`
   - stylize 值：400 = 平衡细节与风格；800 = 风格化更强
   - 需要风格锁定时追加：`--sref [code]`
   - 手部/细节有瑕疵时：交给 NBP 修复

3. **验证与优化**
   - 检查线条是否清晰（大全景/复杂场景线条碎是已知问题）
   - 检查提示词中的空间位置描述是否被正确执行（V7 字面理解增强）
   - 原先氛围感提示词若效果变差，需重写为更具体的描述
   - 中式场景若效果不佳，切换即梦或改用中文提示词

### 📝 避坑指南

- ⚠️ **cref 暂不可用**：角色参考功能在 Niji V7 中不支持，官方正在开发替代方案，现阶段工作流需绕开此功能。

const aiOutput = items[0].json.output || $node["Agent"].json.output;

function robustParse(input) {
    if (typeof input === 'object' && input !== null) return input;
    
    let str = String(input).trim();
    // 1. 去除 Markdown 代码块标记
    str = str.replace(/^```json\s*/i, '').replace(/^```\s*/i, '').replace(/\s*```$/, '');
    
    // 2. 尝试标准解析
    try { return JSON.parse(str); } catch (e) {}
    
    // 3. 修复被截断的 JSON 数组
    if (str.startsWith('[') && !str.endsWith(']')) {
        for (let i = str.length - 1; i >= 0; i--) {
            if (str[i] === '}') {
                const potentialJson = str.substring(0, i + 1) + ']';
                try {
                    const res = JSON.parse(potentialJson);
                    if (Array.isArray(res) && res.length > 0) return res;
                } catch (e) {}
            }
        }
    }
    
    // 4. JS 引擎执行
    try { return (new Function("return (" + str + ");"))(); } catch (e) {}
    
    // 5. 文本兜底：不以 { 或 [ 开头，封装为对象
    if (!str.startsWith('{') && !str.startsWith('[')) {
        return { "text_content": str };
    }

    return null;
}

try {
    if (!aiOutput) throw new Error("AI 没有返回 output 字段");
    
    const parsedData = robustParse(aiOutput);
    
    if (!parsedData) {
        const preview = aiOutput.length > 100 ? aiOutput.slice(0, 100) + "..." : aiOutput;
        throw new Error(`解析彻底失败，原始内容: ${preview}`);
    }

    const resultItems = Array.isArray(parsedData) ? parsedData : [parsedData];
    return resultItems.map(item => ({ json: item }));

} catch (error) {
    throw new Error(`JSON解析错误: ${error.message}`);
}

## 📓 文章 6

> 文档 ID: `MC6TwLmnkiQU8GkvoA0c4JvWnqf`

**来源**: 高手对话AI秘诀：别让 AI 猜，你让它问 | **时间**: 2026-01-11 | **原文链接**: https://mp.weixin.qq.com/s/RS1x9DRx...

---

### 📋 核心分析

**战略价值**: 用"反问式提示词"强迫AI在执行前对齐需求，从根源消灭返工，把模糊意图转化为可交付任务。

**核心逻辑**:

- 提示词越长不等于越好：《Lost in the Middle》研究证明，关键信息出现在长文本中间时模型更容易忽略，50条背景里真正决定成败的3条会被淹没
- AI输出差的根因不是AI不聪明，是你给的是"让AI猜领导意思"的残缺指令，而不是完整的交付说明
- Chain-of-Thought研究验证：引导模型先确认边界、目标、格式、约束再动手，复杂任务表现显著提升
- 2025年研究数据支撑：向生成式AI求助时提问质量更高的人，写作任务表现更好——会问的人效率更高，不会问的人只会堆字数
- 核心操作只有一句话："在你开始执行之前，请先问我几个问题，帮助你把任务做得更好"
- 1-3-1法则把AI的10个问题压缩成4句回答：1个交付物 + 3个关键变量（受众/场景/风格）+ 1个可交付版本指令
- "先提问"看似多两分钟，实际节省的是职场最贵的成本：返工时间（整体可节省约一小时）
- 这个方法的副产品是训练"把模糊需求变成可交付任务"的能力，写邮件、做汇报、对齐需求全部适用
- 返工循环的本质是：先生成→不满意→加长提示词→继续重写→越改越乱，而反问式打断了这个循环
- AI是镜子：给清晰任务得清晰结果，给模糊意图只能猜

---

### 🎯 关键洞察

**为什么"先提问"反而更快**：

原因：人在急着出结果时倾向于跳过需求对齐，直接生成，但信息不完整导致输出偏差，触发返工循环。每次返工不只是重写一次，而是重新理解+重新组织+重新生成，时间成本是指数级的。

动作：在任务开始前强制插入一个"提问-回答"环节，把不确定点变成明确的可执行条件。

结果：第一次生成的版本就接近可用，后续只需少量微调，总耗时反而更短。

### 🛠️ 操作流程

**1. 准备阶段（1-3-1法则，答4句话开工）**

- 第1步：确认1个核心交付物 → 周报 / 纪要 / 方案 / 邮件 / 汇报稿，选一个
- 第2步：准备3个关键变量
  - 给谁看（受众：领导/客户/同事）
  - 用在哪（场景：内部汇报/客户提案/团队同步）
  - 要什么感觉（风格：正式/简洁/有立场）
- 第3步：贴上收尾句，触发生成

**2. 核心执行（3个职场场景照抄）**

场景1 — 写周报（从模板套话→像你本人）：
```
我需要写周报。在生成前请先问我关键问题，确保内容具体、贴合上级关注点，避免空话。
```
准备材料：本周做了什么（3条）+ 结果/进度（过程数据也行）+ 下周风险与需要支持

场景2 — 会议纪要（从流水账→行动清单）：
```
请先确认：纪要给谁看？需要多正式？是否必须包含行动项+负责人+截止时间？确认后再生成。
```

场景3 — 写方案（从正确但空→有立场、可落地）：
```
在写方案前，请先问我：成功标准、时间/预算限制、不可碰的红线、可用资源。问清楚后再输出。
```

**3. 验证与优化**

- 收到AI生成版本后，优先检查它列出的"假设清单"，有偏差的直接纠正
- 不满意时不要加长提示词，而是针对具体偏差点单独追问
- 如果AI一口气问了超过8个问题，用1-3-1法则只回答最核心的4句，其余让AI合理假设
