Google Stitch 大改版：对着画布说话就能做设计
Google 的 AI 设计工具 Stitch 昨天发了一次大更新，从 Labs 实验项目正式升级为 AI 原生设计平台。
不是加个小功能那种升级，是整个产品形态变了。
从一个"用 AI 生成 UI 图片"的工具，直接进化成了一个 AI 原生的软件设计平台。Google 给这
个新方向起了个名字，叫 Vibe Design。
市场反应很直接： Figma 股价当天跌了 8.8%。
7e2b2dfc8750233cf04cbf4f4007ea90.png

这个反应不算夸张。因为 Stitch 这次做的事情，确实踩到了 Figma 最核心的地盘：UI 设计 + 原型 + 协作。而且它选了一条完全不同的路：你不用从线框图开始，直接告诉它"我要做一个健身 App，用户打开就能看到今天的训练计划"，几秒钟出高保真界面。

0b2e34adgaaafeahyof3f5uvbx6dgppqamya.f10002.mp4
Stitch 是什么
Stitch 是 Google Labs 去年 Google I/O 2025 上发布的一个 AI UI 设计工具。最早的能力很简单：你用自然语言描述想要的界面，它帮你生成高保真的 UI 设计稿，可以导出到 Figma 或者直接生成前端代码。
听起来不错，但之前的 Stitch 本质上还是个"生成器"，出的是静态图片。跟真正的设计工具比，差的不是一点半点。
这次更新把这个差距大幅缩小了。
五个核心升级
AI 原生画布
Stitch 重新设计了整个界面，换成了一个基于节点的无限画布。跟传统设计工具最大的区别：这个画布能同时放图片、代码和文字，而且 AI 能理解它们之间的关系。
你可以把一张竞品截图、一份 PRD 文档、一段代码同时丢到画布上，告诉 Stitch"参考这些给我做一个新版本"。它能把这些不同类型的输入当作创作的种子，直接出设计。

0bc344c4qaafq4ai4w56pvuvlz6dzdtqlsaa.f10002.mp4
新加的 Agent Manager（代理管理器）能帮你同时跑多个设计方向。比如你同时在探索三种不同的首页布局，每个方向都有独立的进度追踪，点一下就能跳到对应的屏幕。这个功能的价值在于：设计本身就是一个不断发散再收敛的过程，以前你只能一个方向一个方向试，现在可以让 AI 同时帮你跑五个方向，你来挑最好的那个。
还加了浅色模式，处理深色界面设计的时候对比度更好。
更聪明的设计代理
新的设计代理能理解整个画布的上下文。之前的 AI 设计助手大多是"你选中一个元素，告诉它改什么"，一次只看一个局部。Stitch 新代理看到的是你整个项目的演变过程，所有屏幕、所有版本、所有素材。

0bc3rqawmaab4uacaq52rjuvddgdm2gaczqa.f10002.mp4
你可以这样跟它说：
• "把 Logo 换成我刚上传的那个"：它知道你说的是哪个 Logo，在哪个屏幕上，甚至知道这个 Logo 在多少个页面里用到了
• "根据这些屏幕生成一份产品简介放到画布上"：它看完你所有屏幕的内容，自动提炼核心功能点和卖点，直接输出文案
• "面试我，帮我为我的品牌设计一个新的登陆页面"：它会反过来问你问题，了解你的品牌调性、目标用户、核心诉求，然后再动手设计
• "做这些屏幕的移动端版本"：直接生成对应的手机端设计，移动端和桌面端可以在同一个画布上共存，再也不用来回切项目了
语音设计（预览版）
这个功能我觉得是最有想象力的。你可以直接对着画布说话，Stitch 能"看到"你正在做什么、你选中了什么。
实际用起来是这样的：你一边浏览自己的设计，一边说"这个菜单给我三个不同的样式"，"把这个屏幕的配色换成暖色调"，"导航到仪表盘那个页面"。Stitch 同时处理你的多个请求，你说完一句它就开始做，不用等一个做完再说下一个。
它还能当设计评审用。你说"给我评一下这个页面的设计"，它会从视觉层级、信息密度、交互逻辑几个维度给你反馈。以前找人评设计，要约会议、等排期、解释背景，现在随时开口就行。

OpenClaw Skill技能系统 - 3分钟秒懂教程
用最简单的比喻，让你秒懂什么是 Skill 技能系统

有没有想过，为什么有些人用OpenClaw特别顺手，三两下就能搞定复杂的活儿，而你让小龙虾帮忙做事，它总是"听不懂人话"？
秘密可能就藏在一个叫 Skill（技能） 的小东西里。
今天咱们用大白话聊聊这个 Skill 到底是啥，怎么用，保证你看完就能明白。

一、Skill 到底是个啥？
可以把 OpenClaw 想象成一个超级能干的助手。问题是，每次你让它做同样的事情，都得重新解释一遍。这就好比你每次做宫保鸡丁都要重新查菜谱——烦不烦？
Skill 就是这个菜谱。
把某个任务的步骤写下来存好。下次要做这个任务时，只要说"按那个菜谱来"，OpenClaw 就知道该怎么做了。
image.png

用生活例子来说
教宠物做动作
OpenClaw = 你的智能宠物
Skill = 训练教程
想教宠物"取快递"，不用每次都手把手教。写个教程：
听快递员打电话
去门口
接过快递
说谢谢
学会了这个技能，以后快递来了它自己去。
手机装 APP
OpenClaw = 你的手机
Skill = APP
手机出厂时只能打电话发短信。装了微信能聊天，装了地图能导航。每个 APP 都是一个新技能。

一点点历史：Skill 是怎么来的？
回顾下Skill的来龙去脉：
image.png

2025年10月之前
AI 助手虽然聪明，但有个问题：每次要做同样的事，都要重新告诉它怎么做。开发者得写代码来扩展 AI 的能力，门槛很高。那时候大家都在收集提示词，以便每次问AI的时候，复制给它。
2025年10月中旬
Anthropic（就是做 Claude 的那家公司）搞了个东西叫 Claude Skills。他们想：能不能让每个人都能给 AI 加新能力，不用重复写提示词？核心就是一个 SKILL.md 文件，用 Markdown 写就行，谁都能看懂。这降低了门槛——不用写或者收集提示词也能扩展 AI 能力。
2025年12月18日
Anthropic 把 Agent Skills 发布为开放标准（agentskills.io）。这意味着：
不只是 Claude 能用
其他 AI 也能接入这个标准
大家可以共享技能包
现在（2026年）
像 OpenClaw 这样的开源 AI 助手都支持 Skill 系统。你能在 ClawHub 上找到成千上万个别人写好的技能，直接拿来用。
从技术路线的历史发展，可以清晰看出为什么要发明出Skill这个概念。

二、一张 Skill 卡长什么样？
每张技能卡就是一个文件夹，里面有一本"技能说明书"：
skill/              ← 技能文件夹
  ├── SKILL.md              ← 技能说明书（必须）
  ├── scripts/              ← 技能脚本（可选）
  ├── references/           ← 参考资料库（可选）
  └── assets/               ← 其他资源（可选）
OpenClaw 的 Skill 文件长什么样？
看个实际例子：

---
name: 周报助手
description: 每周五自动生成工作周报
version: 1.0.0
author: 你的名字
tags: [办公, 效率]
allowed-tools: [file-read, file-write]
trigger:
  type: scheduled
  schedule: "0 17 * * 5"  # 每周五下午5点
---
# 周报助手
## 触发条件
- 每周五下午5点自动运行
- 或者用户说"/周报"、"生成周报"
## 执行步骤
1. **读取本周工作记录**
   - 检查 `~/Documents/工作日志/` 目录
   - 读取本周的所有日志文件
2. **整理内容**
   - 提取本周完成的事项
   - 列出遇到的问题
   - 收集下周计划
3. **生成周报**
   - 用简洁的语言总结
   - 分成三个部分：完成、问题、计划
4. **保存文件**
   - 保存到 `~/Documents/周报/2026-03-第X周.md`
   - 发送通知告诉我周报已生成
这个文件的几个部分：
文件头
name: 技能的名字
description: 一句话说明这是干什么的
version: 版本号
author: 谁写的
tags: 标签，方便搜索
allowed-tools: 允许用哪些工具
trigger: 什么时候触发（定时、手动、自动）
文件主体
触发条件
执行步骤
注意事项
Skill的本质就是喂给
